Este desafío le guiará a través del proceso de construcción y envío de una dapp de recaudación de fondos en Stellar usando Soroban. A diferencia de las aplicaciones tradicionales de crowdfunding, crowdfunding dapps (aplicaciones descentralizadas) proporcionan los medios para que los usuarios comprometan fondos a una campaña de crowdfund directamente desde sus carteras digitales, sin necesidad de intermediarios.

En este desafío, aprenderás cómo implementar contratos inteligentes en Futurenet, y cómo interactuar con ellos a través de una interfaz web. En este contexto, el término "nave" se refiere a la finalización del proceso de desarrollo de su dapp, asegurando que funciona como se espera, y es accesible para la interacción y prueba del usuario a través de una interfaz alojada. Sin embargo, es crucial aclarar que, a pesar de su funcionalidad, el dapp no está promocionado ni destinado a desplegarse en un ajuste a nivel de producción en Futurenet. El desafío está diseñado para propósitos educativos, ayudándote a entender cómo se puede construir e interactuar con un dapp con una mayor personalización y desarrollo, tiene el potencial de evolucionarse hacia una solución de financiación completa y lista para usar.

## Checkpoint 0: 📦 Instalar 📚

Comience instalando las dependencias requeridas. También querrá estar seguro de que tiene instalada la versión más actualizada de Rust.

Requerido:

- `soroban-cli alias` (instalado en el siguiente paso)
- `Node` v18: [Descarga Node](https://nodejs.org/en/download/)
- `Cartera de cartera`: [Cartera de cartera](https://freighter.app/)

Primero, clona el repositorio Soroban Dapps Challenge y echa un vistazo a la sucursal `crowdfund`, que contiene el código del contrato inteligente de crowdfund que potencia este dapp:

```sh
git clone https://github.com/stellar/soroban-dapps-challenge.git
cd soroban-dapps-challenge
git checkout crowdfund
```

Luego, instale el alias soroban-cli ejecutando el siguiente comando:

```sh
instalación_soroban carga
```

Soroban CLI es la interfaz de línea de comandos para Soroban. Permite construir, desplegar e interactuar con contratos inteligentes, configurar identidades, generar pares de claves, administrar redes y más. El soroban-cli (alias) que se utiliza en este desafío es una versión fijada del soroban-cli que se utiliza en el Desafío de Dapps Soroban. Esto asegura que el desafío sea reproducible y que todos los participantes estén utilizando la misma versión de Soroban.

## Punto de control 1: 🎬 Desplegar Contratos Inteligentes

Ahora que tiene la sucursal de Crowdfund, es el momento de desplegar los contratos inteligentes en un entorno Sandbox. Desplegar un contrato inteligente en una configuración de producción implica enviar el código de contrato a la red principal de blockchain ( Mainnet ), donde se convierte en parte del contador inmutable de la cadena. Desplegar contratos inteligentes a un entorno Sandbox simula ese proceso sin afectar realmente a Mainnet. Cuando despliegues los contratos inteligentes, en su lugar desplegarás en Futurenet, una red de pruebas con características de vanguardia que aún no han sido implementadas en el Mainnet.

En su terminal, cargue los contratos e inicialícelos en el entorno Sandbox ejecutando los siguientes comandos:

```sh
configuración de ejecución npm
```

Si el comando se ejecuta correctamente, su terminal devolverá una serie de mensajes notificándole sobre la exitosa inicialización de los contratos y la secuencia de posinstalación.

```sh
Contrato desplegado con éxito con ID: CBXHU5BWWTOCZRYX3DMSKCFG7B3K2YG2I5F75ALPQ6GCY6ZES2XKLTI
Despliegue el contrato de recaudación
Contrato desplegado exitosamente con ID: CBKY7UN5VGD4LIQFOBOTSUSQWK67BZZTA23NIEVWSWRR5SAT26JQN2BN
Initialice el contrato de ficha de abundancia

Inicialice el contrato de fondos crowdfund

Done

> soroban-example-dapp@0. .0 contratos de construcción
...
```

El ID del contrato es un identificador único para un contrato inteligente desplegado en una cadena de bloques. Este ID de contrato se utiliza para interactuar y hacer referencia al contrato inteligente, permitiendo a los usuarios invocar funciones del contrato inteligente, enviar transacciones, o interactuar de otro modo con las funcionalidades y datos del contrato inteligente almacenados en la cadena de bloques.

## Punto de control 2: 🤝 Conecta el Frontend al Backend

Ahora que ha desplegado el contrato inteligente, es hora de revisar el frontend de su dapp. El frontend es la interfaz del navegador donde los colaboradores de su campaña de crowdfund conectarán sus carteras digitales y activos de compromiso a la causa de la campaña.

Primero, iniciar el servidor de desarrollo:

```sh
npm run dev
```

Ahora abre tu navegador y visita <http://localhost:3000>. Deberías ser capaz de ver la parte frontal de tu aplicación.

> Nota: Siga las instrucciones de abajo y asegúrese de que ha financiado su dirección de cartera que pretende utilizar desde el navegador, de lo contrario la pantalla dapp estará en blanco una 'Cuenta no encontrada' se imprimirá sólo en la consola del navegador.

Ahora que tienes el frontend corriendo, es el momento de conectarlo con el backend, tu contrato inteligente, que define las reglas y la lógica de la campaña de crowdfund, incluyendo la función de aceptar contribuciones. Si desea profundizar en las especificaciones del contrato, eche un vistazo al video recorrido del código de contrato [here](https://youtu.be/vTz0CQYnMRQ?t=260&feature=shared).

Necesitará añadir algunas lumens de la red Futurenet a su cartera para interactuar con la dapp. Visita https://laboratory.stellar.org/#account-creator?network=futurenet, y sigue las instrucciones para crear y o depositar fondos en una cuenta en Futurenet. Recuerde que estos son lumens de prueba para su uso en Futurenet y no pueden ser usados en Mainnet.

## Punto de control 4: 🚢 ¡Envíalo! 🚁

Ahora que tu dapp es totalmente funcional, es hora de desplegarlo en un entorno de producción. En este paso, aprenderás a desplegar tu dapp en Vercel, una plataforma en la nube para sitios estáticos que ofrece una forma rápida y eficaz de desplegar el frontend de su dapp. Esta sección requiere que tenga una [cuenta de Vercel] y el Vercel cli instalado.

Si no tiene instalado Vercel cli, ejecute el siguiente comando para instalarlo globalmente:

```sh
npm i --global vercel
```

[cuenta de vercel]: https://vercel.com/login

A continuación, necesitará eliminar el directorio `target` para ahorrar espacio para la implementación. Ejecuta el siguiente comando para eliminar el directorio `target`:

```sh
rm -rf target
```

> Nota: Puedes construir este directorio de nuevo ejecutando `soroban contract build` en el directorio `contracts/abundance`.

Luego, elimina cualquier directorio `.vercel` existente en tu proyecto para asegurarte de que estás empezando con una pizarra limpia:

```bash
rm -rf .vercel
```

Luego, ejecuta el siguiente comando para desplegar tu ejemplo dapp:

```bash
vercel --prod
```

Vercel le pedirá que vincule su proyecto local a un nuevo proyecto Vercel. Siga las respuestas a las siguientes preguntas para asegurarse de que su proyecto local está correctamente vinculado a un nuevo proyecto Vercel:

```bash
? ¿Configurar “~/Documents/GitHub/test/soroban-dapps-challenge”? [Y/n] y
? ¿Cuál ámbito debería contener tu proyecto? <VERCEL_UNAME>
? ¿Enlace al proyecto existente? [y/N] n
? ¿Cuál es el nombre de tu proyecto? <PROJECT_NAME>
? ¿En qué directorio se encuentra tu código? ./
```

Luego, continúe a través de las instrucciones (no necesitará modificar ninguna configuración) hasta que alcance el mensaje de finalización similar a lo siguiente:

```sh
🔗 Enlazado a Execuian-dev28/soroban-example-dapp (created .vercel)
🔍 Inspect: https://vercel.com/Thursian-dev28/soroban-example-dapp/C1YZVQSqh6WcyR1uvxz8q2tW1tjD [2s]
✅ Producción: https://soroban-example-dapp-rho.vercel.app [42s]
```

:::tip

Por favor, guarde su url de producción, la necesitará para completar el reto.

## Punto de control 6: ✅ Compruebe su trabajo!

Para completar con éxito este desafío, su trabajo debe ser revisado. Por favor, siga los siguientes pasos:

1. Fork [el repositorio de desafíos](https://github.com/stellar/soroban-dapps-challenge/fork).
2. Rellene el archivo `crowdfund/challenge/output.txt` con su dirección de billetera. El archivo rellenado debe tener el siguiente aspecto:

```sh
Clave pública: GBSXUXZSA2VEXN5VGOWE5ODAJLC575JCMWRJ4FFRDWSTRCJ123456789
```

3. Crea una Pull Request a la sucursal `stellar/soroban-dapps-challenge/crowdfund`. Cuando se crea el PR, las acciones de CI comprobarán los datos del archivo `crowdfund/challenge/output.txt` y actualizarán su progreso.
4. Espere los resultados del pipeline CI/CD.
5. Corregir errores si está presente:

- encontrar la razón de error en los resultados del desafío CI de Crowdfund (puede encontrar un enlace directamente en la solicitud de pull request);
- volver a tu repositorio bifurcado;
- corregir errores y cambios de confirmación. Se comprobará de nuevo el PR existente.

6. Si el pipeline fue exitoso, entonces enhorabuena! ¡Has completado el desafío!👏
