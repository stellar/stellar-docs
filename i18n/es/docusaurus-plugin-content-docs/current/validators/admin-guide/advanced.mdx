## Creando tu propia red privada

El [stellar-core GitHub repository] contiene [un archivo `testnet.md`] que contiene un breve tutorial que demuestra cómo configurar y ejecutar una red de pruebas aislada de corta duración.

## Información de tiempo de ejecución: inicio y parada

Stellar-core puede iniciarse directamente desde la línea de comandos, o a través de un sistema de supervisión como `init`, `upstart` o `systemd`.

Stellar-core puede ser finalizado elegantemente en cualquier momento entregando `SIGINT` o presionando `CTRL-C`. Puede ser interrumpido por la fuerza con `SIGTERM` o `SIGKILL`. Este último puede dejar un archivo de bloqueo obsoleto en el `BUCKET_DIR_PATH`, y puede que necesite eliminar este archivo antes de que stellar-core se reinicie. En términos amplios, todos los componentes están diseñados para recuperarse de la terminación abrupta.

Stellar-core también puede empaquetarse en un sistema de contenedores como Docker, siempre y cuando `BUCKET_DIR_PATH` y la base de datos se almacenen en volúmenes persistentes. Para un ejemplo, vea [`stellar/quickstart`].
