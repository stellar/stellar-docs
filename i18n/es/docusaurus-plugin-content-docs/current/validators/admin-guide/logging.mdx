Stellar Core envía registros a error estándar y `stellar-core.log` por defecto, configurable con el campo `LOG_FILE_PATH`.

Los mensajes de registro se clasifican por niveles de _prioridad_ progresivos: `TRACE`, `DEBUG`, `INFO`, `WARNING`, `ERROR` y `FATAL`. El sistema de registro solo emite esos mensajes en o por encima de su nivel de registro configurado. El sistema de registro solo emite aquellos mensajes en su nivel de registro configurado o superior.

Los mensajes de registro en diferentes niveles de prioridad se pueden codificar por color en el error estándar configurando `LOG_COLOR=true` en el archivo de configuración. Por defecto, no están codificados por colores.

El nivel de registro puede ser controlado por configuración, el flag de línea de comandos `--ll`, o ajustado dinámicamente mediante comandos administrativos (HTTP). Para hacerlo, ejecuta: Para hacerlo, ejecuta:

```bash
sudo -u stellar stellar-core --conf /etc/stellar/stellar-core.cfg http-command "ll?level=debug"
```

mientras tu nodo esté en ejecución.

Los niveles de registro también se pueden ajustar en base a cada partición a través de la interfaz administrativa. Por ejemplo, el sistema de historial puede ser configurado para registro de nivel `DEBUG` ejecutando:

```bash
sudo -u stellar stellar-core --conf /etc/stellar/stellar-core.cfg http-command "ll?level=debug&partition=history"
```

Contra un sistema en funcionamiento.

<Alert>

Por favor, revisa el [referencia de Comandos HTTP] para más información sobre las particiones disponibles para usar en el comando `ll`.

</Alert>

El nivel de registro por defecto es `INFO`, que es moderadamente detallado y debería emitir mensajes de progreso cada pocos segundos durante el funcionamiento normal.
