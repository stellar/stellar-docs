Stellar Core envía registros a error estándar y `stellar-core.log` por defecto, configurable con el campo `LOG_FILE_PATH`.

Los mensajes de registro son clasificados por niveles progresivos de _prioridad_: `TRACE`, `VERBOSE`, `DEBUG`, `INFO`, `WARNING`, `ERROR`, y `FATAL`. El sistema de registro sólo emite esos mensajes a o por encima de su nivel de registro configurado.

Los mensajes de registro a diferentes niveles de prioridad pueden ser codificados en color en error estándar estableciendo `LOG_COLOR=true` en el archivo de configuración. Por defecto no están codificados por colores.

El nivel de registro puede ser controlado por configuración, la línea de comandos `-ll`, o ajustado dinámicamente por comandos administrativos (HTTP). Para hacerlo, ejecutar:

```bash
$ comando stellar-core http-comando "ll?level=debug"
```

mientras el nodo se está ejecutando.

Los niveles de registro también se pueden ajustar sobre una base de partición-por-partición a través de la interfaz administrativa. Por ejemplo, el sistema histórico puede establecerse en `DEBUG`-level logging ejecutando:

```bash
comando http-core stellar-core "ll?level=debug&partition=history"
```

Contra un sistema en ejecución.

<Alert>

Por favor, eche un vistazo a la [referencia de comandos HTTP] para más información sobre las particiones disponibles para su uso en el comando `ll`.

</Alert>

El nivel de registro por defecto es `INFO`, que es moderadamente detallado y debería emitir mensajes de progreso cada pocos segundos bajo una operación normal.

[Referencia de comandos HTTP]: https://github.com/stellar/stellar-core/blob/master/docs/software/commands.md#http-commands
