Hay tres formas comunes de instalar y ejecutar Stellar Core:

1. **Utilizar [packages]publicado (#package-based-installation).** Para uso de producción, recomendamos instalar Stellar Core usando paquetes publicados.

2. **Construir desde [source](#installing-from-source).** En algunas situaciones poco comunes, o si quieres sacrificar la conveniencia para el máximo control, puede ser necesario construir desde el origen.

3. **Usa una [imagen de Docker](#docker-based-installation).** Usar una imagen de Docker es el método más rápido y fácil, así que es una buena elección para muchos desarrolladores.

## Versión de lanzamiento

Sea cual sea el método que utilice, debe asegurarse de instalar la última [release](https://github.com/stellar/stellar-core/releases) ya que estas compilaciones son compatibles con versiones anteriores y son acumulativas.

El esquema de números de versión que seguimos es `protocol_version.release_number.patch_number`, donde:

- `protocol_version` es la versión máxima de protocolo soportada por esa versión (todas las versiones son 100% compatibles con versiones posteriores),
- `release_number` es golpeado cuando un conjunto de nuevas características o correcciones de errores que no afecten al protocolo se incluyen en la versión, y
- `patch_number` es usado cuando una corrección crítica tiene que ser desplegada.

## Instalación basada en paquetes

Si estás usando una versión reciente de Ubuntu, proporcionamos las últimas versiones estables de [`stellar-core`](https://github.com/stellar/stellar-core) y [`stellar-horizon`](https://github.com/stellar/go/tree/master/services/horizon) en formato de paquete binario de Debian.

Puede elegir instalar estos paquetes individualmente, lo que ofrece la mayor flexibilidad, pero requiere la creación **manual** de los archivos de configuración relevantes y la configuración de una base de datos **PostgreSQL**.

## Instalando desde la fuente

El repositorio de código fuente de Stellar Core contiene instrucciones extensas y exhaustivas para construir el software desde el código fuente. Por favor [echa un vistazo a `INSTALL.md`](https://github.com/stellar/stellar-core/blob/master/INSTALL.md) para más información.

## Instalación basada en Docker

### Entornos de desarrollo

SDF mantiene una [imagen de inicio rápido](https://github.com/stellar/quickstart) que incluye el "Cautive Core" de Stellar con Horizon y las bases de datos necesarias de PostgreSQL. Es una forma rápida de configurar una configuración efemeral predeterminada, no validada, que debería funcionar para la mayoría de los desarrolladores. Adicionalmente, la imagen de inicio rápido puede ser preconfigurada para ser usada como un nodo de red principal, Testnet, Futurenet o local.

<Alert>

La imagen de inicio rápido no está pensada para servir como un nodo de instancia a nivel de producción. Por favor, planifica la(s) instancia(s) de producción cuidadosamente.

</Alert>

### Entornos de producción

SDF también mantiene una imagen independiente de Stellar-Core-solo, [`stellar/stellar-core`](https://hub.docker.com/r/stellar/stellar-core).

Ejemplo:

```bash
docker run stellar/stellar-core:latest help
docker run stellar/stellar-core:latest gen-seed
```

Para ejecutar el daemon del núcleo estelar, necesita proporcionar un archivo de configuración:

```bash
# Inicializar postgres DB (ver opción de configuración de DATABASE )
docker run -v "/path/to/config/dir:/etc/stellar/" stellar/stellar-core:latest new-db
# Ejecuta un stellar-core daemon en segundo plano
docker run -d -v "/path/to/config/dir:/etc/stellar/" stellar/stellar-core:latest run
```

La imagen utiliza paquetes deb, por lo que es posible confirmar la suma de comprobación del binario `stellar-core` en la imagen docker coincide con lo que figura en el paquete deb firmado con cifrado. Consulte [documentación de paquetes](https://github.com/stellar/packages/) para obtener información sobre la instalación de los paquetes de Ubuntu. Para calcular esta suma de verificación en la imagen docker que se puede ejecutar:

```bash
docker run --entrypoint=/bin/sha256sum stellar/stellar-core:latest /usr/bin/stellar-core
```
