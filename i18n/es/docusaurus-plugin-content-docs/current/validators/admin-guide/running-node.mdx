## Iniciando tu nodo

Una vez que hayas [configurado tu entorno](./prerequisites.mdx), [configurado tu nodo](./configuring.mdx), configura tu [conjunto de quorum](./configurando. dx#eligiendo-tu-quorum-set), y los archivos seleccionados para `obtener` [historia de](./environment-preparation.mdx#history-archives), estás listo para iniciar Stellar Core.

Usar un comando equivalente a:

```bash
ejecución estelar-núcleo
```

En este punto, estás listo para observar la actividad de tu nodo principal a medida que se une a la red.

Es posible que quieras saltarte adelante y revisar la página [Logging](./logging.mdx) para familiarizarte con la salida de Stellar Core.

## Interactuando con tu instancia

Cuando el nodo se está ejecutando, puede interactuar con Stellar Core a través de un punto final HTTP administrativo. Los comandos pueden ser emitidos usando herramientas HTTP de línea de comandos como `curl`, o ejecutando un comando CLI como

```bash
comando <http-command> stellar-core http-core
```

El endpoint HTTP [no está pensado para ser expuesto a la Internet pública](./prerequisites.mdx#internal-system-access). Normalmente es accesible por los administradores o por una aplicación de nivel medio para enviar transacciones a la red Stellar.

Detalles de los comandos de endpoint _HTTP_ disponibles pueden encontrarse en [stellar-core-GitHub repo](https://github.com/stellar/stellar-core/blob/master/docs/software/commands.md#http-commands). Adicionalmente, [la página de comandos](./commands.mdx) contiene una visión general de algunos de los comandos _CLI_ más útiles (comandos no HTTP) para administrar un nodo core en ejecución.

## Uniéndose a la red

Su nodo pasará por las siguientes fases mientras se une a la red, y puede consultar la salida a continuación usando el endpoint informativo mencionado [here](. monitoring.mdx#general-node-información) :

### Establecer la conexión con otros pares.

Deberías ver el aumento de `authated_count`.

```json
"peers" : {
  "autenticated_count" : 3,
  "pending_count" : 4
},
```

### Consenso de observación

Hasta que el nodo vea un quórum, dirá:

```json
"estado" : "Uniéndose a SCP"
```

Después de observar el consenso, un nuevo campo `quorum` mostrará información sobre las decisiones de red. En este punto, el nodo cambiará a un estado "_Capturando hacia arriba_":

```json
"quorum" : {
  "qset" : {
    "ledger" : 22267866,
    "cost" : 20883268,
    "de acuerdo" : 5,
    "delayed" : 0,
    "disagree" : 0,
    "fail_at" : 3,
    "hash" : "980a24",
    "lag_ms" : 430,
    "ausente" : 0,
    "fase" : "EXTERNALIZE"
  },
  "transitivo" : {
    "intersection" : true,
    "last_check_ledger" : 22267866,
    "node_count" : 21
  }
},
"estado" : "Poniendo al día",
```

### Poniendo al día

Esta es una fase en la que el nodo descarga datos de cualquier archivo configurado. Esta fase comienza con algo como:

```json
"state" : "Papagando",
"status" : [ "Papelear: Punto de control de espera (ETA: 35 segundos)" ]
```

Y luego pasará por las diversas fases de descarga y aplicación del estado, como:

```json
"state" : "Papagando",
"status" : [ "Papagando: descargando archivos ledger 20094/119803 (16%)" ]
```

Puede especificar cuán lejos en el tiempo va su nodo a ponerse al día en su archivo de configuración. Si estableces el campo `CATCHUP_COMPLETE` en `true`, tu nodo reproducirá la _completa historia_ de la red, que puede llevar mucho tiempo. Semanas. Puede acelerar el proceso copiando archivos existentes de otro validador completo, pero sólo necesita repetir el historial de red completo si está configurando un Validador Completo (o posiblemente si necesita volver a hacer el estado Horizon). De lo contrario, puede especificar un punto de partida para catchup usando el campo `CATCHUP_RECENT`. Vea la [completa configuración de ejemplo] para más detalles.

:::info

Los campos de configuración `CATCHUP_COMPLETE` y `CATCHUP_RECENT` son mutuamente excluyentes. Si el campo `CATCHUP_COMPLETE` está establecido en `true`, el campo `CATCHUP_RECENT` será ignorado.

:::

### Sincronizado

Cuando el nodo haya terminado de ponerse al día, su estado cambiará a:

```json
"estado" : "¡Sincronizado!"
```
