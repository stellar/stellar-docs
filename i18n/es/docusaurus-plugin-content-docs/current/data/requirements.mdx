Aquí puede encontrar los requisitos para ejecutar los diversos tipos de nodos que operan la red Stellar. Ofrecemos algunas recomendaciones modestas para cada tipo de recurso, así como un SKU generalmente aplicable para algunos proveedores de computación en la nube populares. Estas pueden requerir personalizaciones o ajustes, dependiendo de su caso de uso específico.

| Tipo de nodo               | CPU                     | RAM   | Disco                                                                            | AWS SKU       | SKU de Google Cloud | Guía de Admin                                          |
| -------------------------- | ----------------------- | ----- | -------------------------------------------------------------------------------- | ------------- | ------------------- | ------------------------------------------------------ |
| Nodo Validador del Núcleo  | 8x Intel Xeon @ 3.4 GHz | 16 GB | SSD de 100 GB NVMe                                                               | [c5d.2xlarge] | [n4-highcpu-8]      | [Haga clic aquí](../validators/admin-guide/README.mdx) |
| Servicio de API de Horizon | 4 vCPU                  | 16 GB | 100 GB SSD >= 1,5K IOPS                                                          | [c5d.xlarge]  | [n4-standard-4]     | [Haz clic aquí](./horizon/admin-guide/overview.mdx)    |
| Horizon PostgreSQL         | 4 vCPU                  | 32 GB | 2 TB\* SSD (NVMe o Almacenamiento Adjunto Directo) >= 7K IOPS | [i4g.xlarge]  | [c3-highmem-8]      | [Haz clic aquí](./horizon/admin-guide/overview.mdx)    |
| RPC Soroban                | 2 vCPU                  | 4 GB  | 30 GB volumen persistente >= 3K IOPS                                             | [c5.large]    | [n4-highcpu-2]      | [Haz clic aquí](./rpc/admin-guide.mdx)                 |

_\* Asumiendo una ventana de retención de 30 días para el almacenamiento de datos._
