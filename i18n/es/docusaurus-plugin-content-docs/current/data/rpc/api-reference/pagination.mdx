title: Paginación
---

La paginación en soroban-rpc es similar a la paginación en Horizon.

Para los métodos que lo soportan, los argumentos de paginación se pasan como un argumento de objeto final con dos valores:

- `cursor`: string - (opcional) Un ID de cadena que apunta a una ubicación específica en una colección de respuestas y se extrae del valor de paginación_token de un registro. Cuando se proporciona un cursor, Soroban-RPC \_no_ incluirá el elemento cuyo id coincide con el cursor en la respuesta. Sólo los elementos que aparecen _después_ del cursor están incluidos.
- `limit`: número - (opcional) El número máximo de registros devueltos. El límite para [getEvents](./methods/getEvents.mdx) puede variar de 1 a 10000 - un límite superior que está hardcoded en Soroban-RPC por razones de rendimiento. Si este argumento no está designado, el valor predeterminado es 100.

Por ejemplo, llamar a un método con un conjunto de parámetros de paginación:

```json
{
  "jsonrpc": "2. ",
  "id": "1",
  "method": "exampleMethod",
  "params": {
    "some": "argument",
    "pagination": {
      "cursor": "1234-1",
      "límite": 100
    }
  }
}
```
