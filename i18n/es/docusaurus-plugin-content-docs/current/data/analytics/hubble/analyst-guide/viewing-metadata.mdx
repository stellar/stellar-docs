title: Visualizando Metadatos

---

Hubble publica metadatos que pueden ayudar a los usuarios a determinar qué tablas consultar, con qué frecuencia se actualiza el conjunto de datos y información general sobre el conjunto de datos.

Hay dos formas de acceder a esta información:

## Explorador de BigQuery

Al acceder a Hubble desde su enlace destacado, el panel del Explorador cargará metadatos sobre el conjunto de datos `crypto-stellar.crypto_stellar`.

Usa el Toggle para ver el contenido del Conjunto de Datos. Hacer clic en el nombre de una tabla cargará lo siguiente:

- _Esquema_ - información detallada sobre el esquema de la tabla, incluyendo definiciones de columnas y tipos de datos. Visualizar el esquema ayuda a escribir una consulta SQL
- _Detalles_ - información general sobre la propia tabla, incluyendo particionado, agrupamiento y tamaño de la tabla. Visualizar detalles ayuda con la optimización de consultas
- _Vista previa_ - datos de muestra en bruto de la tabla. Los datos presentados son equivalentes a ejecutar una declaración `SELECT *`

## INFORMATION_SCHEMA

BigQuery admite vistas definidas por el sistema y de solo lectura que proporcionan información de metadatos sobre los objetos de BigQuery. Se pueden consultar las vistas a través de SQL desde la interfaz de usuario de BigQuery o desde las Bibliotecas de Clientes.

:::note

Las consultas ejecutadas contra el `INFORMATION_SCHEMA` no pueden ser almacenadas en caché y **incurrirán** en cargos por procesamiento de datos por cada ejecución.

:::

Desde el Editor de BigQuery, la siguiente consulta listará todas las tablas en Hubble:

<CodeExample>

```sql
# List all tables in Hubble
#standardSQL
select *
from `crypto-stellar.crypto_stellar`.INFORMATION_SCHEMA.TABLES;
```

</CodeExample>

Si quieres detalles sobre una tabla en particular, puedes devolver el esquema de la tabla:

<CodeExample>

```sql
# List all columns for the accounts table
select table_name,
  column_name,
  is_nullable,
  data_type,
  is_partitioning_column
from `crypto-stellar.crypto_stellar`.INFORMATION_SCHEMA.COLUMNS
where table_name = "accounts";
```

</CodeExample>

Más sobre el `INFORMATION_SCHEMA` se puede encontrar [aquí](https://cloud.google.com/bigquery/docs/information-schema-intro).
