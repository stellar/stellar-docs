| -------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------- | ------------------- | ----------- | ----- |
| contract_id          | ID del contrato Soroban                                                                                                                                                                                                                                                                         | cadena         |                     | Sí          |       |
| contract_key_type    | Tipo de clave de contrato que es un ScVal que puede tener los siguientes valores                                                                                                                                                                                                                | cadena         |                     |             |       |
| contract_durability  | El contrato puede ser temporal o persistente                                                                                                                                                                                                                                                    | cadena         |                     |             |       |
| asset_code           | La representación del código del activo en la red, de 4 o 12 caracteres.                                                                                                                                                                                                                        | cadena         |                     |             |       |
| asset_issuer         | La dirección de la cuenta del emisor original del activo que creó el activo.                                                                                                                                                                                                                    | cadena         |                     |             |       |
| asset_type           | El identificador para el tipo de código del activo, puede ser alfanumérico con 4 caracteres, 12 caracteres o el activo nativo de la red, XLM.                                                                                                                                                   | cadena         |                     |             |       |
| balance_holder       | La dirección/cuenta que posee el saldo del activo en los datos del contrato.                                                                                                                                                                                                                    | cadena         |                     |             |       |
| balance              | La cantidad de unidades de XLM que posee la cuenta.                                                                                                                                                                                                                                             | cadena         |                     |             |       |
| last_modified_ledger | El número de secuencia del ledger cuando se modificó la entrada del ledger (este firmante único para la cuenta). Las eliminaciones no cuentan como una modificación y mostrarán el número de secuencia de modificación previo.                                                                  | entero         |                     | Sí          |       |
| ledger_entry_change  | Código que describe el tipo de cambio aplicado a la entrada del ledger.                                                                                                                                                                                                                         | entero         |                     | Sí          |       |
| deleted              | Indica si la entrada del ledger (identificador del saldo) ha sido eliminada o no. Una vez que una entrada es eliminada, no puede recuperarse.                                                                                                                                                   | booleano       |                     | Sí          |       |
| batch_id             | Representación en cadena del run id para un DAG dado en Airflow. Tiene la forma de `scheduled__<batch_end_date>-<dag_alias>`. Los ids de lote son únicos para cada lote y ayudan con la monitorización y capacidades de reejecución.                                                            | cadena         |                     | Sí          |       |
| batch_run_date       | La fecha de inicio para el intervalo del lote. Junto con la fecha en el batch_id, la fecha representa el intervalo de ledgers procesados. La fecha de ejecución del lote puede considerarse como un proxy de closed_at para un ledger.                                                          | fecha y hora   |                     | Sí          |       |
| batch_insert_ts      | La marca temporal en UTC cuando un lote de registros fue insertado en la base de datos. Este campo puede ayudar a identificar si un lote se ejecutó en tiempo real o como parte de una ejecución retroactiva.                                                                                   | marca temporal |                     | Sí          |       |
| closed_at            | Marca temporal en UTC cuando este ledger cerró y fue confirmado en la red. Se espera que los ledgers cierren aproximadamente cada 5 segundos.                                                                                                                                                   | marca temporal |                     | Sí          |       |
| ledger_sequence      | El número de secuencia de este ledger. Representa el orden del ledger dentro de la blockchain de Stellar. Cada ledger tiene un número de secuencia único que incrementa con cada nuevo ledger, asegurando que los ledgers se procesen en el orden correcto.                                     | entero         |                     | Sí          |       |
| ledger_key_hash      | Hash de la clave del ledger utilizado para identificar entradas de ledger de datos de contrato o código de contrato caducando.                                                                                                                                                                  | cadena         |                     | Sí          |       |
| key                  | La clave codificada usada para identificar un dato específico del contrato. La clave codificada tiene dos componentes (tipo y valor), donde tipo describe el tipo de dato y valor describe el valor codificado del tipo de dato para los datos del contrato.                                    | json           |                     |             |       |
| key_decoded          | La versión legible o decodificada de la clave. Esto proporciona un formato comprensible de la clave, facilitando su interpretación y uso en análisis.                                                                                                                                           | json           |                     |             |       |
| val                  | El valor codificado asociado con la clave en los datos del contrato. El valor codificado tiene dos componentes (tipo y valor), donde tipo describe el tipo de dato y valor describe el valor codificado del tipo de dato para los datos del contrato.                                           | json           |                     |             |       |
| val_decoded          | La versión legible o decodificada del valor. Esto proporciona una representación clara y comprensible del valor, facilitando su interpretación y uso en análisis.                                                                                                                               | json           |                     |             |       |
| contract_data_xdr    | La codificación XDR (Representación Externa de Datos) de los datos del contrato. XDR es un formato estándar usado para serializar y deserializar datos, asegurando interoperabilidad entre sistemas distintos. Este campo contiene los datos serializados en bruto del contrato en formato XDR. | cadena         |                     |             |       |
