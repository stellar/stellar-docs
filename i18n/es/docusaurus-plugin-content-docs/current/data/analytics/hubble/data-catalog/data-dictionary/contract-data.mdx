## Metadatos de la tabla

| Propiedad             | Configuración                                                                                                |
| --------------------- | ------------------------------------------------------------------------------------------------------------ |
| Clave(s) natural(es)  | ledger_key_hash, closed_at                                                                                   |
| Campo(s) de partición | closed_at (partición por MES)                                                                                |
| Campo(s) agrupado(s)  | last_modified_ledger, contract_id                                                                            |
| Documentación         | [dbt docs](http://www.stellar-dbt-docs.com/#!/source/source.stellar_dbt_public.crypto_stellar.contract_data) |

## Detalles de la columna

| Nombre                           | Descripción                                                                                                                                                                                                                                                                                          | Tipo de dato    | Valores de dominio | ¿Requerido? | Notas |
| -------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------- | ------------------ | ----------- | ----- |
| contract_id                      | ID de contrato de Soroban                                                                                                                                                                                                                                                                            | cadena          |                    | Sí          |       |
| tipo_de_clave_del_contrato       | Tipo de clave del contrato que es un ScVal que puede tener los siguientes valores                                                                                                                                                                                                                    | cadena          |                    |             |       |
| durabilidad_del_contrato         | El contrato puede ser temporal o persistente                                                                                                                                                                                                                                                         | cadena          |                    |             |       |
| código_de_activo                 | El código de representación de 4 o 12 caracteres del activo en la red.                                                                                                                                                                                                                               | cadena          |                    |             |       |
| emisor_del_activo                | La dirección de cuenta del emisor original del activo que creó el activo.                                                                                                                                                                                                                            | cadena          |                    |             |       |
| tipo_de_activo                   | El identificador del tipo de código de activo, puede ser un alfanumérico de 4 caracteres, 12 caracteres o el activo nativo de la red, XLM.                                                                                                                                                           | cadena          |                    |             |       |
| titular_del_balance              | La dirección/cuenta que sostiene el balance del activo en los datos del contrato                                                                                                                                                                                                                     | cadena          |                    |             |       |
| balance                          | El número de unidades de XLM sostenidas por la cuenta                                                                                                                                                                                                                                                | cadena          |                    |             |       |
| last_modified_ledger             | El número de secuencia del libro mayor cuando se modificó la entrada del libro mayor (este firmante único para la cuenta). Las eliminaciones no cuentan como una modificación y mostrarán el número de secuencia de modificación anterior                                                            | número entero   |                    | Sí          |       |
| cambio_de_entrada_en_libro_mayor | Código que describe el tipo de cambio de entrada en el libro mayor que se aplicó a la entrada del libro mayor.                                                                                                                                                                                       | número entero   |                    | Sí          |       |
| eliminado                        | Indica si la entrada en el libro mayor (id de balance) ha sido eliminada o no. Una vez que se elimina una entrada, no se puede recuperar.                                                                                                                                                            | booleano        |                    | Sí          |       |
| batch_id                         | Representación en cadena del id de ejecución para un DAG dado en Airflow. Toma la forma de "scheduled\_\_\<batch_end_date>-\<dag_alias>". Los ids de lote son únicos para el lote y ayudan con las capacidades de monitoreo y reejecución                                                            | cadena          |                    | Sí          |       |
| batch_run_date                   | La fecha de inicio para el intervalo de lotes. Cuando se toma con la fecha en el batch_id, la fecha representa el intervalo de libros mayores procesados. La fecha de ejecución del lote se puede ver como un proxy de closed_at para un libro mayor.                                                | fecha y hora    |                    | Sí          |       |
| batch_insert_ts                  | La marca de tiempo en UTC cuando se insertó un lote de registros en la base de datos. Este campo puede ayudar a identificar si un lote se ejecutó en tiempo real o como parte de un backfill                                                                                                         | marca de tiempo |                    | Sí          |       |
| closed_at                        | Marca de tiempo en UTC cuando este libro mayor se cerró y se comprometió a la red. Se espera que los libros mayores se cierren aproximadamente cada 5 segundos                                                                                                                                       | marca de tiempo |                    | Sí          |       |
| secuencia_del_libro_mayor        | El número de secuencia de este libro mayor. Representa el orden del libro mayor dentro de la blockchain de Stellar. Cada libro mayor tiene un número de secuencia único que se incrementa con cada nuevo libro mayor, asegurando que los libros mayores se procesen en el orden correcto.            | número entero   |                    | Sí          |       |
| ledger_key_hash                  | Hash de clave del libro mayor utilizado para identificar datos del contrato que están caducando o entradas en el libro mayor del código de contrato                                                                                                                                                  | cadena          |                    | Sí          |       |
| clave                            | La clave codificada utilizada para identificar un elemento específico de datos del contrato. La clave codificada tiene dos componentes (tipo y valor) donde el tipo describe el tipo de dato y el valor describe el valor codificado del tipo de dato para los datos del contrato                    | json            |                    |             |       |
| key_decoded                      | La versión legible por humanos o decodificada de la clave. Esto proporciona un formato comprensible de la clave, facilitando su interpretación y uso en el análisis                                                                                                                                  | json            |                    |             |       |
| val                              | El valor codificado asociado con la clave en los datos del contrato. El valor codificado tiene dos componentes (tipo y valor) donde el tipo describe el tipo de dato y el valor describe el valor codificado del tipo de dato para los datos del contrato                                            | json            |                    |             |       |
| val_decoded                      | La versión legible por humanos o decodificada del valor. Esto proporciona una representación clara y comprensible del valor, facilitando su interpretación y uso en el análisis                                                                                                                      | json            |                    |             |       |
| contract_data_xdr                | La codificación XDR (Representación de Datos Externa) de los datos del contrato. XDR es un formato estándar utilizado para serializar y deserializar datos, asegurando la interoperabilidad entre diferentes sistemas. Este campo contiene los datos del contrato en formato XDR crudo y serializado | cadena          |                    |             |       |
