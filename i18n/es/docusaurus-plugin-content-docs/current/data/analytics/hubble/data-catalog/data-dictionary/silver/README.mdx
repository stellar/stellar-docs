Datos decodificados, transformados y filtrados. Estas tablas transforman las tablas bronce en tablas más fáciles de usar al unir y aplanar tablas y filtrar información para casos de uso específicos.

## Cuándo usar

Estas tablas son ideales para una experiencia más curada mientras se necesita buscar y filtrar para casos de uso específicos, como información sobre un contrato específico.

Un ejemplo es obtener eventos relevantes de Stellar para calcular el suministro circulante usando la tabla `token_transfers_raw`. En teoría, puedes obtener la misma información de la tabla bronce `history_contract_events`, pero la tabla `token_transfers_raw` ya ha procesado, filtrado y aplanado los eventos relevantes para eventos SEP-41 y operaciones clásicas.

## Dimensiones de Cambio Lento (SCD Tipo 2)

Algunas tablas silver están modeladas como tablas [SCD Tipo 2](https://github.com/stellar/stellar-dbt-public/blob/master/docs/snapshot.md). Esto significa que rastrean el historial completo de cambios de un registro en el tiempo manteniendo múltiples versiones con rangos `valid_from` / `valid_to`.

Estas tablas son especialmente útiles para responder preguntas históricas como:

- ¿Cuál era el estado de un contrato en un momento dado?
- ¿Cómo evolucionó una cuenta, una trustline o un balance a lo largo de diferentes períodos?
- ¿Qué Fondos de Liquidez estaban activos en una fecha determinada y cómo cambiaron sus propiedades con el tiempo?

> **_NOTA:_** Frecuencia de snapshot diaria: Los snapshots se ejecutan diariamente, lo que significa que capturan cambios interdiarios, pero no múltiples cambios que ocurren dentro del mismo día.
