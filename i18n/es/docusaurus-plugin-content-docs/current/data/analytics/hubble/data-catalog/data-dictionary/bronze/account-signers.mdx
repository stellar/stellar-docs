## Metadatos de la tabla

| Propiedad             | Configuración                                                                                                  |
| --------------------- | -------------------------------------------------------------------------------------------------------------- |
| Clave(s) natural(es)  | account_id, signer, closed_at                                                                                  |
| Campo(s) de partición | batch_run_date (partición MENSUAL)                                                                             |
| Campo(s) agrupado(s)  | account_id, signer, last_modified_ledger                                                                       |
| Documentación         | [dbt docs](http://www.stellar-dbt-docs.com/#!/source/source.stellar_dbt_public.crypto_stellar.account_signers) |

## Detalles de columnas

| Nombre               | Descripción                                                                                                                                                                                                                                              | Tipo de dato | Valores de dominio | ¿Requerido? | Notas |
| -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------ | ------------------ | ----------- | ----- |
| account_id           | El identificador único de la cuenta.                                                                                                                                                                                                                     | string       |                    | Sí          |       |
| signer               | La dirección de la cuenta autorizada para firmar (autorizar) transacciones en nombre de otra cuenta. Este proceso se denomina multi-sig.                                                                                                                 | string       |                    | Sí          |       |
| weight               | El peso numérico del firmante. Todos los pesos en una transacción se suman para determinar si la transacción cumple con el umbral requerido para ser completada.                                                                                         | float        |                    | Sí          |       |
| sponsor              | La dirección de la cuenta del patrocinador que paga las reservas de esta entrada del ledger.                                                                                                                                                             | string       |                    | No          |       |
| last_modified_ledger | El número de secuencia del ledger cuando se modificó la entrada del ledger (este firmante único para la cuenta). Las eliminaciones no cuentan como modificaciones y mostrarán el número de secuencia de la modificación previa.                          | integer      |                    | Sí          |       |
| ledger_entry_change  | Código que describe el tipo de cambio aplicado a la entrada del ledger.                                                                                                                                                                                  | integer      |                    | Sí          |       |
| deleted              | Indica si la entrada del ledger (balance id) ha sido eliminada o no. Una vez eliminada una entrada, no puede recuperarse.                                                                                                                                | boolean      |                    | Sí          |       |
| batch_id             | Representación en cadena del ID de ejecución para un DAG dado en Airflow. Toma la forma de `scheduled__<batch_end_date>-<dag_alias>`. Los IDs de batch son únicos para cada lote y ayudan en el monitoreo y capacidades de reejecución.                  | string       |                    | Sí          |       |
| batch_run_date       | La fecha de inicio para el intervalo del batch. Cuando se toma junto con la fecha en batch_id, representa el intervalo de ledgers procesados. La fecha de ejecución del batch puede considerarse un sustituto de closed_at para un ledger.               | datetime     |                    | Sí          |       |
| batch_insert_ts      | El timestamp en UTC cuando un lote de registros fue insertado en la base de datos. Este campo ayuda a identificar si un lote se ejecutó en tiempo real o como parte de un relleno (backfill).                                                            | timestamp    |                    | Sí          |       |
| closed_at            | Timestamp en UTC cuando este ledger se cerró y comprometió en la red. Se espera que los ledgers se cierren aproximadamente cada 5 segundos.                                                                                                              | timestamp    |                    | Sí          |       |
| ledger_sequence      | El número de secuencia de este ledger. Representa el orden del ledger dentro de la blockchain Stellar. Cada ledger tiene un número de secuencia único que incrementa con cada nuevo ledger, asegurando que los ledgers se procesen en el orden correcto. | integer      |                    | Sí          |       |
