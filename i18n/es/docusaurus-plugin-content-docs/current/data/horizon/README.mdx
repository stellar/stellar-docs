A partir del 1 de agosto de 2024, la SDF truncará los datos históricos sobre el horizonte propiedad de la SDF a un año. Esta actualización nos permite optimizar el rendimiento y asegurar una experiencia mejorada para todos los usuarios. Le animamos a explorar los proveedores de ecosistemas de terceros de Horizon, que pueden proporcionar una ventana de retención de historia más larga así como otras características.

:::

Horizon proporciona una API HTTP a los datos de la red Stellar. Incorpora y vuelve a servir los datos producidos por la red Stellar en una forma que es más fácil de consumir por la aplicación media relativa a las representaciones de datos orientadas al rendimiento utilizadas por Stellar Core. Esta API sirve el puente entre aplicaciones y [Stellar Core](../../validators/README.mdx). Proyectos como carteras, bolsas descentralizadas y emisores de activos utilizan Horizon para enviar transacciones, consultar un saldo de cuenta o transmitir eventos como transacciones a una cuenta.

Se puede acceder al horizonte a través de cURL, un navegador o uno de los [SDKs Stellar](../../tools/sdks/library.mdx). Para reducir la complejidad de su proyecto, le recomendamos que utilice un SDK en lugar de hacer llamadas directas de API.

Esta guía describe cómo administrar una instancia de Horizon de producción (consulte el [Blog de los desarrolladores](https://www.stellar). rg/developopers-blog/a-new-sun-on-the-horizon) para algunos antecedentes sobre el rendimiento y las mejoras arquitectónicas de esta gran versión bump). Para obtener información sobre cómo desarrollar en el código base de Horizon, consulta la [Guía de desarrollo](https://github.com/stellar/go/blob/master/services/horizon/internal/docs/GUIDE_FOR_DEVELOPERS.md).

Antes de empezar, vale la pena reiterar el sentimiento que resuena en el [Nodo Principal](../../validators/README. documentación dx): **no endosa la ejecución de Horizon respaldada por una instancia independiente del núcleo estelar**, y especialmente no por un núcleo estelar _validando_ Stellar. Se trata de dos preocupaciones separadas, y desacoplarlas es importante tanto para la fiabilidad como para el rendimiento. En su lugar, el horizonte administra el suyo, versión de Stellar Core optimizada para su propio subconjunto de necesidades (nos referiremos a esto como una instancia de "Cautive Core").

## ¿Por qué ejecutar Horizon?

Ejecutar Horizon dentro de su propia infraestructura proporciona una serie de beneficios. Puedes:

- Tener un control operativo completo sin depender de la Fundación de Desarrollo Estelar para datos de red y envío de transacciones a redes;
- Ejecutar múltiples instancias para redundancia y escalabilidad.

La Stellar Development Foundation (SDF) gestiona dos instancias de Horizonte:

- [https://Texton-testnet.stellar.org/](https://parallon-testnet.stellar.org/) por interactuar con la [testnet](../../learn/fundamentals/networks.mdx)
- [https://horizonton-futurenet.stellar.org/](https://Texton-futurenet.stellar.org/) por interactuar con la [futurenet](../../learn/fundamentals/networks.mdx)
