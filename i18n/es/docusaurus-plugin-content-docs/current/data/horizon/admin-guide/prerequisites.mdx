El servicio Horizon es responsable de sincronizar con la red Stellar y procesar los datos del libro de notas. Para entender el alcance de los servicios de Horizon, por favor lea la sección [configuring](./configuring.mdx) antes de pasar a los requisitos previos para el cálculo.

El servicio Horizon puede ser [installed](./installing.mdx) en metal desnudo o en una máquina virtual. Está soportado nativamente tanto en sistemas operativos Linux como en Windows.

## Modelo de despliegue de una sola instancia

Para una configuración básica usando el [modelo de Despliegue de una sola Instancia](./configuring.mdx#single-instance-deployment), necesitarás una suma de dos perfiles de computación distintos:

- Uno para alojar el servicio Horizon
- Otro para alojar el servidor PostgreSQL

### Requisitos de hardware

Las especificaciones mínimas de hardware para ejecutar eficazmente Horizon son las siguientes:

#### Instancia Horizon Calcular :

| Tipo de nodo               | CPU    | RAM   | Disco                 | AWS SKU      | SKU de Google Cloud |
| -------------------------- | ------ | ----- | --------------------- | ------------ | ------------------- |
| Servicio de API de Horizon | 4 vCPU | 16 GB | 100 GB SSD >= 3K IOPS | [c5d.xlarge] | [n4-standard-4]     |

_\* Asumiendo una ventana de retención de 30 días para el almacenamiento de datos._

#### Instancia de calculo de la base de datos PostgreSQL:

| Tipo de nodo       | CPU    | RAM   | Disco                                                                            | AWS SKU      | SKU de Google Cloud |
| ------------------ | ------ | ----- | -------------------------------------------------------------------------------- | ------------ | ------------------- |
| Horizon PostgreSQL | 4 vCPU | 32 GB | 2 TB\* SSD (NVMe o Almacenamiento Adjunto Directo) >= 7K IOPS | [i4g.xlarge] | [c3-highmem-8]      |

_\* Asumiendo una ventana de retención de 30 días para el almacenamiento de datos._

Tenga en cuenta que se requiere un mínimo de la versión 12 de PostgreSQL.

Estas especificaciones asumen una ventana de retención de 30 días para el almacenamiento de datos. Para una ventana de retención más larga, los requisitos del sistema serán más altos. Para obtener más información sobre la ingestión de datos, retención de historial y gestión de almacenamiento, consulte la sección [ingestion](./ingestion.mdx).

## Despliegue de múltiples instancias

Para lograr una alta disponibilidad, redundancia y alto rendimiento, consulte la documentación [scaling](./scaling.mdx). Proporciona una visión detallada de varias estrategias de despliegue que puede emplear, dependiendo del SLA que necesite su instancia Horizon para lograr.

## Acceso a la red

- Asegúrese de que la instancia Horizon puede establecer una conexión con la instancia de base de datos PostgreSQL. El puerto predeterminado para PostgreSQL es 5432.

- Se requiere una conexión de red estable y rápida con Internet para cualquier instancia Horizon que ejecute el rol de ingestión. Esto es para asegurar que tiene conectividad de salida eficiente a hosts remotos en el [conjunto de quorum](../../../validators/admin-guide/configuring.mdx#choosing-your-quorum-set) y [URL de archivo](../../../validators/admin-guide/environment-preparation.mdx#history-archives) para la red Stellar elegida. Durante la ingesión, la instancia Horizon se comunica con estos equipos, recibiendo datos de transacción de red a través de su subproceso del núcleo cautivo local.

:::note

Los requisitos de hardware pueden aumentar a medida que la red estelar crezca y/o si estás compartiendo recursos o usando configuraciones personalizadas.
