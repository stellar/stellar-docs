Horizon es un componente central de la plataforma Stellar: proporciona una API HTTP a los datos de la red Stellar. Incorpora y vuelve a servir los datos producidos por la red Stellar en una forma que es más fácil de consumir por la aplicación media relativa a las representaciones de datos orientadas al rendimiento utilizadas por Stellar Core.

Esta guía describe cómo administrar una instancia de Horizon de producción (consulte el [Blog de los desarrolladores](https://www.stellar). rg/developopers-blog/a-new-sun-on-the-horizon) para algunos antecedentes sobre el rendimiento y las mejoras arquitectónicas de esta gran versión bump). Para obtener información sobre cómo desarrollar en el código base de Horizon, consulta la [Guía de desarrollo](https://github.com/stellar/go/blob/master/services/horizon/internal/docs/GUIDE_FOR_DEVELOPERS.md).

Antes de empezar, vale la pena reiterar el sentimiento que resuena en [Ejecutar un Nodo Principal](../../../validadators/README. Guía dx): **no endosamos Horizon respaldado por una instancia independiente del núcleo estelar**, y especialmente no por un núcleo estelar _validando_ Stellar. Se trata de dos preocupaciones separadas, y desacoplarlas es importante tanto para la fiabilidad como para el rendimiento. En su lugar, el horizonte administra el suyo, versión de Stellar Core optimizada para su propio subconjunto de necesidades (nos referiremos a esto como una instancia de "Cautive Core").

## ¿Por qué ejecutar Horizon?

Ejecutar Horizon dentro de su propia infraestructura proporciona una serie de beneficios. Puedes:

- Tener un control operativo completo sin depender de la Fundación de Desarrollo Estelar para datos de red y envío de transacciones a redes;
- Ejecutar múltiples instancias para redundancia y escalabilidad.
