Para instalar Horizon en entornos de producción o no de desarrollo, recomendamos lo siguiente basado en la infraestructura de destino:

### Diploma Barata

- Si el host es Debian Linux, instale binarios precompilados [desde repositorios](#package-manager) usando un gestor de paquetes.
- Para cualquier otro host, descarga [binarios de lanzamiento preconstruidos](#prebuilt-releases) de Stellar Horizon y Core para la arquitectura y el sistema de operación del host o [compile desde la fuente](https://github.com/stellar/go/blob/master/services/horizon/internal/docs/GUIDE_FOR_DEVELOPERS.md#building-horizon).

### Contenedor

- No Orchestrated: si el entorno de despliegue objetivo no incluye un orquestador de contenedores como Kubernetes, entonces esto significa que tiene la intención de ejecutar la imagen de liberación Horizon desde [dockerhub. om/stellar/stellar-horizon](https://hub.docker.com/r/stellar/stellar-horizon) como contenedor directamente con Docker daemon en el anfitrión. Elige la etiqueta de la imagen Horizon para la versión específica de lanzamiento y luego tira la imagen usando `docker pull stellar/stellar-horizon:<tag_version>` para llevarla localmente al host.
- Orchestrated: cuando el entorno de destino tiene orquestación de contenedores, como el clúster de Kubernetes, recomendamos usar el [Horizon Helm Chart](https://github. om/estelar/casco charts/árbol/main/charts/horizon) para gestionar la instalación y el ciclo de vida de implementación de la imagen Horizonte como contenedor(es) en el clúster.

Para instalar Horizon en entornos de desarrollo, consulte [Horizon README](https://github.com/stellar/go/blob/master/services/horizon/README.md#try-it-out) desde el repositorio de código fuente para ver las opciones disponibles.

### Notas sobre la instalación

#### Gestor de paquetes

SDF publica nuevas versiones en sus repositorios Ubuntu personalizados. Sigue [esta guía](https://github.com/stellar/packages/blob/master/docs/adding-the-sdf-stable-repository-to-your-system.md#adding-the-sdf-stable-repository-to-your-system) para añadir el repositorio SDF estable a tu sistema host. Si está interesado en instalar versiones candidatas de software que todavía tienen acceso a estable, consulte [Agregar el repositorio de pruebas de Bleeding Edge Testing Repository](https://github.com/stellar/packages/blob/master/docs/adding-the-sdf-stable-repository-to-your-system.md#adding-the-bleeding-edge-testing-repository). Por último, [instalar paquete](https://github.com/stellar/packages/blob/master/docs/installing-individual-packages.md#installing-individual-packages) describe los diversos comandos que estos paquetes hacen disponibles.

Para continuar con la instalación:

<CodeExample>

```bash
sudo apt update
sudo apt install stellar-horizonon stellar-core
```

</CodeExample>

#### Versiones precompiladas

Consulte la lista de [Horizon releases](https://github.com/stellar/go/releases) y [Core releases](https://github.com/stellar/stellar-core/releases). Copie los binarios a PATH.

#### Verificar Instalaciones de Bare-Metal

Ejecuta `stellar-ceilon --help` desde un terminal. Si se muestra la ayuda para Horizon, su instalación fue exitosa.

Algunos shells (como [zsh](https://www.zsh.org/)) cachean búsquedas de PATH. Puede que necesites limpiar tu caché (usando `rehash` en zsh, por ejemplo) o reiniciar tu shell antes de intentar ejecutar el comando anterior.

#### Instalación de carta de yelmo

Si el despliegue puede hacerse en Kubernetes, hay un [Horizon Helm Chart](https://github.com/stellar/helm-charts/blob/main/charts/horizon) disponible. Instala la [herramienta CLI Helm](https://helm.sh/docs/intro/install/), si aún no estás en tu estación de trabajo, como mínimo la versión 3. A continuación, añada el repositorio Stellar a la lista de repos del cliente del timón y confirme que puede ver la lista de versiones disponibles para el repositorio:

<CodeExample>

```bash
helm repo add stellar https://helm.stellar.org/charts
helm repo update stellar
helm search repo stellar/horizonte --versions --devel
```

</CodeExample>

Espere para instalar el Chart Helado Horizon, se hará después de que [Configuring](./configuring.mdx) se haya completado y en [Running](./running.mdx).

Si Kubernetes no es una opción, los gráficos del timón pueden ser una buena referencia para mostrar cómo configurar y ejecutar el contenedor Horizon Docker. Sólo tiene que ejecutar el comando Helm con `template` para mostrar los manifiestos de Kubeneretes generados, que demuestran todas las configuraciones de contenedores necesarias:

<CodeExample>

```bash
git clone https://github.com/stellar/helm-charts; cd helm-charts
helm template -f charts/horizon/values.yaml charts/horizon/
```

</CodeExample>
