Horizon proporciona un mecanismo de streaming para recibir eventos en tiempo real. En lugar de enviar solicitudes repetidas al Horizon para actualizaciones por lotes, se establece una conexión entre un cliente y Horizon con actualizaciones a un flujo de respuesta de endpoint a medida que los nuevos ledgers se cierran y se producen actualizaciones.

Esto reduce las peticiones que no devuelven datos y permite actualizaciones instantáneas del cliente.

Todos los atributos para los extremos que permiten la transmisión son los mismos que las respuestas regulares. Un llamante puede iniciar la transmisión configurando 'Aceptar: text/event-stream' en la cabecera HTTP al hacer la solicitud. Estudia un ejemplo de uso de streaming en el [tutorial Seguir pagos recibidos](../../../../build/guides/basics/follow-received-payments).
