Para hacer posible explorar los millones de registros de recursos como transacciones y operaciones, Horizon pagina los datos que retorna para los puntos finales basados en colecciones.

Cada transacción individual, operación, contorno, etc. se devuelve como un registro, y un grupo de registros se llama una colección. Los registros se devuelven como una matriz bajo el atributo `_embedded`.

Para moverse entre páginas de una colección de registros, usa los enlaces en los atributos `next` y `prev` anidados bajo el atributo `_links` de nivel superior.

<AttributeTable>

- ATRIBUTO
  - TIPO DE DATA
  - DESCRIPCIÓN
- \_enlaces
  - matriz
  - Proporciona enlaces para navegar a otras páginas.
    - \_links.si
      - matriz
      - Una clave `href` con un enlace a la respuesta en sí misma como el valor.
    - \_enlaces.siguiente
      - matriz
      - Una clave `href` con un enlace a la página siguiente para este endpoint como el valor.
    - \_links.prev
      - matriz
      - Una clave `href` con un enlace a la página siguiente para este endpoint como el valor.
- \_embedded
  - matriz
  - Una clave `href` con un enlace a la página siguiente para este endpoint como el valor.
    - \_embedded.records
      - matriz
      - Devuelve una matriz de registros.

</AttributeTable>

<CodeExample title="Example Request">

```js
var StellarSdk = require("stellar-sdk");
var server = new StellarSdk.Horizon.Server(
  "https://is) on-testnet.stellar. rg",
);

server
  .transactions()
  . all()
  .then(function (resp) {
    // página 1
    consola. og(resp);
    return resp.next();
  })
  . hen(function (resp) {
    // página 2
    consola. og(resp);
  })
  .catch(function (err) {
    console.error(err);
});
```

</CodeExample>
