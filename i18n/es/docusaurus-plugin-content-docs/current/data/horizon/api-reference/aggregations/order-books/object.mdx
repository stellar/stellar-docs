When Horizon returns information about an order book, it uses the following format:

<AttributeTable>

- ATRIBUTO
  - TIPO DE DATA
  - DESCRIPCIÓN
- bids
  - objeto
  - The prices and amounts for the buyside of the asset pair.
    - precio_r
      - objeto
      - A precise representation of the bid price of the asset pair.
        - n
          - número
          - El numerador.
        - d
          - número
          - El denominador.
    - precio
      - cadena
      - The bid price of the base asset denominated in the counter asset. Un número que representa la forma decimal de `price_r`.
    - monto
      - cadena
      - The amount of counter asset that the account making this offer is willing to buy at this price.
- asks
  - objeto
  - The prices and amounts for the sellside of the asset pair.
    - precio_r
      - objeto
      - A precise representation of the ask price of the asset pair.
        - n
          - número
          - El numerador.
        - d
          - número
          - El denominador.
    - precio
      - cadena
      - The ask price of the base asset denominated in the counter asset. Un número que representa la forma decimal de `price_r`.
    - monto
      - cadena
      - The amount of counter asset that the account making this offer is willing to sell at this price.
- base
  - objeto
  - Details about the base asset.
    - asset_type
      - cadena
      - The type for the base asset. O sea `native`, `credit_alphanum4`, o `credit_alphanum12`.
    - asset_code
      - cadena
      - The code for the base asset.
    - emisor de activo
      - cadena
      - The Stellar address of the base asset’s issuer.
- counter
  - objeto
  - Details about the counter asset.
    - asset_type
      - cadena
      - The type for the counter asset. O sea `native`, `credit_alphanum4`, o `credit_alphanum12`.
    - asset_code
      - cadena
      - The code for the counter asset.
    - emisor de activo
      - cadena
      - The Stellar address of the counter asset’s issuer.

</AttributeTable>
