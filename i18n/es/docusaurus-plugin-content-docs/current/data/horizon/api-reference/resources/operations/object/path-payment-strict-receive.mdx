Envía un pago de una cuenta a otra en una ruta a través de las libretas de pedidos, comenzando como un activo y terminando como otro. Pagos de ruta que son `Strict Receive` designan la cantidad de pago en el activo recibido.

Vea los \[errores [`Path Payment Strict Receive`](../../../errors/result-codes/operation-specific/path-payment-strict-receive.mdx).

<AttributeTable>

- ATRIBUTO
  - TIPO DE DATA
  - DESCRIPCIÓN
- asset_type
  - cadena
  - El tipo de activo que se está recibiendo. O sea `native`, `credit_alphanum4`, o `credit_alphanum12`.
- asset_code
  - cadena
  - El código del activo que se está recibiendo. Aparece si el `asset_type` no es `native`.
- emisor de activo
  - cadena
  - La dirección estelar del emisor del activo que se recibe. Aparece si el `asset_type` no es `native`.
- de
  - cadena
  - Clave pública del remitente de pagos.
- a
  - cadena
  - La clave pública del destinatario del pago.
- monto
  - cadena
  - Cantidad recibida designada en el activo de destino.
- ruta
  - array de objetos
  - Los activos intermediarios por los que se salta esta ruta.
    - asset_code
      - cadena
      - El código de este activo intermediario.
    - emisor de activo
      - cadena
      - La dirección estelar del emisor del activo intermediario.
    - asset_type
      - cadena
      - El tipo del activo intermediario. O sea `native`, `credit_alphanum4`, o `credit_alphanum12`.
- importe_origen
  - cadena
  - Importe enviado designado en el activo fuente.
- origen_máx
  - cadena
  - La cantidad máxima a enviar designada en el activo fuente.
- tipo_fuente
  - cadena
  - El tipo del activo de origen. O sea `native`, `credit_alphanum4`, o `credit_alphanum12`.
- código fuente
  - cadena
  - El código del activo fuente.
- emisor de recursos
  - cadena
  - La dirección Stellar del emisor del activo fuente.

</AttributeTable>
