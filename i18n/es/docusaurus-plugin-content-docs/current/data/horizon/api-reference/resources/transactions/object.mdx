:::info

La transacción meta `result_meta_xdr` será eliminada de la API Horizon alojado de SDF (`horizon.stellar.org`) en Q3 2024. En lugar de usar Horizon para acceder a metadatos de transacción, los desarrolladores podrían acceder a él con el punto final [`getTransactions`](../../../rpc/api-reference/methods/getTransactions.mdx) del Soroban RPC.

:::

Cuando Horizon devuelve información sobre una transacción, utiliza el siguiente formato:

<AttributeTable>

- ATRIBUTO
  - TIPO DE DATA
  - DESCRIPCIÓN
- id
  - cadena
  - Un identificador único para esta transacción.
- ficha_paginando
  - número
  - Un valor del cursor para usar en [pagination](../../structure/pagination/README.mdx).
- exitoso
  - boolean
  - Indica si esta transacción fue exitosa o no.
- hash
  - cadena
  - Un hash con codificación hexadecimal SHA-256 de esta transacción con codificación [XDR](../../../../../learn/encyclopedia/data-format/xdr.mdx)-codificado.
- contador
  - número
  - El número de secuencia del contador en el que esta transacción fue incluida.
- creado_en
  - ISO8601 string
  - La fecha en la que se creó esta transacción.
- cuenta de origen
  - cadena
  - La cuenta que origina la transacción.
- secuencia_cuenta origen
  - cadena
  - El número de secuencia de la cuenta de origen que esta transacción consumió.
- fee_cobrado
  - número
  - La comisión (en [stroops](../../../../../learn/fundamentals/lumens.mdx)) pagada por la cuenta de origen para aplicar esta transacción al contador.
- tarifa máxima
  - número
  - La cuota máxima (en [stroops](../../../../../learn/fundamentals/lumens.mdx)) de que la cuenta de origen estaba dispuesta a pagar.
- contador_operación
  - número
  - El número de operaciones contenidas en esta transacción.
- envoltorio_xdr
  - cadena
  - Una cadena codificada en base64 de la estructura raw `TransactionEnvelope` XDR para esta transacción.
- resultado_xdr
  - cadena
  - Una cadena codificada en base64 de la estructura raw `TransactionResult` XDR para esta transacción.
- meta_xdr
  - cadena
  - **_[Para ser obsoleto en Q3]_** Una cadena codificada en base64 de la estructura raw `TransactionMeta` XDR para esta transacción
- fee_meta_xdr
  - cadena
  - Una cadena codificada en base64 de la estructura XDR `LedgerEntryChanges` cruda producida al cobrar comisiones por esta transacción.
- nota
  - cadena
  - El memo opcional adjunto a una transacción.
- memo_type
  - cadena
  - El tipo de memo. Los valores Potential incluyen `MEMO_TEXT`, `MEMO_ID`, `MEMO_HASH`, `MEMO_RETURN`.
- firmas
  - cadena
  - Un array de firmas utilizadas para firmar esta transacción.
- valida_después
  - RFC3339 fecha y hora cadena
  - La fecha después de la cual una transacción es válida. Este campo está obsoleto en lugar de `preconditions.time_bounds.min_time` y será eliminado en Horizon v3.
- valida_antes
  - RFC3339 fecha y hora cadena
  - La fecha antes de la cual una transacción es válida. Este campo está obsoleto en lugar de `preconditions.time_bounds.max_time` y será eliminado en Horizon v3.
- precondiciones
  - objeto
  - Un conjunto de condiciones previas de la transacción que afectan a su validez.
    - límite de tiempo
      - objeto
      - El intervalo de tiempo para el cual esta transacción es válida, con límites como marcas de tiempo UNIX sin firmar
        - min_vez
          - cadena
          - el límite inferior
        - max_hora
          - cadena
          - el límite superior
    - límite de bordes
      - objeto
      - El rango de datos para el cual esta transacción es válida, como enteros sin signo de 32 bits.
        - min_libro
          - número
          - el límite inferior
        - max_ledger
          - número
          - el límite superior
    - secuencia de cuenta
      - cadena
      - Conteniendo un entero positivo de 64 bits con signo que representa el número de secuencia de cuenta más bajo de origen para el cual la transacción es válida.
    - edad de la secuencia de la cuenta
      - número
      - La duración mínima del tiempo (en segundos como un entero sin signo de 64 bits) que debe haber pasado desde el número de secuencia de la cuenta de origen cambiado para que la transacción sea válida.
    - min_secuencia_cuenta_espacio_bordes
      - número
      - Un entero sin signo de 32 bits que representa el número mínimo de perdedores que deben haber cerrado desde el número de secuencia de la cuenta fuente cambiado para que la transacción sea válida.
    - extra_signers
      - array de cadenas
      - La lista de hasta dos firmas adicionales que deben tener las firmas correspondientes para que esta transacción sea válida.

</AttributeTable>
