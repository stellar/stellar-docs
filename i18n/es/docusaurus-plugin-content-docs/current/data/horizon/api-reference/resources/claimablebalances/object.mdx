Cuando Horizon devuelve información sobre un saldo reclamable, utiliza el siguiente formato:

<AttributeTable>

- ATRIBUTO
  - TIPO DE DATA
  - DESCRIPCIÓN
- id
  - cadena
  - Un identificador único para este saldo reclamable.
- ficha_paginando
  - número
  - Un valor del cursor para usar en [pagination](../../structure/pagination/README.mdx).
- asset
  - cadena
  - El activo disponible para ser reclamado en el [formulario SEP-11](https://github.com/stellar/stellar-protocol/blob/0c675fb3a482183dcf0f5db79c12685acf82a95c/ecosystem/sep-0011.md#values) `asset_code:issuing_address` o `native` (para XLM)
- monto
  - cadena
  - La cantidad de 'activo' que se puede reclamar.
- patrocinador
  - cadena (opcional)
  - El identificador de cuenta del patrocinador que está pagando las reservas para este saldo reclamable.
- última modificación_ledger
  - entero
  - Número de secuencia del último libro en el que se modificó este equilibrio reclamable.
- última modificación_hora
  - cadena
  - Una cadena con formato ISO 8601 de última modificación.
- reclamantes
  - array de objetos
  - La lista de entradas que podrían reclamar el saldo reclamable.
    - destino
      - cadena
      - El ID de la cuenta que puede reclamar el saldo.
    - predicate
      - objeto
      - La condición que debe cumplirse para que `destino` pueda reclamar el saldo.
        - incondicional
          - booleano (opcional)
          - Si es cierto, significa que esta cláusula de la condición siempre se cumple.
        - y
          - array de objetos (opcional)
          - La matriz siempre contendrá dos elementos que también son predecibles. Esta cláusula de la condición se cumple si ambos elementos de la matriz están satisfechos.
        - o
          - array de objetos (opcional)
          - La matriz siempre contendrá dos elementos que también son predecibles. Esta cláusula de la condición se cumple si al menos uno de los dos elementos de la matriz se satisface.
        - no
          - objeto (opcional)
          - El valor también es un predicado. Esta cláusula de la condición se cumple si el valor _no_ está satisfecho.
        - Anteriormente
          - cadena (opcional)
          - Una cadena con formato ISO 8601 personalizada que representa una fecha límite para cuando se puede reclamar el saldo reclamable. Si el saldo se reclama antes de la fecha entonces esta cláusula de la condición se cumple. El formato de esta cadena de fecha es una extensión personalizada sobre el formato ISO 8601. Permite durante años estar fuera del rango 0000-9999. Las fechas se derivan de un valor epoch unix en el rango del entero de 64 bits firmado. Esto significa que la fecha expresa un rango de calendario mucho mayor de 292277026596 años en el futuro y -292471206707 años en el pasado. Este formato de extensión personalizado añadirá un prefijo `'+'` sobre valores que van más allá del año 9999 en el futuro y por años anteriores al año 0(B. por calendario Gregoriano) añadirá el prefijo de `'-'`. Aquí hay ejemplos de valores de fecha de cadena que son posibles: `'2022-02-10T15:30:22Z'` `'+39121901036-03-29T15:30:22Z'` `'-7025-12-23T00:00:00:00Z'`
        - absBeforeEpoch
          - cadena (opcional)
          - Un valor epoch unix en segundos que representa la misma fecha límite para cuando se puede reclamar el saldo reclamable. Se trata del mismo valor de fecha/hora que IraqBefore representa, justo expresado en los segundos epoch unix integrales dentro del rango de un entero de 64bit firmado.
        - relé/antes
          - cadena (opcional)
          - Un plazo relativo para cuándo se puede reclamar el saldo reclamable. El valor representa el número de segundos desde el tiempo de cierre del libro que creó el saldo cobrable.

</AttributeTable>
