Cuando Horizon devuelve información sobre una cuenta, utiliza el siguiente formato:

<AttributeTable>

- ATRIBUTO
  - TIPO DE DATA
  - DESCRIPCIÓN
- id
  - cadena
  - Un identificador único para esta cuenta.
- cuenta_id
  - cadena
  - La clave pública de esta cuenta está codificada en una representación de cadenas base32.
- secuencia
  - número
  - Número de secuencia actual de esta cuenta. Para uso al enviar la próxima transacción de esta cuenta.
- secuencia_libro
  - número
  - El número sin signo del libro de 32 bits del número de secuencia [age](../../../../../../learn/glossary.mdx#sequence-number).
- secuencia_hora
  - cadena
  - La marca de tiempo UNIX de 64-bit no firmada del número de secuencia [age](../../../../../learn/glossary.mdx#sequence-number).
- subentry_count
  - número
  - El número de subentradas en esta cuenta.
- dominio_inicio
  - cadena
  - El dominio que aloja el archivo `stellar.toml` de esta cuenta.
- última modificación_ledger
  - número
  - El ID del último libro de valores que incluyó cambios en esta cuenta.
- num_patrocinio
  - número
  - El número de reservas patrocinadas por esta cuenta.
- num_patrocinado
  - número
  - El número de reservas patrocinadas para esta cuenta.
- patrocinador
  - cadena (opcional)
  - El ID de la cuenta del patrocinador que está pagando las reservas para esta cuenta.
- umbrales
  - objeto
  - Las operaciones tienen diferentes niveles de acceso. Este campo especifica umbrales para diferentes niveles de acceso, así como el peso de la clave maestra.
    - umbral bajo
      - número
      - El peso requerido para una transacción válida, incluyendo las operaciones de Trust y Bump Sequence.
    - umbral medido
      - número
      - El peso requerido para una transacción válida, incluyendo la Crear Cuenta, Pago, Path Payment, Gestionar oferta de compra, Administre la oferta de venta, cree una oferta de venta pasiva, cambie de confianza, inflación y gestione las operaciones de datos.
    - umbral alto
      - número
      - El peso requerido para una transacción válida, incluyendo las operaciones de Fusionado y Opciones de Conjunto de Cuentas.
- banderas
  - objeto
  - Las banderas denotan la habilitación/deshabilitación de ciertos privilegios de emisor de activos.
    - auten_inmutable
      - boolean
      - Si se establece en `true`, ninguna de las siguientes banderas puede ser cambiada.
    - auth_required
      - boolean
      - Si se establece en `true`, cualquiera que quiera tener un activo emitido por esta cuenta primero debe ser aprobado por esta cuenta.
    - auth_revocable
      - boolean
      - Si se establece en `true`, esta cuenta puede congelar el saldo de un titular de un activo emitido por esta cuenta.
    - auth_clawback_habilitado
      - boolean
      - Si se establece en `true`, las líneas de confianza creadas para los activos emitidos por esta cuenta tienen [clawbacks](../../../../learn/glossary.mdx#clawback) activadas.
- saldos
  - objeto
  - Los activos que posee esta cuenta.
    - saldo
      - cadena
      - El número de unidades de un activo en posesión de esta cuenta.
    - comprar_pasivos
      - cadena (opcional)
      - La suma de todas las ofertas de compra propiedad de esta cuenta para este activo.
    - responsabilidad_selling
      - cadena (opcional)
      - La suma de todas las ofertas vendidas propiedad de esta cuenta para este activo.
    - límite
      - número (opcional)
      - La cantidad máxima de este activo que esta cuenta está dispuesta a aceptar. Especificado al abrir una línea de confianza.
    - asset_type
      - cadena
      - O sea `native`, `credit_alphanum4`, `credit_alphanum12`, o `liquidity_pool_shares`.
    - asset_code
      - cadena (opcional)
      - El código de este activo.
    - emisor de activo
      - cadena (opcional)
      - La dirección estelar del emisor de este activo.
    - patrocinador
      - cadena (opcional)
      - El ID de la cuenta del patrocinador que está pagando las reservas para este fideicomiso.
- firmantes
  - array de objetos
  - Las claves públicas y los pesos asociados que se pueden utilizar para autorizar transacciones para esta cuenta. Utilizado para multi-firma.
    - llave_pública
      - cadena
      - **REMOVADO en 0.17.0: USE `key` INSTEAD.**
    - peso
      - número
      - El peso numérico de un firmante. Utilizado para determinar si una transacción cumple con los requisitos de `umbral`.
    - patrocinador
      - cadena (opcional)
      - El ID de la cuenta del patrocinador que está pagando las reservas para este firmante.
    - llave
      - cadena
      - Un hash de caracteres dependiente del tipo de firmante.
    - tipo
      - cadena
      - El tipo de hash para este firmante.
        - ed25519_public_key
          - saltar
          - Una clave pública estelar normal.
        - sha256_hash
          - saltar
          - El hash SHA256 de algún `x` arbitrario. Añadir una firma de este tipo permite a cualquiera que conozca x firmar una transacción desde esta cuenta. _Nota: Una vez que esta transacción sea transmitida, x será conocida públicamente._
        - preauth_tx
          - saltar
          - El hash de una transacción preautorizada. Este firmante se elimina automáticamente de la cuenta cuando una transacción coincidente se aplica correctamente.
- datos
  - objeto
  - Una matriz de campos de datos de la cuenta.

</AttributeTable>
