Cuando Horizon devuelve información sobre un activo, utiliza el siguiente formato:

<AttributeTable>

- ATRIBUTO
  - TIPO DE DATA
  - DESCRIPCIÓN
- asset_type
  - cadena
  - Tipo de activo. O `credit_alphanum4` o `credit_alphanum12`.
- asset_code
  - cadena
  - Código de este recurso
- emisor de activo
  - cadena
  - La dirección estelar del emisor de este activo.
- cuentas
  - objeto
  - El número de cuentas agrupadas por cada estado de bandera de confianza.
- saldos num_reclamables
  - número
  - El número actual de claimable_balances para este activo.
- num_contratos
  - número
  - El número actual de contratos Soroban que poseen este activo.
- batería num_liquidez
  - número
  - El número actual de piscinas de liquidez que posean este activo.
- saldos
  - objeto
  - El número de unidades emitidas para este activo agrupado por cada estado de bandera de trustline.
- monto cobrable_balances_cobro
  - cadena
  - Número de unidades para este activo en poder de todos los saldos cobrables.
- contracts_amount
  - cadena
  - Número de unidades para este activo en posesión de todos los contratos de Soroban.
- monto de la liquidez_pools_amount
  - cadena
  - El número de unidades para este activo en posesión de todas las reservas de liquidez.
- monto
  - cadena
  - Número de unidades autorizadas emitidas para este activo. Esto será obsoleto en Horizon v3.
- num_cuentas
  - número
  - El número de cuentas que han emitido una línea de confianza a este activo. Si la bandera `auth_required` para el emisor de este activo se establece en `true`, este número sólo incluye las cuentas que han establecido una línea de confianza para el activo y han sido autorizadas a mantener el activo. Esto será obsoleto en Horizon v3.
- banderas
  - objeto
  - Las banderas denotan la habilitación/deshabilitación de ciertos privilegios de emisor de activos.
    - auten_inmutable
      - boolean
      - Si se establece en `true`, ninguna de las siguientes banderas puede ser cambiada.
    - auth_required
      - boolean
      - Si se establece en `true`, cualquiera que quiera tener un activo emitido por esta cuenta primero debe ser aprobado por esta cuenta.
    - auth_revocable
      - boolean
      - Si se establece en `true`, esta cuenta puede congelar el saldo de un titular de un activo emitido por esta cuenta.
- ficha_paginando
  - número
  - Un valor del cursor para usar en [pagination](../../structure/pagination/README.mdx).

</AttributeTable>
