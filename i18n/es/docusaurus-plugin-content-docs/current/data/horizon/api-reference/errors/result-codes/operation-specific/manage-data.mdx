Estos son códigos de resultado que comunican el éxito (200) o la falla (400) específica de la operación 'Administrar datos'.

Aprende más sobre la [operación 'Administrar datos'](../../../../../../learn/fundamentals/transactions/list-of-operations.mdx#manage-data).

<AttributeTable>

- OpSuccess
  - Ficha de datos
  - La operación de gestión de datos se ha ejecutado correctamente.
- aún no soportado\_
  - MANAGE_DATA_NOT_TITLE
  - La red aún no se ha movido a este cambio de protocolo. Este fallo significa que la red no soporta esta característica todavía.
- no se encontró el nombre de datos
  - Nombre_NOMBRE_DESCRIPTION
  - Intentando eliminar una entrada de datos que no está ahí. Esto sucederá si el nombre está configurado (y el valor no lo está), pero la cuenta no tiene una entrada de datos con ese nombre.
- op_low_reserve
  - MANAGE_BTN
  - Esta cuenta no tiene suficiente XLM para satisfacer el incremento mínimo de reserva XLM causado por la adición de una subentrada y aún así satisfacer sus pasivos de venta XLM. Por cada nuevo DataEntry añadido a una cuenta, la reserva mínima de XLM que la cuenta debe mantener aumenta.
- op_data_invalid_name
  - MANAGE_DATA_INVALID_NAME
  - Name not a valid string.

</AttributeTable>
