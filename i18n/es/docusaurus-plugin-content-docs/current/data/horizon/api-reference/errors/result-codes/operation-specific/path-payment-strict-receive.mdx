Estos son códigos de resultado que comunican el éxito (200) o el fallo (400) específico de la operación `Path Payment Strict Receive`.

Aprenda más acerca de la [operación de recepción estricta de pago de la ruta\`](../../../../../../../learn/fundamentals/transactions/list-of-operations.mdx#path-payment-strict-receive).

<AttributeTable>

- RESULTAR CÓDIGO
  - STELLAR PROTOCOL CODE
  - DESCRIPCIÓN
- OpSuccess
  - PATH_PAYMENT_STRICT
  - El pago se completó correctamente.
- OpMalformado
  - PATH_PAYMENT_STRICT_RECEIVE
  - La entrada para este pago de ruta no es válida.
- Optimizado
  - PATH_PAYMENT_STRICT
  - La cuenta de origen (remitente) no tiene suficientes lumens para enviar la cantidad de pago manteniendo su propia reserva mínima.
- Oportunidad de confianza
  - PATH_PAYMENT_STRICT_RECEIVE
  - A la cuenta de origen le falta la línea de confianza apropiada.
- Opcionalmente no autorizado
  - PATH_PAYMENT_STRICT
  - La cuenta de origen no está autorizada a enviar este activo.
- OpNoDestinación
  - PATH_PAYMENT_STRICT
  - La cuenta de destino no existe.
- OnoTrust
  - PATH_PAYMENT
  - La cuenta de destino no tiene una línea de confianza para el activo a enviar.
- Opno autorizada
  - PATH_PAYMENT_STRICT
  - La cuenta de destino no está autorizada a mantener este activo.
- OpLineFull
  - PATH_PAYMENT
  - La cuenta de destino (receptor) no tiene límites suficientes para recibir la cantidad y aún así satisfacer sus pasivos de compra.
- OpNoIssuer
  - PATH_PAYMENT
  - Falta el emisor de uno de los activos.
- Ofertas OpTooFewers
  - PATH_PAYMENT_STRICT_RECEIVE
  - No hay ruta de ofertas que conecten el activo de envío y el activo de destino. Estelar sólo considera caminos de longitud 5 o más cortos.
- OpCrossSelf
  - PATH_PAYMENT_STRICT
  - Este pago de ruta cruzaría una de sus propias ofertas.
- OpOverSourceMax
  - PATH_PAYMENT_STRICT
  - Las rutas que podrían enviar la cantidad de destino del activo de destino superarían el máximo de envío.

</AttributeTable>
