Estos son códigos de resultado que comunican el éxito (200) o la falla (400) específica para la operación `Pago`, que a menudo falla porque la cuenta receptora no confía en el emisor del activo que se envía.

Aprende más sobre la [operación de pago](../../../../../../learn/fundamentals/transactions/list-of-operations.mdx#payment).

<AttributeTable>

- RESULTAR CÓDIGO
  - STELLAR PROTOCOL CODE
  - DESCRIPCIÓN
- OpSuccess
  - PAYMENT_TITLE
  - El pago se completó correctamente.
- OpMalformado
  - PAYMENT_MALFORMED
  - La entrada al pago no es válida.
- Optimizado
  - PAYMENT_UNDERFUNDADO
  - La cuenta de origen (remitente) no tiene suficientes lumens para enviar la cantidad de pago manteniendo su propia reserva mínima.
- Oportunidad de confianza
  - PAYMENT_SRC
  - La cuenta de origen no tiene una línea de confianza para el activo que está intentando enviar.
- Opcionalmente no autorizado
  - PAYMENT_SRC_NOT_AUT
  - La cuenta de origen no está autorizada a enviar este activo.
- OpNoDestinación
  - NO_DESTINATION
  - La cuenta de destino no existe.
- OnoTrust
  - PAYMENT_NO_TITLE
  - La cuenta de destino no tiene una línea de confianza para el activo a enviar.
- Opno autorizada
  - PAYMENT_NOT_AUTORIZED
  - La cuenta de destino no está autorizada a mantener este activo.
- OpLineFull
  - PAYMENT_TITLE
  - La cuenta de destino (receptor) no tiene límites suficientes para recibir la cantidad y aún así satisfacer sus pasivos de compra.
- OpNoIssuer
  - PAYMENT_NO_ISSUER
  - El emisor del activo no existe.

</AttributeTable>
