Estos son códigos de resultado que comunican el éxito (200) o la falla (400) específica de la operación `Permitir Trust`.

Aprende más sobre la [operación `Permitir Trust`](../../../../../../learn/fundamentals/transactions/list-of-operations.mdx#allow-trust).

<AttributeTable>

- OpSuccess
  - Torrent
  - La operación de confianza fue exitosa.
- OpMalformado
  - ALLOW_MALFORMED
  - El activo especificado en el tipo no es válido. Además, este error ocurre cuando se especifica el activo nativo.
- op_no_trustline
  - No hay ninguna línea
  - El trustor no tiene una línea de confianza con el emisor que realiza esta operación.
- op_no_obligatorio
  - ALLOW_PLAYLIST_POPUP_TITLE
  - La cuenta de origen (emisor que realiza esta operación) no requiere confianza. En otras palabras, no tiene que tener la bandera AUTH_REQUIRED_FLAG.
- revocar
  - ALLOW_PLAYLIST_TITLE
  - La cuenta de origen está tratando de revocar la línea de confianza del trustor, pero no puede hacerlo.
- no permitido
  - Todos los objetos no están disponibles
  - La cuenta de origen intentó permitir una línea de confianza para sí misma, la cual no está permitida porque una cuenta no puede crear una línea de confianza consigo misma.
- op_low_reserve
  - TODOS_PLAYLIST_LABEL
  - Los saldos reclamables no se pueden crear al revocar debido a reservas bajas

</AttributeTable>
