# Introducción al RPC

:::info

Stellar-RPC fue renombrado de Soroban-RPC en Noviembre de 2024. Más contexto sobre esta decisión se puede encontrar en nuestro [blog de desarrolladores].

:::

Stellar RPC es una herramienta ligera que proporciona acceso en tiempo real a los datos de la red Stellar. Al igual que los nodos RPC en otros ecosistemas de Blockchain, permite a los desarrolladores consultar la red de manera eficiente. Ya sea que estés construyendo una billetera no custodiada, emitiendo activos o monitoreando la actividad de la red, Stellar RPC está diseñado para proporcionar una infraestructura confiable y estable que cualquiera puede operar.

Para cualquier nuevo constructor que venga a Stellar, Stellar RPC debe ser tu punto de partida: está diseñada para alinearse con las crecientes necesidades del ecosistema. El RPC se puede acceder a través de cURL o uno de los [Stellar SDKs](../../../tools/sdks/README.mdx).

## ¿Por qué ejecutar RPC?

Ejecutar RPC dentro de tu propia infraestructura proporciona una serie de beneficios. Puedes:

- Tener control operativo total sin depender de ningún proveedor externo para los datos de red y la presentación de transacciones. ¡La única forma de aprovechar el verdadero poder de un blockchain descentralizado!
- Evita la sobrecarga adicional de interactuar directamente con [Stellar Core], cuyo enfoque principal es el rendimiento y, por lo tanto, proporciona una API muy limitada
- Evita el sobrecoste añadido de almacenar muchos más datos de los que tu aplicación realmente necesita, como sería el caso al ejecutar [Horizon]

Lo que Stellar RPC no es:

- Un indexador para datos históricos. El RPC mantiene como máximo 7 días de datos históricos.
- Un servicio backend principal para tu aplicación. Usa RPC como tu puerta de entrada a la Blockchain, pero ingiere e indexa solo los datos que te interesan.
- Un reemplazo directo de Horizon. Horizon proporciona varias características de indexación que no son comúnmente admitidas por los nodos RPC. Creemos que estas oportunidades comerciales deberían ser transferidas a aplicaciones de terceros (indexadores, proveedores de análisis, etc.) y alejadas de la SDF.
