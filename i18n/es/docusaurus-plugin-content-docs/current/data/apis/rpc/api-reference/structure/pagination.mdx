title: Paginación
---

La paginación en stellar-rpc es similar a la paginación en Horizon.

Para los métodos que lo admiten, los argumentos de paginación se pasan como un último argumento objeto con dos valores:

- `cursor`: cadena - (opcional) Una cadena opaca que actúa como un token de paginación. Cada respuesta incluirá un campo `cursor` que se puede incluir en una solicitud subsiguiente para obtener la siguiente página de resultados.
- `limit`: número - (opcional) El número máximo de registros devueltos. El límite para [getEvents](../methods/getEvents.mdx) puede oscilar entre 1 y 10000, un límite superior codificado en Stellar-RPC por razones de rendimiento. Si este argumento no se designa, por defecto es 100.

Por ejemplo, al llamar a un método con el parámetro de paginación establecido:

```json
{
  "jsonrpc": "2.0",
  "id": "1",
  "method": "exampleMethod",
  "params": {
    "some": "argument",
    "pagination": {
      "cursor": "1234-1",
      "limit": 100
    }
  }
}
```
