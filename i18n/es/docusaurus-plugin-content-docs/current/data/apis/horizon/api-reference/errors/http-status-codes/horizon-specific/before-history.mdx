Un servidor Horizon puede configurarse para conservar solo una parte del historial de la red estelar almacenado en su base de datos.

El error `before_history` devuelve un [código de error `410`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Status/410) y ocurre cuando un cliente solicita una pieza de información (como una página de transacciones o una sola operación) que el servidor puede identificar positivamente como fuera del rango de historial grabado.

<ExampleResponse title="Respuesta de ejemplo para un código de estado 'Antes de la Historia'">

```json
{
  "type": "https://stellar.org/horizon-errors/before_history",
  "title": "Data Requested Is Before Recorded History",
  "status": 410,
  "detail": "This horizon instance is configured to only track a portion of the stellar network's latest history. This request is asking for results prior to the recorded history known to this horizon instance."
}
```

</ExampleResponse>
