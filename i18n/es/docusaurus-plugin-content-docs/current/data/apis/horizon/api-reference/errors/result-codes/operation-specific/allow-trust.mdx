Estos son códigos de resultado que comunican éxito (200) o fracaso (400) específicos para la operación `Permitir confianza`.

Obtén más información sobre la operación [`Permitir confianza`](../../../../../../../learn/fundamentals/transactions/list-of-operations.mdx#allow-trust).

<AttributeTable>

- OpÉxito
  - ALLOW_TRUST_SUCCESS
  - La operación de confianza fue exitosa.
- OpMalformado
  - ALLOW_TRUST_MALFORMED
  - El activo especificado en el tipo es inválido. Además, este error ocurre cuando se especifica el activo nativo.
- op_no_trustline
  - ALLOW_TRUST_NO_TRUST_LINE
  - El fideicomisario no tiene una línea de confianza con el emisor que realiza esta operación.
- op_not_required
  - ALLOW_TRUST_TRUST_NOT_REQUIRED
  - La cuenta fuente (el emisor realizando esta operación) no requiere confianza. En otras palabras, no necesita tener el flag AUTH_REQUIRED_FLAG establecido.
- op_cant_revoke
  - ALLOW_TRUST_CANT_REVOKE
  - La cuenta fuente está intentando revocar la línea de confianza del fideicomisario, pero no puede hacerlo.
- op_self_not_allowed
  - ALLOW_TRUST_SELF_NOT_ALLOWED
  - La cuenta fuente intentó permitir una línea de confianza para sí misma, lo cual no está permitido porque una cuenta no puede crear una línea de confianza con ella misma.
- op_low_reserve
  - ALLOW_TRUST_LOW_RESERVE
  - Los saldos reclamables no pueden ser creados al revocar debido a reservas bajas

</AttributeTable>
