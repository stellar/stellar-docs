Cuando Horizon devuelve información sobre una oferta, utiliza el siguiente formato:

<AttributeTable>

- ATRIBUTO
  - TIPO DE DATO
  - DESCRIPCIÓN
- id
  - cadena
  - Un identificador único para esta oferta.
- token_de_paginación
  - número
  - Un valor de cursor para usar en [paginación](../../structure/pagination/README.mdx).
- vendedor
  - cadena
  - El ID de la cuenta que hace esta oferta.
- vendiendo
  - código de activo
  - El activo que esta oferta quiere vender.
- comprando
  - código de activo
  - El activo que esta oferta quiere comprar.
- cantidad
  - cadena
  - La cantidad de `vendiendo` que la cuenta que hace esta oferta está dispuesta a vender.
- precio_r
  - objeto
  - Una representación precisa del precio de compra y venta de los activos ofrecidos.
    - n
      - número
      - El numerador.
    - d
      - número
      - El denominador.
- precio
  - cadena
  - Cuántas unidades de `comprando` se necesitan para obtener 1 unidad de `vendiendo`. Un número que representa la forma decimal de `precio_r`.
- último_libro_de_ledger_modificado
  - entero
  - El número de secuencia del último libro de ledger en el que se modificó esta oferta.
- último_tiempo_de_modificación
  - cadena
  - Una cadena formateada en ISO 8601 de la última hora de modificación.
- patrocinador
  - cadena (opcional)
  - El ID de la cuenta del patrocinador que está pagando las reservas para esta oferta.

</AttributeTable>

El [último libro de ledger](../../structure/consistency.mdx) que se conoce en Horizon se incluye como un encabezado HTTP en la respuesta.
