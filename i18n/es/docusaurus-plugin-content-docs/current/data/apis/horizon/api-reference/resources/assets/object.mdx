Cuando Horizon devuelve información sobre un activo, utiliza el siguiente formato:

<AttributeTable>

- ATRIBUTO
  - TIPO DE DATO
  - DESCRIPCIÓN
- tipo_de_activo
  - cadena
  - El tipo de este activo. Ya sea `credit_alphanum4` o `credit_alphanum12`.
- código_de_activo
  - cadena
  - El código de este activo
- emisor_de_activo
  - cadena
  - La dirección Stellar del emisor de este activo.
- cuentas
  - objeto
  - El número de cuentas agrupadas por cada estado de la bandera de la línea de confianza.
- num_saldos_reclamables
  - número
  - El número actual de saldos_reclamables para este activo.
- num_contratos
  - número
  - El número actual de contratos de Soroban que poseen este activo.
- num_fondos_de_liquidez
  - número
  - El número actual de fondos de liquidez que poseen este activo.
- saldos
  - objeto
  - El número de unidades emitidas para este activo agrupadas por cada estado de la bandera de la línea de confianza.
- monto_de_saldos_reclamables
  - cadena
  - El número de unidades para este activo que posee todos los saldos reclamables.
- monto_de_contratos
  - cadena
  - El número de unidades para este activo que poseen todos los contratos de Soroban.
- monto_de_fondos_de_liquidez
  - cadena
  - El número de unidades para este activo que poseen todos los fondos de liquidez.
- banderas
  - objeto
  - Las banderas indican el habilitar/deshabilitar ciertos privilegios del emisor de activos.
    - auth_inmutable
      - booleano
      - Si se establece en `true`, ninguna de las siguientes banderas puede ser cambiada.
    - auth_requerido
      - booleano
      - Si se establece en `true`, cualquier persona que desee poseer un activo emitido por esta cuenta debe ser aprobada primero por esta cuenta.
    - auth_revocable
      - booleano
      - Si se establece en `true`, esta cuenta puede congelar el saldo de un poseedor de un activo emitido por esta cuenta.
- token_de_paginación
  - número
  - Un valor de cursor para usar en [paginación](../../structure/pagination/README.mdx).

</AttributeTable>
