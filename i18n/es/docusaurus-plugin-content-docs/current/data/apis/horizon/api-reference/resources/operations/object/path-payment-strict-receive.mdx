Envía un pago de una cuenta a otra a través de un camino en los libros de órdenes, comenzando como un activo y terminando como otro. Los pagos por ruta que son `Recibir Estricto` designan la cantidad de pago en el activo recibido.

Consulta los [errores de `Pago por Ruta Estricto Recibir`](../../../errors/result-codes/operation-specific/path-payment-strict-receive.mdx).

<AttributeTable>

- ATRIBUTO
  - TIPO DE DATO
  - DESCRIPCIÓN
- tipo_de_activo
  - cadena
  - El tipo de activo que se recibe. Puede ser `nativo`, `credito_alphanum4`, o `credito_alphanum12`.
- codigo_de_activo
  - cadena
  - El código del activo que se recibe. Aparece si el `tipo_de_activo` no es `nativo`.
- emisor_de_activo
  - cadena
  - La dirección Stellar del emisor del activo que se recibe. Aparece si el `tipo_de_activo` no es `nativo`.
- de
  - cadena
  - La clave pública del remitente del pago.
- a
  - cadena
  - La clave pública del destinatario del pago.
- cantidad
  - cadena
  - Cantidad recibida designada en el activo de destino.
- ruta
  - array de objetos
  - Los activos intermedios por los que pasa este camino.
    - codigo_de_activo
      - cadena
      - El código para este activo intermedio.
    - emisor_de_activo
      - cadena
      - La dirección Stellar del emisor del activo intermedio.
    - tipo_de_activo
      - cadena
      - El tipo del activo intermedio. Puede ser `nativo`, `credito_alphanum4`, o `credito_alphanum12`.
- cantidad_fuente
  - cadena
  - Cantidad enviada designada en el activo fuente.
- maximo_fuente
  - cadena
  - La cantidad máxima a enviar designada en el activo fuente.
- tipo_de_activo_fuente
  - cadena
  - El tipo del activo fuente. Puede ser `nativo`, `credito_alphanum4`, o `credito_alphanum12`.
- codigo_de_activo_fuente
  - cadena
  - El código del activo fuente.
- emisor_de_activo_fuente
  - cadena
  - La dirección Stellar del emisor del activo fuente.

</AttributeTable>
