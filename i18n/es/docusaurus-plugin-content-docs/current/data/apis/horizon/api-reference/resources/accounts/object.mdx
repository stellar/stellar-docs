Cuando Horizon devuelve información sobre una cuenta, utiliza el siguiente formato:

<AttributeTable>

- ATRIBUTO
  - TIPO DE DATO
  - DESCRIPCIÓN
- id
  - cadena
  - Un identificador único para esta cuenta.
- account_id
  - cadena
  - La clave pública de esta cuenta codificada en una representación de cadena base32.
- secuencia
  - número
  - El número de secuencia actual de esta cuenta. Para usar al enviar la próxima transacción de esta cuenta.
- secuencia_ledger
  - número
  - El número del ledger de 32 bits sin firmar del [edad](../../../../../../learn/glossary.mdx#sequence-number) del número de secuencia.
- secuencia_tiempo
  - cadena
  - La marca de tiempo UNIX de 64 bits sin firmar de la [edad](../../../../../../learn/glossary.mdx#sequence-number) del número de secuencia.
- count_subentradas
  - número
  - El número de subentradas en esta cuenta.
- dominio_principal
  - cadena
  - El dominio que aloja el archivo `stellar.toml` de esta cuenta.
- ledger_última_modificación
  - número
  - La ID del último ledger que incluyó cambios en esta cuenta.
- num_sponsoring
  - número
  - El número de reservas patrocinadas por esta cuenta.
- num_sponsored
  - número
  - El número de reservas patrocinadas para esta cuenta.
- patrocinador
  - cadena (opcional)
  - La ID de cuenta del patrocinador que está pagando las reservas por esta cuenta.
- thresholds
  - objeto
  - Las operaciones tienen diferentes niveles de acceso. Este campo especifica los umbrales para diferentes niveles de acceso, así como el peso de la clave maestra.
    - umbral_bajo
      - número
      - El peso requerido para una transacción válida que incluye las operaciones Permitir Confianza y Aumentar Secuencia.
    - umbral_medio
      - número
      - El peso requerido para una transacción válida que incluye las operaciones Crear Cuenta, Pago, Pago por Ruta, Gestionar Oferta de Compra, Gestionar Oferta de Venta, Crear Oferta de Venta Pasiva, Cambiar Confianza, Inflación y Gestionar Datos.
    - umbral_alto
      - número
      - El peso requerido para una transacción válida que incluye las operaciones Fusionar Cuenta y Establecer Opciones.
- flags
  - objeto
  - Los flags indican la habilitación/deshabilitación de ciertos privilegios del emisor de activos.
    - auth_inmutable
      - booleano
      - Si se establece en `true`, ninguno de los siguientes flags puede ser cambiado.
    - auth_requerido
      - booleano
      - Si se establece en `true`, cualquiera que quiera poseer un activo emitido por esta cuenta debe ser aprobado primero por esta cuenta.
    - auth_revocable
      - booleano
      - Si se establece en `true`, esta cuenta puede congelar el saldo de un propietario de un activo emitido por esta cuenta.
    - auth_clawback_enabled
      - booleano
      - Si se establece en `true`, las líneas de confianza creadas para activos emitidos por esta cuenta tienen habilitados los [clawbacks](../../../../../../learn/glossary.mdx#clawback).
- saldos
  - objeto
  - Los activos que posee esta cuenta.
    - saldo
      - cadena
      - El número de unidades de un activo que posee esta cuenta.
    - pasivos_de_compra
      - cadena (opcional)
      - La suma de todas las ofertas de compra que posee esta cuenta para este activo.
    - pasivos_de_venta
      - cadena (opcional)
      - La suma de todas las ofertas de venta que posee esta cuenta para este activo.
    - límite
      - número (opcional)
      - La cantidad máxima de este activo que esta cuenta está dispuesta a aceptar. Especificado al abrir una línea de confianza.
    - tipo_de_activo
      - cadena
      - Ya sea `nativo`, `crédito_alphanum4`, `crédito_alphanum12`, o `liquidity_pool_shares`.
    - codigo_de_activo
      - cadena (opcional)
      - El código para este activo.
    - emisor_de_activo
      - cadena (opcional)
      - La dirección Stellar del emisor de este activo.
    - patrocinador
      - cadena (opcional)
      - La ID de cuenta del patrocinador que está pagando las reservas por esta línea de confianza.
- firmantes
  - array de objetos
  - Las claves públicas y los pesos asociados que se pueden usar para autorizar transacciones de esta cuenta. Usado para multi-firma.
    - clave_publica
      - cadena
      - **ELIMINADO en 0.17.0: USE `key` EN SU LUGAR.**
    - peso
      - número
      - El peso numérico de un firmante. Usado para determinar si una transacción cumple con los requisitos de `umbral`.
    - patrocinador
      - cadena (opcional)
      - La ID de cuenta del patrocinador que está pagando las reservas por este firmante.
    - clave
      - cadena
      - Un hash de caracteres dependiente del tipo de firmante.
    - tipo
      - cadena
      - El tipo de hash para este firmante.
        - clave_publica_ed25519
          - saltar
          - Una clave pública Stellar normal.
        - hash_sha256
          - saltar
          - El hash SHA256 de algún `x` arbitrario. Agregar una firma de este tipo permite a cualquiera que conozca x firmar una transacción de esta cuenta. _Nota: Una vez que esta transacción sea transmitida, x será conocido públicamente._
        - transacción_auth_preautorizada
          - saltar
          - El hash de una transacción preautorizada. Este firmante se elimina automáticamente de la cuenta cuando una transacción coincidente se aplica correctamente.
- datos
  - objeto
  - Un arreglo de campos de datos de cuenta.

</AttributeTable>
