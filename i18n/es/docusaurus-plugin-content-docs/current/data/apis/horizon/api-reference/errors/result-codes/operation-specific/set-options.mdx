Estos son códigos de resultado que comunican éxito (200) o fallo (400) específicos para la operación `Establecer Opciones`.

Aprende más sobre la [operación `Establecer Opciones`](../../../../../../../learn/fundamentals/transactions/list-of-operations.mdx#set-options).

<AttributeTable>

- OpÉxito
  - Éxito en la configuración de opciones
  - Opciones establecidas con éxito.
- OpBajoReserva
  - SET_OPTIONS_LOW_RESERVE
  - Esta cuenta no tiene suficiente XLM para satisfacer el incremento de reserva mínima de XLM causado por agregar un subentrada y seguir satisfaciendo sus responsabilidades de venta de XLM. Por cada nuevo firmante agregado a una cuenta, la reserva mínima de XLM que esa cuenta debe mantener aumenta.
- op_demasiados_firmantes
  - SET_OPTIONS_TOO_MANY_SIGNERS
  - 20 es el número máximo de firmantes que una cuenta puede tener, y agregar otro firmante excedería eso.
- op_flags_malos
  - SET_OPTIONS_BAD_FLAGS
  - Las flags establecidas y/o eliminadas son inválidas por sí solas o en combinación.
- op_inflación_inválida
  - SET_OPTIONS_INVALID_INFLATION
  - La cuenta de destino establecida en el campo de inflación no existe.
- op_no_se_puede_cambiar
  - SET_OPTIONS_CANT_CHANGE
  - Esta cuenta ya no puede cambiar la opción que desea cambiar.
- op_bandera_desconocida
  - SET_OPTIONS_UNKNOWN_FLAG
  - La cuenta está intentando establecer una bandera que es desconocida.
- op_umbral_fuera_de_rango
  - SET_OPTIONS_THRESHOLD_OUT_OF_RANGE
  - El valor para el peso de una clave o umbral es inválido.
- op_firmante_malo
  - SET_OPTIONS_BAD_SIGNER
  - Cualquier firmante adicional agregado a la cuenta no puede ser la clave maestra.
- op_dominio_hogar_inválido
  - SET_OPTIONS_INVALID_HOME_DOMAIN
  - El dominio hogar está mal formado.

</AttributeTable>
