Cuando Horizon devuelve información sobre una ruta, utiliza el siguiente formato:

<AttributeTable>

- ATRIBUTO
  - TIPO DE DATO
  - DESCRIPCIÓN
- tipo_de_activo_origen
  - cadena
  - El tipo para el activo fuente. Puede ser `native`, `credit_alphanum4`, o `credit_alphanum12`.
- código_de_activo_origen
  - cadena
  - El código para el activo fuente.
- emisor_de_activo_origen
  - cadena
  - La dirección Stellar del emisor del activo fuente.
- monto_origen
  - cadena
  - Un costo estimado para realizar un pago de destination_amount en esta ruta. Adecuado para usar en el campo `sendMax` de una operación de pago por ruta.
- tipo_de_activo_destino
  - cadena
  - El tipo para el activo de destino. Puede ser `native`, `credit_alphanum4`, o `credit_alphanum12`.
- código_de_activo_destino
  - cadena
  - El código para el activo de destino.
- emisor_de_activo_destino
  - cadena
  - La dirección Stellar del emisor del activo de destino.
- monto_destino
  - cadena
  - La cantidad de destino especificada en la búsqueda que encontró esta ruta.
- ruta
  - array de objetos
  - Los activos intermedios por los que pasa esta ruta.
    - código_de_activo
      - cadena
      - El código para este activo intermedio.
    - emisor_de_activo
      - cadena
      - La dirección Stellar del emisor del activo intermedio.
    - tipo_de_activo
      - cadena
      - El tipo para el activo intermedio. Puede ser `native`, `credit_alphanum4`, o `credit_alphanum12`.

</AttributeTable>

El [último libro mayor](../../structure/consistency.mdx) conocido por Horizon se incluye como un encabezado HTTP en la respuesta.
