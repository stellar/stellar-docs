Cuando Horizon devuelve información sobre un saldo reclamable, utiliza el siguiente formato:

<AttributeTable>

- ATRIBUTO
  - TIPO DE DATOS
  - DESCRIPCIÓN
- id
  - cadena
  - Un identificador único para este saldo reclamable.
- token_de_paginación
  - número
  - Un valor de cursor para usar en [paginación](../../structure/pagination/README.mdx).
- activo
  - cadena
  - El activo disponible para ser reclamado en el [formulario SEP-11](https://github.com/stellar/stellar-protocol/blob/0c675fb3a482183dcf0f5db79c12685acf82a95c/ecosystem/sep-0011.md#values) `asset_code:issuing_address` o `nativo` (para XLM)
- cantidad
  - cadena
  - La cantidad de `activo` que puede ser reclamada.
- patrocinador
  - cadena (opcional)
  - El id de cuenta del patrocinador que está pagando las reservas para este saldo reclamable.
- último_libro_modificado
  - entero
  - El número de secuencia del último libro en el que se modificó este saldo reclamable.
- último_tiempo_modificado
  - cadena
  - Una cadena formateada en ISO 8601 de la última hora de modificación.
- reclamantes
  - array de objetos
  - La lista de entradas que podrían reclamar el saldo reclamable.
    - destino
      - cadena
      - El ID de cuenta que puede reclamar el saldo.
    - predicado
      - objeto
      - La condición que debe ser satisfecha para que `destino` pueda reclamar el saldo.
        - incondicional
          - booleano (opcional)
          - Si es verdadero, significa que esta cláusula de la condición siempre se satisface.
        - y
          - array de objetos (opcional)
          - El array siempre contendrá dos elementos que también son predicados. Esta cláusula de la condición se satisface si ambos de los dos elementos en el array se satisfacen.
        - o
          - array de objetos (opcional)
          - El array siempre contendrá dos elementos que también son predicados. Esta cláusula de la condición se satisface si al menos uno de los dos elementos en el array se satisface.
        - no
          - objeto (opcional)
          - El valor también es un predicado. Esta cláusula de la condición se satisface si el valor _no_ se satisface.
        - absAntes
          - cadena (opcional)
          - Una cadena personalizada formateada en ISO 8601 que representa un plazo para cuándo se puede reclamar el saldo reclamable. Si el saldo es reclamado antes de la fecha, entonces esta cláusula de la condición se satisface. El formato de esta cadena de fecha es una extensión personalizada sobre el formato ISO 8601. Permite que los años estén fuera del rango 0000-9999. Las fechas se derivan de un valor de época unix en el rango de un entero firmado de 64 bits. Esto significa que la fecha expresa un rango de calendario mucho más grande de 292277026596 años hacia el futuro y -292471206707 años hacia el pasado. Este formato de extensión personalizada añadirá un prefijo `'+'` en los valores que superen el año 9999 en el futuro y para los años que están antes del año 0 (A.C. según el calendario gregoriano) se añadirá un prefijo `'-'`. Aquí hay ejemplos de valores de cadenas de fecha que son posibles: `'2022-02-10T15:30:22Z'` `'+39121901036-03-29T15:30:22Z'` `'-7025-12-23T00:00:00Z'`
        - absAntesÉpoca
          - cadena (opcional)
          - Un valor de época unix en segundos que representa la misma fecha de plazo para cuándo se puede reclamar el saldo reclamable. Es el mismo valor de fecha/hora que representa absAntes, solo expresado en segundos enteros de época unix dentro del rango de un entero firmado de 64 bits.
        - relAntes
          - cadena (opcional)
          - Un plazo relativo para cuándo se puede reclamar el saldo reclamable. El valor representa el número de segundos desde el tiempo de cierre del libro que creó el saldo reclamable.

</AttributeTable>
