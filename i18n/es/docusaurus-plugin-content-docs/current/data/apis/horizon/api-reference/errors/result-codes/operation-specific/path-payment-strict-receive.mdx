---
title: Códigos de Resultado de Recepción Estricta de Pago por Ruta
sidebar_position: 30
---

Estos son códigos de resultado que comunican éxito (200) o fracaso (400) específicos de la operación `Recepción Estricta de Pago por Ruta`.

Aprender más sobre la operación [`Recepción Estricta de Pago por Ruta`](../../../../../../../learn/fundamentals/transactions/list-of-operations.mdx#path-payment-strict-receive).

<AttributeTable>

- CÓDIGO DE RESULTADO
  - CÓDIGO DEL PROTOCOLO STELLAR
  - DESCRIPCIÓN
- Éxito de operación
  - PATH_PAYMENT_STRICT_RECEIVE_SUCCESS
  - El pago se completó con éxito.
- Operación malformada
  - PATH_PAYMENT_STRICT_RECEIVE_MALFORMED
  - La entrada para este pago por ruta es inválida.
- Operación no financiada
  - PATH_PAYMENT_STRICT_RECEIVE_UNDERFUNDED
  - La cuenta de origen (remitente) no tiene suficientes lúmenes para enviar el monto del pago mientras mantiene su propia reserva mínima.
- Operación de origen sin confianza
  - PATH_PAYMENT_STRICT_RECEIVE_SRC_NO_TRUST
  - La cuenta de origen carece de la línea de confianza adecuada.
- Operación de origen no autorizada
  - PATH_PAYMENT_STRICT_RECEIVE_SRC_NOT_AUTHORIZED
  - La cuenta de origen no está autorizada para enviar este activo.
- Operación sin destino
  - PATH_PAYMENT_STRICT_RECEIVE_NO_DESTINATION
  - La cuenta de destino no existe.
- Operación sin confianza
  - PATH_PAYMENT_STRICT_RECEIVE_NO_TRUST
  - La cuenta de destino no tiene una línea de confianza para el activo que se está enviando.
- Operación no autorizada
  - PATH_PAYMENT_STRICT_RECEIVE_NOT_AUTHORIZED
  - La cuenta de destino no está autorizada para poseer este activo.
- Operación de línea completa
  - PATH_PAYMENT_STRICT_RECEIVE_LINE_FULL
  - La cuenta de destino (receptor) no tiene límites suficientes para recibir el monto y seguir satisfaciendo sus obligaciones de compra.
- Operación sin emisor
  - PATH_PAYMENT_STRICT_RECEIVE_NO_ISSUER
  - Falta el emisor de uno de los activos.
- Operación con pocas ofertas
  - PATH_PAYMENT_STRICT_RECEIVE_TOO_FEW_OFFERS
  - No hay un camino de ofertas que conecte el activo de envío y el activo de destino. Stellar solo considera caminos de longitud 5 o más cortos.
- Operación de cruce propio
  - PATH_PAYMENT_STRICT_RECEIVE_OFFER_CROSS_SELF
  - Este pago por ruta cruzaría una de sus propias ofertas.
- Operación sobre límite de origen
  - PATH_PAYMENT_STRICT_RECEIVE_OVER_SENDMAX
  - Los caminos que podrían enviar la cantidad de destino del activo de destino excederían el máximo de envío.

</AttributeTable>
