Envía un pago de una cuenta a otra a través de la ruta a través de los libros de órdenes, comenzando como un activo y terminando como otro. Los pagos de rutas que son `Envío Estricto` designan la cantidad del pago en el activo enviado.

Consulta los [errores de `Envío Estricto de Pago de Rutas`](../../../errors/result-codes/operation-specific/path-payment-strict-send.mdx).

<AttributeTable>

- ATRIBUTO
  - TIPO DE DATO
  - DESCRIPCIÓN
- tipo_de_activo
  - cadena
  - El tipo de activo que se está recibiendo. Puede ser `nativo`, `crédito_alphanum4` o `crédito_alphanum12`.
- código_de_activo
  - cadena
  - El código del activo que se está recibiendo. Aparece si el `tipo_de_activo` no es `nativo`.
- emisor_de_activo
  - cadena
  - La dirección Stellar del emisor del activo que se está recibiendo. Aparece si el `tipo_de_activo` no es `nativo`.
- de
  - cadena
  - La clave pública del remitente del pago.
- a
  - cadena
  - La clave pública del destinatario del pago.
- cantidad
  - cadena
  - Cantidad recibida designada en el activo de destino.
- ruta
  - array de objetos
  - Los activos intermedios que se atraviesan en esta ruta.
    - código_de_activo
      - cadena
      - El código de este activo intermedio.
    - emisor_de_activo
      - cadena
      - La dirección Stellar del emisor del activo intermedio.
    - tipo_de_activo
      - cadena
      - El tipo del activo intermedio. Puede ser `nativo`, `crédito_alphanum4` o `crédito_alphanum12`.
- cantidad_fuente
  - cadena
  - Cantidad enviada designada en el activo fuente.
- mín_destino
  - cadena
  - La cantidad mínima del activo de destino que se espera recibir.
- tipo_de_activo_fuente
  - cadena
  - El tipo del activo fuente. Puede ser `nativo`, `crédito_alphanum4` o `crédito_alphanum12`.
- código_de_activo_fuente
  - cadena
  - El código del activo fuente. Aparece si el `tipo_de_activo` no es `nativo`.
- emisor_de_activo_fuente
  - cadena
  - La dirección Stellar del emisor del activo fuente. Aparece si el `tipo_de_activo` no es `nativo`.

</AttributeTable>
