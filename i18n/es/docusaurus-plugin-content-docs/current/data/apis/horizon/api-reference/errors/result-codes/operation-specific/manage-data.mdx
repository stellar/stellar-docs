Estos son códigos de resultado que comunican éxito (200) o fracaso (400) específicos para la operación de `Administrar Datos`.

Aprende más sobre la operación de [`Administrar Datos`](../../../../../../../learn/fundamentals/transactions/list-of-operations.mdx#manage-data).

<AttributeTable>

- OpSuccess
  - Éxito de operación
  - La operación de administrar datos se ha ejecutado con éxito.
- op_no_soportado_aun
  - MANAGE_DATA_NOT_SUPPORTED_YET
  - La red aún no ha realizado este cambio de protocolo. Este fallo significa que la red aún no admite esta función.
- op_nombre_datos_no_encontrado
  - MANAGE_DATA_NAME_NOT_FOUND
  - Intentando eliminar una Entrada de Datos que no está presente. Esto ocurrirá si el Nombre está establecido (y el Valor no) pero la Cuenta no tiene una Entrada de Datos con ese Nombre.
- op_bajo_reserva
  - MANAGE_DATA_LOW_RESERVE
  - Esta cuenta no tiene suficiente XLM para satisfacer el aumento de reserva mínima de XLM causado por agregar una subentrada y aún cumplir con sus responsabilidades de venta de XLM. Por cada nueva Entrada de Datos añadida a una cuenta, la reserva mínima de XLM que esa cuenta debe mantener aumenta.
- op_nombre_datos_no_válido
  - MANAGE_DATA_INVALID_NAME
  - Nombre no es una cadena válida.

</AttributeTable>
