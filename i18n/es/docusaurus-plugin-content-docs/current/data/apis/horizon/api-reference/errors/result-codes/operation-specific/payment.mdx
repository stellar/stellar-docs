Estos son códigos de resultado que comunican éxito (200) o fracaso (400) específicos de la operación `Payment`, que a menudo falla porque la cuenta receptora no confía en el emisor del activo que se envía.

Obtén más información sobre la operación [`Payment`](../../../../../../../learn/fundamentals/transactions/list-of-operations.mdx#payment).

<AttributeTable>

- CÓDIGO DE RESULTADO
  - CÓDIGO DEL PROTOCOLO STELLAR
  - DESCRIPCIÓN
- Éxito de operación
  - ÉXITO_DE_PAGO
  - El pago se completó con éxito.
- Operación malformada
  - PAYMENT_MALFORMED
  - La entrada del pago no es válida.
- Operación subfinanciada
  - PAYMENT_UNDERFUNDED
  - La cuenta de origen (remitente) no tiene suficientes lumens para enviar la cantidad del pago y mantener su propia reserva mínima.
- Operación sin confianza en la fuente
  - PAYMENT_SRC_NO_TRUST
  - La cuenta de origen no tiene una línea de confianza para el activo que está intentando enviar.
- Operación no autorizada en la fuente
  - PAYMENT_SRC_NOT_AUTHORIZED
  - La cuenta de origen no está autorizada para enviar este activo.
- Operación sin destino
  - PAYMENT_NO_DESTINATION
  - La cuenta de destino no existe.
- Operación sin confianza
  - PAYMENT_NO_TRUST
  - La cuenta de destino no tiene una línea de confianza para el activo que se está enviando.
- Operación no autorizada
  - PAYMENT_NOT_AUTHORIZED
  - La cuenta de destino no está autorizada para poseer este activo.
- Operación de línea llena
  - PAYMENT_LINE_FULL
  - La cuenta de destino (receptor) no tiene límites suficientes para recibir la cantidad y aún satisfacer sus obligaciones de compra.
- Operación sin emisor
  - PAYMENT_NO_ISSUER
  - El emisor del activo no existe.

</AttributeTable>
