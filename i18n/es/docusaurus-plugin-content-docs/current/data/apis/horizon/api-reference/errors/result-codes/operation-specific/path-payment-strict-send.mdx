---
title: Códigos de Resultado de Envío Estricto de Pago por Ruta
sidebar_position: 40
---

Estos son códigos de resultado que comunican éxito (200) o fallo (400) específicos de la operación `Envío Estricto de Pago por Ruta`.

Conoce más sobre la operación [`Envío Estricto de Pago por Ruta`](../../../../../../../learn/fundamentals/transactions/list-of-operations.mdx#path-payment-strict-send).

<AttributeTable>

- CÓDIGO DE RESULTADO
  - CÓDIGO DE PROTOCOLO DE STELLAR
  - DESCRIPCIÓN
- Éxito de operación
  - PATH_PAYMENT_STRICT_SEND_SUCCESS
  - El pago se completó con éxito.
- Operación malformada
  - PATH_PAYMENT_STRICT_SEND_MALFORMED
  - La entrada para este envío por ruta es inválida.
- Operación subfinanciada
  - PATH_PAYMENT_STRICT_SEND_UNDERFUNDED
  - La cuenta fuente (remitente) no tiene suficientes lúmenes para enviar el monto del pago manteniendo su propia reserva mínima.
- Operación sin confianza en el origen
  - PATH_PAYMENT_STRICT_SEND_SRC_NO_TRUST
  - La cuenta fuente no tiene la línea de confianza apropiada.
- Operación no autorizada desde el origen
  - PATH_PAYMENT_STRICT_SEND_SRC_NOT_AUTHORIZED
  - La cuenta fuente no está autorizada para enviar este activo.
- Operación sin destino
  - PATH_PAYMENT_STRICT_SEND_NO_DESTINATION
  - La cuenta de destino no existe.
- Operación sin confianza
  - PATH_PAYMENT_STRICT_SEND_NO_TRUST
  - La cuenta de destino no tiene una línea de confianza para el activo que se envía.
- Operación no autorizada
  - PATH_PAYMENT_STRICT_SEND_NOT_AUTHORIZED
  - La cuenta de destino no está autorizada para poseer este activo.
- Operación línea llena
  - PATH_PAYMENT_STRICT_SEND_LINE_FULL
  - La cuenta de destino (receptor) no tiene límites suficientes para recibir el monto y aún satisfacer sus obligaciones de compra.
- Operación sin emisor
  - PATH_PAYMENT_STRICT_SEND_NO_ISSUER
  - Falta el emisor de uno de los activos.
- Operación ofrece muy pocas
  - PATH_PAYMENT_STRICT_SEND_TOO_FEW_OFFERS
  - No hay un camino de ofertas que conecte el activo de envío con el activo de destino. Stellar solo considera caminos de longitud 5 o cortos.
- Operación cruzar a sí mismo
  - PATH_PAYMENT_STRICT_SEND_OFFER_CROSS_SELF
  - Este envío por ruta cruzaría una de sus propias ofertas.
- Operación bajo el mínimo de destino
  - PATH_PAYMENT_STRICT_SEND_UNDER_DESTMIN
  - Los caminos que podrían enviar el monto de destino del activo de destino no cumplirían con el mínimo de destino.

</AttributeTable>
