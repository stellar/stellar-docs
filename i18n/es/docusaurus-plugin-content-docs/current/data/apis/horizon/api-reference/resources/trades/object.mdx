Cuando Horizon devuelve información sobre un comercio, utiliza el siguiente formato:

<AttributeTable>

- ATRIBUTO
  - TIPO DE DATOS
  - DESCRIPCIÓN
- id
  - cadena
  - Un identificador único para este comercio.
- token_de_paginación
  - número
  - Un valor de cursor para usar en [paginación](../../structure/pagination/README.mdx).
- hora_de_cierre_del_libro
  - cadena
  - Una cadena formateada en ISO 8601 de cuándo se cerró el ledger con este comercio.
- cuenta_base
  - cadena (opcional)
  - El ID de cuenta de la parte base para este comercio.
- id_de_oferta_base
  - cadena (opcional)
  - El ID de la oferta base. Si esta oferta fue consumida inmediata y completamente, este será un ID sintético.
- id_de_pool_de_liquidez_base
  - cadena (opcional)
  - El ID del fondo de liquidez base. Si este comercio se ejecutó contra un fondo de liquidez.
- monto_base
  - cadena
  - La cantidad del activo base que fue transferido de `base_account` a `counter_account`.
- tipo_de_activo_base
  - cadena
  - El tipo del activo base. Puede ser `native`, `credit_alphanum4`, o `credit_alphanum12`.
- código_de_activo_base
  - cadena
  - El código del activo base.
- emisor_de_activo_base
  - cadena
  - La dirección Stellar del emisor del activo base.
- cuenta_contraparte
  - cadena (opcional)
  - El ID de cuenta de la parte base para este comercio.
- id_de_oferta_contraparte
  - cadena (opcional)
  - El ID de la oferta en contra. Si esta oferta fue consumida inmediata y completamente, este será un ID sintético.
- id_de_pool_de_liquidez_contraparte
  - cadena (opcional)
  - El ID del fondo de liquidez en contra. Si este comercio se ejecutó contra un fondo de liquidez.
- monto_contraparte
  - cadena
  - La cantidad del activo en contra que fue transferido de `counter_account` a `base_account`.
- tipo_de_activo_contraparte
  - cadena
  - El tipo del activo en contra. Puede ser `native`, `credit_alphanum4`, o `credit_alphanum12`.
- código_de_activo_contraparte
  - cadena
  - El código del activo en contra.
- emisor_de_activo_contraparte
  - cadena
  - La dirección Stellar del emisor del activo en contra.
- precio
  - objeto
  - Un objeto de un numerador y un denominador que representa el precio de la oferta original. Para derivar el precio, divide `n` entre `d`. `n` y `d` están representados como cadenas porque pueden ser enteros de 64 bits.
    - n
      - cadena
      - El numerador.
    - d
      - cadena
      - El denominador.
- base_es_vendedor
  - booleano
  - Indica qué parte es el vendedor.

</AttributeTable>
