El servicio Horizon es responsable de sincronizarse con la red Stellar y procesar los datos del ledger. Para entender el alcance de los servicios de Horizon, por favor lee la sección de [configuración](./configuring.mdx) antes de pasar a los requisitos previos para la computación.

El servicio Horizon se puede [instalar](./installing.mdx) en hardware físico o en una máquina virtual. Está admitido de forma nativa en sistemas operativos Linux y Windows.

## Modelo de Despliegue de Instancia Única

Para una configuración básica utilizando el [modelo de Despliegue de Instancia Única](./configuring.mdx#single-instance-deployment), necesitarás la suma de dos perfiles de computación distintos:

- Uno para alojar el servicio Horizon
- Otro para alojar el servidor PostgreSQL

### Requisitos de hardware

Las especificaciones mínimas de hardware para ejecutar Horizon de manera efectiva son las siguientes:

#### Instancia de Computación de Horizon:

| Tipo de Nodo            | CPU    | RAM   | Disco                 | AWS SKU      | Google Cloud SKU |
| ----------------------- | ------ | ----- | --------------------- | ------------ | ---------------- |
| Servicio API de Horizon | 4 vCPU | 16 GB | 100 GB SSD >= 3K IOPS | [c5d.xlarge] | [n4-standard-4]  |

_\* Asumiendo una ventana de retención de 30 días para el almacenamiento de datos._

#### Instancia de Computación de Servidor de Base de Datos PostgreSQL:

| Tipo de Nodo          | CPU    | RAM   | Disco                                                 | AWS SKU      | Google Cloud SKU |
| --------------------- | ------ | ----- | ----------------------------------------------------- | ------------ | ---------------- |
| PostgreSQL de Horizon | 4 vCPU | 32 GB | 2 TB\* SSD (NVMe o Almacenamiento Directo) >= 7K IOPS | [i4g.xlarge] | [c3-highmem-8]   |

_\* Asumiendo una ventana de retención de 30 días para el almacenamiento de datos._

Ten en cuenta que se requiere un mínimo de la versión 12 de PostgreSQL.

Estas especificaciones suponen una ventana de retención de 30 días para el almacenamiento de datos. Para una ventana de retención más larga, los requisitos del sistema serán más altos. Para más información sobre la ingestión de datos, retención de histórico y gestión del almacenamiento, consulta la sección de [ingestión](./ingestion.mdx).

## Despliegue de Múltiples Instancias

Para lograr alta disponibilidad, redundancia y alto rendimiento, consulta la documentación de [escalado](./scaling.mdx). Proporciona una descripción detallada de varias estrategias de despliegue que puedes emplear, dependiendo del SLA que necesites que tu instancia de Horizon logre.

## Acceso a la Red

- Asegúrate de que la instancia de Horizon pueda establecer una conexión con la instancia de base de datos PostgreSQL. El puerto predeterminado para PostgreSQL es 5432.

- Se requiere una conexión de red estable y rápida con Internet para cualquier instancia de Horizon que ejecute el rol de ingestión. Esto es para asegurar que tenga conectividad saliente eficiente con hosts remotos en el [quorum set](../../../../validators/admin-guide/configuring.mdx#choosing-your-quorum-set) y [archivos de historial](../../../../validators/admin-guide/environment-preparation.mdx#history-archives) para la red Stellar elegida. Durante la ingestión, la instancia de Horizon se comunica con estos hosts, recibiendo datos de transacciones de red a través de su subproceso cautivo local core.

:::note

Los requisitos de hardware pueden aumentar a medida que la red Stellar crece y/o si compartes recursos o usas configuraciones personalizadas.
