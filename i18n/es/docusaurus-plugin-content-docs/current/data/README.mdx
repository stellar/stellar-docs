title: Vista general de datos

Los RPC, índices de datos, Hubble y Horizon son plataformas o herramientas que facilitan la comunicación entre usuarios y aplicaciones y la red Stellar. Con estas diversas plataformas, los usuarios pueden consultar datos de red, enviar transacciones, e interactuar con contratos inteligentes sin necesidad de entender los detalles de bajo nivel de Stellar Core.

Esta sección te guiará por las diferencias entre las distintas plataformas y herramientas, qué plataforma o herramienta es mejor para qué caso de uso, y luego enlazar a sus diferentes ubicaciones de documentación.

## [RPC](./rpc/README.mdx)

La RPC proporciona principalmente información que la red Stellar tiene actualmente en visión, es decir, la RPC (Red Stellar). **el estado actual**, que incluye los saldos actuales de todas las cuentas, el estado actual de los contratos inteligentes, y cualquier otra información relevante que constituya la condición actual de la cadena de bloques. Tiene la capacidad de enviar una transacción a la red y consultar el estado de las transacciones anteriores (sujeto a la ventana de retención de siete días, las transacciones anteriores devolverán una respuesta `NOT_FOUND`). El RPC está pensado para ser simple, mínimo y escalable.

El RPC proporciona los servicios de disponibilidad de datos necesarios para la mayoría de los casos de uso en tiempo real, como aplicaciones y carteras, mientras estén interesados principalmente en los datos del estado actual o estén dispuestos a ingerir eventos en su propia infraestructura para apoyar la presentación de informes, consultas analíticas y/o históricas.

Dado que el RPC no contiene la historia completa y holística de la red, debería usar Hubble para análisis de datos más complejos en períodos de tiempo más largos.

Si el RPC no satisface tus necesidades, por favor dinos por qué en [Stellar Developer Discord](https://discord. g/stellardev) o presenta un problema en [RPC repo](https://github.com/stellar/soroban-rpc).

Tienes la opción de [configurar tu propia instancia RPC](./rpc/admin-guide.mdx) o usar un servicio disponible públicamente desde [un proveedor de infraestructura](./rpc/rpc-providers.mdx).

## [Indexadores de datos](../tools/developer-tools/README.mdx#data-indexers)

Los índices de datos son herramientas especializadas que procesan e indexan datos de blockchain, haciéndolos más accesibles y consultables para los usuarios finales. Transforman datos de blockchain en un formato más estructurado con el que los usuarios finales pueden interactuar.

Los índices de datos tienen capacidades avanzadas de consulta y análisis mejorados. Proporcionan características como análisis estadístico de la actividad de blockchain, visualización de los flujos de transacción, o rastreo de métricas DeFi — capacidades que van más allá de la búsqueda básica de transacciones para obtener datos de estado actuales o históricos.

Los índices de datos son una opción potencialmente más fácil de usar y rentable para los usuarios. Echa un vistazo a varios índices de datos disponibles para la red Stellar en nuestra [sección Herramientas](../tools/developer-tools/README.mdx#data-indexers).

## [Hubble](./hubble/README.mdx)

Hubble es un almacén de datos BigQuery mantenido por SDF, de código abierto, disponible públicamente y que proporciona un registro histórico integral de la red Stellar. Es una plataforma de sólo lectura y no tiene la capacidad de enviar transacciones a la red como puede con RPC.

Hubble es genial para el análisis de datos a gran escala, complejos y exploratorios y para la visualización de transacciones, comisiones y otras tendencias a nivel de red a lo largo del tiempo. Por ejemplo, puedes usar Hubble para empaquetar juntos los contadores de un mes calcular la tarifa media de transacción para un determinado tipo de transacción, y el tiempo medio para que se incluya en el contador. Y entonces usted puede programar para recibir ese informe cada mes.

Hubble no es bueno para casos de uso que necesitan disponibilidad de datos en tiempo real, como carteras o aplicaciones, o para consultas simples, como buscar un ID de transacción. Para ellos, desea utilizar el RPC o los índices de datos.

La [documentación de Hubble](./hubble/README. dx) se centra en el análisis de datos exploratorios, describe diferentes maneras en que los desarrolladores o usuarios finales pueden conectarse a Hubble (programáticamente o a través de una interfaz de internet), y proporciona orientación sobre la optimización de consultas (los usuarios se cargan por consulta, por lo que es importante fabricar la consulta).

## [Horizon](./horizon/README.mdx)

Horizon es una API para acceder e interactuar con los datos de la red Stellar. No almacena datos de contratos inteligentes. Sin embargo, Horizon tiene una visión limitada del Contrato de Activos Estelares (SAC) y los tipos de operaciones de contratos inteligentes (i.e. se trata de transacciones con operaciones de invokeHostFunction y operaciones relacionadas con activos, ya se realicen a través del SAC de un activo o utilizando las operaciones de transacción incorporadas).

Horizón almacena tres tipos de datos (estado actual, estado histórico y estado derivado) en una base de datos, y los datos están disponibles en tiempo real para uso transaccional, lo que hace que Horizon sea más caro e intensivo en recursos para operar. Si estás considerando usar Horizon sobre el RPC, háznoslo saber en [Stellar Developer Discord](https://discord. g/stellardev) o presenta un problema en [RPC repo](https://github.com/stellar/soroban-rpc) y háganos saber por qué!

Puedes [ejecutar tu propia instancia de Horizon](./horizon/admin-guide/README.mdx) o usar uno de los servicios de Horizon disponibles públicamente desde [estos proveedores de infraestructura](./horizon/Execuon-providers.mdx).
