# Requisitos previos

## 1. Cuenta de la Plataforma en la Nube

Galexie exporta los metadatos del ledger de Stellar a Google Cloud Storage (GCS) o Amazon Simple Storage Service (S3). Necesitarás la cuenta y las credenciales correspondientes para el servicio de almacenamiento en la nube que elijas usar.

### Cuenta de Google Cloud Platform (GCP) para GCS

- Permisos para crear un nuevo bucket de GCS, o
- Acceso a un bucket existente con permisos de lectura/escritura.

### Cuenta de Amazon Web Services (AWS) para S3

- Permisos para crear un nuevo bucket S3, o
- Acceso a un bucket existente con permisos de lectura y escritura.

## 2. Tiempo de ejecución de contenedores (Recomendado)

### Kubernetes

- Kubernetes 1.19+

### Ejecutar el contenedor Galexie en una Instancia

- Instancia como AWS EC2 o VM de GCP
- Cualquier máquina host con un tiempo de ejecución de contenedores compatible con OCI instalado como Docker ([Guía de instalación de Docker](https://docs.docker.com/engine/install/)).

> **_NOTA:_** Aunque es posible instalar Galexie de forma nativa (sin un tiempo de ejecución de contenedores), esto requiere la gestión manual de dependencias y se recomienda solo para usuarios avanzados.

## Requisitos de Hardware

Los requisitos mínimos de hardware para ejecutar Galexie son:\
**RAM**: 16 GB\
**CPU**: 4 vCPUs\
**Disco persistente**: 100 GB con al menos 5K IOPS
