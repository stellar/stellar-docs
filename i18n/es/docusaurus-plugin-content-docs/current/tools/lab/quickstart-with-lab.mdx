# Usar Lab con Quickstart

## Descripción general

[Quickstart](https://github.com/stellar/quickstart) proporciona una manera fácil de configurar un entorno de red Stellar local. Específicamente, la imagen de docker de Quickstart agrupa Stellar Core con Horizon, RPC, Friendbot y las bases de datos PostgreSQL necesarias. Ahora es posible usar Stellar Lab como una interfaz en Quickstart.

## Requisitos previos

- [Stellar CLI](/docs/tools/cli/install-cli)
- [Docker](https://www.docker.com/)

## Iniciar Quickstart

Quickstart se puede iniciar para diferentes redes. En este ejemplo, iniciaremos Quickstart para `testnet`. Inicia Quickstart con el Stellar CLI usando el siguiente comando:

```sh
stellar network container start testnet
```

Quickstart generalmente se iniciará en `http://localhost:8000`. Con esta información, ahora podemos configurar Stellar Lab para usar la red local.

## Configurar Stellar Lab para usar Horizon Local y RPC Local

![Lab: selector de red](/assets/lab/lab-custom-network.png)

1. Abre el selector de red de Stellar Lab en la esquina superior derecha.
2. Selecciona la red `Personalizada` en el menú desplegable.
3. Para la URL de RPC, ingresa `http://localhost:8000/rpc`.
4. Para la URL de Horizon, ingresa `http://localhost:8000.`.
5. Para el [pase de red](/docs/networks), ingresa `Test SDF Network ; September 2015`
6. Haz clic en el botón `Cambiar a la red personalizada` y cambia a usar tu Quickstart.

Ahora deberías poder usar los puntos finales de Horizon, los puntos finales de RPC y Friendbot en Lab, con solicitudes que se envían a tu entorno local (Quickstart). Lab te proporciona una interfaz amigable para interactuar con tu entorno local.
