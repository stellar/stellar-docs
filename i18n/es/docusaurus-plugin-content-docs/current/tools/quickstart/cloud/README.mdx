# Despliegue en la nube

Quickstart puede desplegarse en una plataforma en la nube para crear una red de pruebas compartible para pruebas coordinadas y multiusuario.

Cualquier plataforma que admita un entorno de ejecución de contenedores puede desplegar la imagen de Quickstart:

```
docker.io/stellar/quickstart:latest
```

Utiliza las siguientes variables de entorno para configurar la imagen:

- `NETWORK` - La red que el contenedor configurará o a la que se conectará. Uno de:
  - `local` - Se creará una nueva red local para desarrollo/pruebas.
  - `testnet` - El contenedor se conectará a la testnet pública.
- `NETWORK_PASSPHRASE` - Para redes `local`, la frase de contraseña de red que se usará.
- `RANDOMIZE_NETWORK_PASSPHRASE` - Para redes `local`, la frase de contraseña tendrá un sufijo aleatorio.
- `LIMITS` - Los límites de red que se configurarán para los contratos inteligentes. Uno de:
  - `testnet` - Límites que igualarán la testnet pública.
  - `unlimited` - Límites máximos admitidos por la red bajo cualquier configuración. Útil para pruebas experimentales antes de la optimización.

:::warning

Quickstart está diseñado para desarrollo y pruebas, no para producción.

:::
