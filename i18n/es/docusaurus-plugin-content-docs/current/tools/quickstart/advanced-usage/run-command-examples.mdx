# Ejecutar comandos

A continuación, se presenta una lista de varias maneras en que puedes ejecutar el contenedor Quickstart, anotadas para ilustrar qué opciones están habilitadas. También se recomienda que aprendas y te familiarices con el comando de docker. Iniciar una red dev/test local efímera solamente:

<Tabs groupId="platform" defaultValue={getPlatform()}>

<TabItem value="unix" label="macOS/Linux">

```sh
docker run -d \
  -p "8000:8000" \
  --name stellar \
  stellar/quickstart \
  --local
```

</TabItem>

<TabItem value="windows" label="Windows (PowerShell)">

```powershell
docker run -d `
  -p "8000:8000" `
  --name stellar `
  stellar/quickstart `
  --local
```

</TabItem>

</Tabs>

Iniciar un nodo testnet efímero en primer plano:

<Tabs groupId="platform" defaultValue={getPlatform()}>

<TabItem value="unix" label="macOS/Linux">

```sh
docker run --rm -it \
  -p "8000:8000" \
  --name stellar \
  stellar/quickstart \
  --testnet
```

</TabItem>

<TabItem value="windows" label="Windows (PowerShell)">

```powershell
docker run --rm -it `
  -p "8000:8000" `
  --name stellar `
  stellar/quickstart `
  --testnet
```

</TabItem>

</Tabs>

Iniciar un nuevo nodo persistente usando el directorio del host `/str`:

<Tabs groupId="platform" defaultValue={getPlatform()}>

<TabItem value="unix" label="macOS/Linux">

```sh
docker run -it --rm \
  -p "8000:8000" \
  -v "/str:/opt/stellar" \
  --name stellar \
  stellar/quickstart \
  --testnet
```

</TabItem>

<TabItem value="windows" label="Windows (PowerShell)">

```powershell
docker run -it --rm `
  -p "8000:8000" `
  -v "/str:/opt/stellar" `
  --name stellar `
  stellar/quickstart `
  --testnet
```

</TabItem>

</Tabs>
