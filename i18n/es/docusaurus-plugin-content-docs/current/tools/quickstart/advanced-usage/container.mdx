# Contenedor

Quickstart se empaqueta como una imagen de contenedor de Docker y proporciona una configuración efímera predeterminada y no validante que debería funcionar para la mayoría de los desarrolladores. Al configurar un contenedor con esta imagen usando un volumen basado en el host (descrito a continuación en la sección "Configuración"), un operador obtiene acceso a la personalización completa de la configuración y la persistencia de datos.

La imagen de Quickstart [docker](https://hub.docker.com/r/stellar/quickstart) utiliza el siguiente software:

- [PostgreSQL](https://www.postgresql.org)
- [stellar-core](https://github.com/stellar/stellar-core)
- [horizon](https://github.com/stellar/go/tree/master/services/horizon)
- [friendbot](https://github.com/stellar/go/tree/master/services/friendbot)
- [stellar-rpc](https://github.com/stellar/stellar-rpc/tree/main/cmd/stellar-rpc)
- [supervisord](http://supervisord.org/)

El CLI de Stellar tiene comandos para iniciar y detener un contenedor de Quickstart, y también un comando para obtener los registros de un contenedor en ejecución. Consulta la [documentación](../../cli/stellar-cli.mdx#stellar-container) del CLI de Stellar para más información.
