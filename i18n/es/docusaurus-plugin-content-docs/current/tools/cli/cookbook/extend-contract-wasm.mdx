Puedes usar Stellar CLI para extender el TTL del bytecode Wasm de un contrato. Esto se puede hacer de dos formas: si tienes o no el contrato compilado localmente. Si tienes el binario compilado en tu máquina local:

```bash
stellar contract extend \
    --source S... \
    --network testnet \
    --wasm ../relative/path/to/soroban_contract.wasm \
    --ledgers-to-extend 535679 \
    --durability persistent
```

Este ejemplo usa 535,679 ledgers como el nuevo TTL de archivo. Este es el valor máximo permitido para este argumento en la CLI. Esto corresponde aproximadamente a 30 días (con un promedio de cierre de ledger de 5 segundos).

Si no tienes el binario compilado en tu máquina local, aún puedes usar la CLI para extender el TTL del bytecode. Necesitarás conocer el hash Wasm del código de contrato instalado:

```bash
stellar contract extend \
    --source S... \
    --network testnet \
    --wasm-hash <hex-encoded-wasm-hash> \
    --ledgers-to-extend 535679 \
    --durability persistent
```

:::info

Puedes aprender más sobre cómo encontrar el hash Wasm correcto para una instancia de contrato [aquí (JavaScript)](../../../build/guides/rpc/retrieve-contract-code-js.mdx) y [aquí (Python)](../../../build/guides/rpc/retrieve-contract-code-python.mdx).
