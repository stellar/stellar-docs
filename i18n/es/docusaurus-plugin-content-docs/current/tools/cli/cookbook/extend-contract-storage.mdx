Puedes usar el CLI de Stellar para extender el TTL de la entrada de almacenamiento persistente de un contrato. Para una entrada de almacenamiento que usa un `Symbol` simple como clave de almacenamiento, puedes ejecutar un comando así:

```bash
stellar contract extend \
    --source S... \
    --network testnet \
    --id C... \
    --key COUNTER \
    --ledgers-to-extend 535679 \
    --durability persistent
```

Este ejemplo usa 535,679 libros contables como el nuevo TTL de archivo. Este es el valor máximo permitido para este argumento en el CLI. Esto corresponde aproximadamente a 30 días (promediando tiempos de cierre de libro contable de 5 segundos).

Si tu entrada de almacenamiento usa una clave más avanzada, como `Balance(Address)` en un contrato de token, necesitarás proporcionar la clave en forma XDR codificada en base64:

```bash
stellar contract extend \
    --source S... \
    --network testnet \
    --id C... \
    --key-xdr AAAABgAAAAHXkotywnA8z+r365/0701QSlWouXn8m0UOoshCtNHOYQAAAA4AAAAHQmFsYW5jZQAAAAAB \
    --ledgers-to-extend 535679 \
    --durability persistent
```

:::info

Asegúrate de consultar nuestra [guía sobre cómo crear claves XDR para el libro contable](../../../build/guides/rpc/generate-ledger-keys-python.mdx) para obtener ayuda generándolas.

:::
