Puedes usar la CLI de Stellar para extender el TTL de una instancia de contrato como sigue:

```bash
stellar contract extend \
    --source S... \
    --network testnet \
    --id C... \
    --ledgers-to-extend 535679 \
    --durability persistent
```

Este ejemplo usa 535,679 libros contables como el nuevo TTL para archivo. Este es el valor máximo permitido para este argumento en la CLI. Esto corresponde aproximadamente a 30 días (calculando un promedio de 5 segundos por cierre de libro contable).

Cuando extiendes una instancia de contrato, esto incluye:

- la propia instancia del contrato
- cualquier entrada `env.storage().instance()` dentro del contrato
- el código Wasm del contrato
