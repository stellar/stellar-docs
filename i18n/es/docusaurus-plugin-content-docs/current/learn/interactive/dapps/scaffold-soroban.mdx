## Dapps Soroban demostrativas

import ReactPlayer from "react-player";

El equipo **Soroban** ha invertido un esfuerzo considerable en la implementación de contratos. Han construido CLI y bibliotecas, permitiendo a los constructores de contratos crear e invocar usando Rust, que forma el “backend” de Soroban. Sin embargo, el “frontend”, que involucra bibliotecas cliente JS, requiere atención.

## Fondo

Durante el Soroban Hackathon, reconocimos que si bien las bibliotecas frontend eran funcionales, su experiencia de usuario estaba lejos de ser ideal. Además, la falta de ejemplos claros hizo más difícil a los creadores de dapp diseñar y entender la UX de Soroban.

Para hacer frente a esto, el equipo de **Ingeniería de Cartera**, en colaboración con el equipo Soroban, ha decidido lanzar “Scaffold Soroban”, una colección de dapps de demostración. Estas dapps demuestran funcionalidades básicas de una manera estructurada y fácil de seguir, centrándose principalmente en cómo construir/desplegar invocaciones de contratos de Soroban.

## Dapp Demos

Para una fácil accesibilidad, hemos compilado las dapps en un [único repositorio](https://github.com/stellar/scaffold-soroban), que contiene las siguientes aplicaciones:

## Cómo explorar las Dapps en Scaffold-Soroban

Para comenzar a utilizar estos ejemplos, vaya a [Scaffold-Soroban](https://scaffold-soroban.stellar.org/) y elija el nombre de la dapp en la lista desplegable "select demo":

- **Pago**: Elige "pago".
- **Token Minter**: Selecciona "mint-token".
- **Atomic Swap**: Opt for "atomic-swap".

¡Acérquese y descubra el poder de Soroban!

## Funcionalidad Contra las Dapps

Con la introducción de estos dapps, vamos a profundizar en algunas de sus [características destacadas](https://github.com/stellar/soroban-react-atomic-swap/blob/main/src/helpers/soroban.ts) que muestran la potencia y la innovación detrás de las aplicaciones:

Funcionalidad detrás de las dapps es extensa y diversa, aprovechando la biblioteca `@stellar/stellar-sdk` para integrarse con el Soroban RPC y facilitar la comunicación directa con Soroban utilizando una interfaz RPC JSON en la red Stellar. Están equipados para comunicarse a través de diferentes configuraciones de red, ya que incorporan las funcionalidades `RPC_URLS` y `getServer`, proporcionar adaptabilidad que es crucial para diversos escenarios de desarrollo e implementación. Los usuarios pueden recuperar información de token fácil de usar sin comprometerse con operaciones complejas de blockchain, utilizando funciones como `getTokenSymbol`, `getTokenName` y `getTokenDecimals`.

Una característica significativa es la función 'simulateTx', que permite a los usuarios previsualizar el resultado de una transacción antes de ejecutarla. `simulateTx` permite a los usuarios enviar una invocación de contrato de prueba ejecutando primero una simulación de la invocación del contrato como se define en la transacción entrante. Los resultados se aplican a una nueva copia de la transacción, que se devuelve con la huella del libro y el conjunto de autorizaciones, haciéndola lista para firmar y enviar. La transacción devuelta también tendrá una cuota actualizada, la suma de la cuota fijada en la transacción entrante con las tasas de recursos del contrato estimadas a partir de la simulación. Es recomendable comprobar la comisión de la transacción devuelta y validar o tomar las medidas apropiadas para la interacción con el usuario para confirmar que es aceptable.

También se proporcionan otras utilidades como `accountToScVal` y `numberToI128` para simplificar la creación de transacciones convirtiendo entradas fáciles de usar en los formatos esperados por el Soroban RPC en la red Stellar. Además, las dapps se construyen con funciones de ayuda prefabricadas como la función 'makePayment', que facilita las operaciones de "transferencia" y también incluye memos para información adicional relacionada con la transacción.

El código referenciado muestra varias funcionalidades incluyendo el envío de transacciones, recuperación de información de tokens, simulación de transacciones, construyendo canjes, y autorizando llamadas a contratos, todos ellos listos para ser clonados, personalizados y ampliados para satisfacer sus necesidades e ideas únicas.
