Las operaciones son objetos que representan un cambio deseado en el libro de contabilidad y se envían a la red agrupada en una transacción. Para cada operación, hay un tipo de resultado exitoso o fallido. En caso de éxito, el usuario puede reunir información sobre el efecto de la operación. En caso de fallo, el usuario puede aprender más sobre el error.

Obtenga más información sobre las transacciones y operaciones en nuestra [Sección de Operaciones y Transacciones](./operations-and-transactions.mdx).

Actualmente hay 26 operaciones que puede utilizar en la red Stellar, estas operaciones, sus definiciones, SDKs, umbrales, parámetros y errores están listados a continuación.

## Crear cuenta

Crea y financia una nueva cuenta con el saldo inicial especificado

**SDKs**: [JavaScript](http://stellar.github.io/js-stellar-sdk/Operation.html#.createAccount) | [Java](https://github.com/lightsail-network/java-stellar-sdk/blob/master/src/main/java/org/stellar/sdk/CreateAccountOperation.java) | [Go](https://godoc.org/github.com/stellar/go/txnbuild#CreateAccount)\
**Threshold**: Medium\
**Result**: `CreateAccountResult`\
**Parámetros**: **SDKs**:

| Parámetro     | Tipo         | Descripción                                                                                |
| ------------- | ------------ | ------------------------------------------------------------------------------------------ |
| Destino       | ID de cuenta | Dirección de cuenta creada y financiada.                                                   |
| Saldo inicial | entero       | Cantidad de XLM a enviar a la cuenta recién creada. Este XLM viene de la cuenta de origen. |

**Posibles errores**:

| Error                     | Código | Descripción                                                                                                                                                                                                          |
| ------------------------- | ------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| CREAR CUENT_MALFORMED     | \-1    | El `destino` no es válido.                                                                                                                                                                                           |
| CREATE_CUENT_UNDERFUNDADO | \-2    | La cuenta de origen que realiza el comando no tiene fondos suficientes para dar a `destino` la cantidad de `saldo inicial` de XLM y mantener su reserva mínima XLM además de satisfacer sus pasivos de venta de XLM. |
| CREATE_CUENT_DESCRIPTION  | \-3    | Esta operación crearía una cuenta con menos del número mínimo de XLM que una cuenta debe tener.                                                                                                                      |
| CREATE_ACCOUNT_TITLE      | \-4    | La cuenta `destination` ya existe.                                                                                                                                                                                   |

## Pago

Envía una cantidad en un activo específico a una cuenta de destino

**SDKs**: [JavaScript](http://stellar.github.io/js-stellar-sdk/Operation.html#.payment) | [Java](https://github.com/lightsail-network/java-stellar-sdk/blob/master/src/main/java/org/stellar/sdk/PaymentOperation.java) | [Go](https://godoc.org/github.com/stellar/go/txnbuild#Payment)\
**Threshold**: Medium\
**Result**: `PaymentResult`\
**Parámetros**:

| Parámetros    | Tipo         | Descripción                                  |
| ------------- | ------------ | -------------------------------------------- |
| Destino       | ID de cuenta | Dirección de la cuenta que recibe el pago.   |
| Asset         | asset        | Activo para enviar a la cuenta de destino.   |
| Cantidad      | entero       | Cantidad del activo a enviar.                |

**Posibles errores**:

| Error                 | Código | Descripción                                                                                                                                                                                                                                                           |
| --------------------- | ------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| PAYMENT_MALFORMED     | \-1    | La entrada al pago no es válida.                                                                                                                                                                                                                                      |
| PAYMENT_UNDERFUNDADO  | \-2    | La cuenta de origen (remitente) no tiene fondos suficientes para enviar `cantidad` y aún así satisfacer sus responsabilidades de venta. Tenga en cuenta que si envía XLM entonces el remitente debe mantener adicionalmente su reserva mínima XLM. |
| PAYMENT_SRC           | \-3    | La cuenta de origen no confía en el emisor del activo que está intentando enviar.                                                                                                                                                                                     |
| PAYMENT_SRC_NOT_AUT   | \-4    | La cuenta de origen no está autorizada a enviar este pago.                                                                                                                                                                                                            |
| NO_DESTINATION        | \-5    | La cuenta receptora no existe. Tenga en cuenta que este error **no** será devuelto si la cuenta receptora es el emisor de `asset`.                                                                                                                                    |
| PAYMENT_NO_TITLE      | \-6    | El receptor no confía en el emisor del activo enviado. Para más información, vea la [sección de Activos](../stellar-data-structures/assets.mdx).                                                                                                                      |
| PAYMENT_NOT_AUTORIZED | \-7    | La cuenta de destino no está autorizada por el emisor del activo para mantener el activo.                                                                                                                                                                             |
| PAYMENT_TITLE         | \-8    | La cuenta de destino (receptor) no tiene límites suficientes para recibir `cantidad` y todavía satisfacer sus pasivos de compra.                                                                                                                   |

## Envío estricto de pago por ruta

Un pago donde el activo enviado puede ser diferente al activo recibido; permite al usuario especificar la cantidad del activo a enviar

Más información sobre los pagos en la ruta: [Entrada Enciclopedia de Pagos en la ruta](../../encyclopedia/transactions-specialized/path-payments.mdx)

**SDKs**: [JavaScript](http://stellar.github.io/js-stellar-sdk/Operation.html#.pathPaymentStrictSend) | [Java](https://github.com/lightsail-network/java-stellar-sdk/blob/master/src/main/java/org/stellar/sdk/PathPaymentStrictSendOperation.java) | [Go](https://godoc.org/github.com/stellar/go/txnbuild#PathPaymentStrictSend)\
**Threshold**: Medium\
**Result**: `PathPaymentStrictSendResult`\
**Parámetros**:

| Parámetros        | Tipo             | Descripción                                                                                                                                                                                                                                                                                      |
| ----------------- | ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Enviar activo     | asset            | El activo deducido de la cuenta del remitente.                                                                                                                                                                                                                                                   |
| Enviar monto      | entero           | La cantidad de `enviar activo` a deducir (excluyendo honorarios).                                                                                                                                                                                                             |
| Destino           | ID de cuenta     | ID de cuenta del destinatario.                                                                                                                                                                                                                                                                   |
| Activo de destino | asset            | El activo que recibe la cuenta de destino.                                                                                                                                                                                                                                                       |
| Mínimo destino    | entero           | La cantidad mínima de `activo de destino` que la cuenta de destino puede recibir.                                                                                                                                                                                                                |
| Ruta              | lista de activos | Los activos (además de `enviar activo` y `recurso de destino`) involucrados en las ofertas que la ruta toma. Por ejemplo, si solo puede encontrar una ruta de USD a EUR a través de XLM y BTC, la ruta sería USD -> XLM -> BTC -> EUR y el campo `path` contendría XLM y BTC. |

**Posibles errores**:

| Error                           | Código | Descripción                                                                                                                                                                                                                                      |
| ------------------------------- | ------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| PATH_PAYMENT_SEND               | \-1    | La entrada a esta ruta de pago no es válida.                                                                                                                                                                                                     |
| PATH_PAYMENT_SEND_DESDERFUND    | \-2    | La cuenta de origen (remitente) no tiene fondos suficientes para enviar y todavía satisfacer sus pasivos de venta. Tenga en cuenta que si envía XLM entonces el remitente debe mantener adicionalmente su reserva mínima XLM. |
| PATH_PAYMENT                    | \-3    | La cuenta de origen no confía en el emisor del activo que está intentando enviar.                                                                                                                                                                |
| PATH_PAYMENT_STRICT             | \-4    | La cuenta de origen no está autorizada a enviar este pago.                                                                                                                                                                                       |
| PATH_PAYMENT                    | \-5    | La cuenta de destino no existe.                                                                                                                                                                                                                  |
| PATH_PAYMENT                    | \-6    | La cuenta de destino no confía en el emisor del activo enviado. Para más información, vea la [sección de Activos](../stellar-data-structures/assets.mdx).                                                                                        |
| PATH_PAYMENT_SEND_NOT_AUTORIZED | \-7    | La cuenta de destino no está autorizada por el emisor del activo para mantener el activo.                                                                                                                                                        |
| PATH_PAYMENT                    | \-8    | La cuenta de destino no tiene límites suficientes para recibir `cantidad de destino` y aún así satisfacer sus pasivos de compra.                                                                                                                 |
| PATH_PAYMENT                    | \-10   | No hay ninguna ruta de ofertas que conecten el recurso `enviar el recurso` y `recurso de destino`. Estelar sólo considera caminos de longitud 5 o más cortos.                                                                                    |
| PATH_PAYMENT                    | \-11   | El pago cruzaría una de sus propias ofertas.                                                                                                                                                                                                     |
| PATH_PAYMENT_SEND_SEND_DESTMIN  | \-12   | Las rutas que podrían enviar `cantidad de destino` de `activo de destino` no estarían a la altura de `destino mío`.                                                                                                                              |

## Administrar oferta de compra

Crea, actualiza o elimina una oferta para comprar una cantidad específica de un activo para otro

Más información sobre ofertas de venta pasivas: [Liquidez en Stellar: SDEX y piscinas de liquidez](../../encyclopedia/sdex/liquidity-on-stellar-sdex-liquidity-pools.mdx)

**SDKs**: [JavaScript](http://stellar.github.io/js-stellar-sdk/Operation.html#.manageBuyOffer) | [Java](https://github.com/lightsail-network/java-stellar-sdk/blob/master/src/main/java/org/stellar/sdk/ManageBuyOfferOperation.java) | [Go](https://godoc.org/github.com/stellar/go/txnbuild#ManageBuyOffer)\
**Threshold**: Medium\
**Result**: `ManageBuyOfferResult`\
**Parámetros**:

| Parámetros   | Tipo                     | Descripción                                                                                                                             |
| ------------ | ------------------------ | --------------------------------------------------------------------------------------------------------------------------------------- |
| Vender       | asset                    | Activo que el creador de la oferta está vendiendo.                                                                                      |
| Comprando    | asset                    | Activo que el creador de la oferta está comprando.                                                                                      |
| Cantidad     | entero                   | Cantidad de compra que se compra. Establezca en `0` si desea eliminar una oferta existente.                                             |
| Precio       | {numerator, denominator} | Precio de 1 unidad de 'comprar' en términos de 'vender'. Por ejemplo, si quieres comprar 30 XLM y vender 5 BTC, el precio sería {5,30}. |
| ID de oferta | entero sin signo         | El ID de la oferta. `0` para una nueva oferta. Establecer el ID de oferta existente para actualizar o eliminar.                         |

**Posibles errores**:

| Error                             | Código | Descripción                                                                                                                                                                                                                                                                                                                                       |
| --------------------------------- | ------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| OFER_MALFORMED                    | \-1    | La entrada es incorrecta y resultaría en una oferta no válida.                                                                                                                                                                                                                                                                                    |
| MANAGE_POPUP_POPUP_TITLE          | \-2    | La cuenta que crea la oferta no tiene una línea de confianza para el activo que está vendiendo.                                                                                                                                                                                                                                                   |
| OFER_POPUP_PLAYLIST               | \-3    | La cuenta que crea la oferta no tiene una línea de confianza para el activo que está comprando.                                                                                                                                                                                                                                                   |
| BANAGE_BUY_OFER_BUY_NOT_AUTORIZED | \-4    | La cuenta que crea la oferta no está autorizada a vender este activo.                                                                                                                                                                                                                                                                             |
| OFERT_OFER_SELL_NOT_AUT           | \-5    | La cuenta que crea la oferta no está autorizada a comprar este activo.                                                                                                                                                                                                                                                                            |
| MANAGE_POPUP_BTN                  | \-6    | La cuenta de creación de la oferta no tiene límites suficientes para recibir 'comprar' y todavía satisfacer sus pasivos de compra.                                                                                                                                                                                                                |
| DESDERFUNDADO                     | \-7    | La cuenta de creación de la oferta no tiene límites suficientes para enviar 'vender' y aún así satisfacer sus responsabilidades de venta. Tenga en cuenta que si la venta de XLM entonces la cuenta debe mantener adicionalmente su reserva mínima de XLM, que se calcula asumiendo que esta oferta no se ejecutará completamente inmediatamente. |
| MANAGE_BUT_OFER_CROS_SELF         | \-8    | La cuenta tiene una oferta opuesta de precio igual o menor activa, por lo que la cuenta creando esta oferta inmediatamente se cruzaría.                                                                                                                                                                                                           |
| OFERT_NOTIF_DESCRIPTION           | \-11   | Una oferta con ese 'offerID' no se encuentra.                                                                                                                                                                                                                                                                                                     |
| OFER_DESCRIPTION                  | \-12   | La cuenta de creación de esta oferta no tiene suficiente XLM para satisfacer el incremento mínimo de reserva XLM causado por la adición de una subentrada y aún así satisfacer sus pasivos de venta XLM. Para cada oferta que una cuenta cree, la cantidad mínima de XLM que la cuenta debe tener aumentará.                                      |

| CLAWBACK_CLAIMABLE                        | \-1    | No existe ClaimableBalanceEntry que coincida con el BalanceID de entrada.              |
| CLAWBACK_CLAIMABLE                        | \-2    | La cuenta de origen no es el emisor del activo en el saldo cobrable.                   |
| CLAWBACK_BALANCE_NOT_CLAWBACK_DESCRIPTION | \-3    | `The CLAIMABLE_BALANCE_CLAWBACK_ENABLED_FLAG` no está establecido para este trustline. |

## Establecer banderas de confianza

Permite emitir la cuenta para configurar las banderas de autorización y línea de confianza a un activo

El parámetro Activo es del tipo `TrustLineAsset`. Si está modificando una línea de confianza a un activo regular (es decir, uno en un formato Code:Emisor), esto es equivalente al tipo de Activo. Sin embargo, si está modificando una línea de confianza a una acción de billar, esto se compone de un ID único de la batería de liquidez.

Aprende más sobre banderas: [Entrada del glosario de banderas](../../glossary.mdx#flags)

**Threshold**: Bajo\
**Resultado**: `SetTrustLineFlagsResult`\
**Parámetros**:

| Parámetros      | Tipo           | Descripción                                                                                                                                                                                                                                                                                          |
| --------------- | -------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Fiduciario      | ID de cuenta   | La cuenta que estableció esta línea de confianza.                                                                                                                                                                                                                                                    |
| Asset           | TrustLineAsset | La línea de confianza de activos cuyas banderas se están modificando.                                                                                                                                                                                                                                |
| SetFlags        | entero         | Una o más banderas (combinadas a través de bitwise-OR) que indican qué banderas establecer. Las posibles banderas son: 1 si el trustor está autorizado a transaccionar con el activo o 2 si el trustor está autorizado a mantener ofertas pero no a realizar otras transacciones. |
| Borrar banderas | entero         | Una o más banderas (combinadas a través de OR en sentido de bits) que indican qué banderas despejar. Las posibilidades incluyen las de SetFlags así como 4, lo que impide que el emisor recupere su activo (tanto de cuentas como de saldos cobrables).        |

**Posibles errores**:

| Error               | Código | Descripción                                                                                                                                                                                                                                                                                               |
| ------------------- | ------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| FLAGS_MALFORMED     | \-1    | Esto puede suceder por varias razones: el activo especificado por AssetCode y AssetIssuer no es válido; el emisor de activos no es la cuenta de origen; el Trustor es la cuenta fuente; se especifica el activo nativo; o las banderas que se están estableciendo/eliminando conflictos o no son válidas. |
| PLAYLIST_BTN        | \-2    | El Trustor no tiene una línea de confianza con el emisor que realiza esta operación.                                                                                                                                                                                                                      |
| PLAYLIST_BTN        | \-3    | El emisor está tratando de revocar la autorización de la línea de confianza de Trustor, pero no puede hacerlo porque AUTH_REVOCABLE_FLAG no está establecido en la cuenta.                                                                                                                                |
| FLAGS_INVALID_TITLE | \-4    | Si el estado final de la línea de confianza tiene tanto AUTHORIZED_FLAG (1) como AUTHORIZED_TO_MAINTAIN_LIABILITIES_FLAG (2) definido, que son mutuamente excluyentes.                                                                                              |
| PLAYLIST_BTN        | \-5    | Los saldos reclamables no se pueden crear en base a la revocación de las líneas de confianza de activos (o acciones compartidas) asociadas con una piscina de liquidez debido a las reservas bajas.                                                                                    |

## Depósito de piscina de liquidez

Deposita activos en una piscina de liquidez, aumentando las reservas de una piscina de liquidez a cambio de acciones de la piscina

Los parámetros de esta operación dependen del pedido de activos en el fondo de liquidez: “A” se refiere al primer activo de la batería de liquidez, y “B” se refiere al segundo activo del pozo de liquidez.

Si la piscina está vacía, entonces esta operación deposita maxAmountA de A y maxAmountB de B en la piscina. Si la piscina no está vacía, entonces esta operación deposita a la mayoría maxAmountA de A y maxAmountB de B en la piscina. Los importes reales depositados se determinan utilizando las reservas actuales de la piscina. Puede utilizar estos parámetros para controlar un porcentaje de deslizamiento.

Aprenda más sobre las piscinas de liquidez: [Entrada Enciclopedia de piscinas de liquidez](../../encyclopedia/sdex/liquidity-on-stellar-sdex-liquidity-pools.mdx)

**Umbral**: Medio\
**Resultado**: `LiquidityPoolDepositResult`\
**Parámetros**:

| Parámetros        | Tipo                     | Descripción                                               |
| ----------------- | ------------------------ | --------------------------------------------------------- |
| Liquidez Pool ID  | liquididad-PoolID        | El PoolID para la Planta de Liquidez en el que depositar. |
| Cantidad Máxima A | entero                   | Cantidad máxima del primer activo a depositar.            |
| Cantidad Máxima B | entero                   | Cantidad máxima del segundo activo a depositar.           |
| Precio mínimo     | {numerator, denominator} | Depósito mínimo/depósito.                                 |
| Precio máximo     | {numerator, denominator} | Depósito máximoA/Depósito.                                |

**Posibles errores**:

| Error                      | Código | Descripción                                                                     |
| -------------------------- | ------ | ------------------------------------------------------------------------------- |
| IQUIDITY_POOL              | \-1    | Una o más de las entradas a la operación estaban mal formadas.                  |
| IQUIDITY_POOL              | \-2    | No existe ninguna línea de confianza para uno de los activos que se depositan.  |
| POOL_DEPOSIT_NOT_AUTORIZED | \-3    | La cuenta no tiene autorización para uno de los activos.                        |
| POOL_DEPOSIT_UNDERFUNDADED | \-4    | No hay suficiente saldo de uno de los activos para realizar el depósito.        |
| IQUIDITY_POPUP_DESCRIPTION | \-5    | La línea de confianza de la acción de la piscina no tiene un límite suficiente. |
| BAD_BAD_PRICE              | \-6    | El precio del depósito está fuera de los límites dados.                         |
| IQUIDITY_POOL              | \-7    | Las reservas de la piscina de liquidez están llenas.                            |

## Invocar función de host

Invoca e implementa contratos inteligentes de Soroban con `InvokeHostFunctionOp`.

La opción `InvokeHostFunctionOp` puede utilizarse para realizar las siguientes operaciones de Soroban:

- Invocar funciones de contrato: `HOST_FUNCTION_TYPE_INVOKE_CONTRACT`
- Subida Wasm de los contratos: `HOST_FUNCTION_TYPE_UPLOAD_CONTRACT_WASM`
- Desplegar nuevos contratos usando Wasm subido o implementaciones incorporadas: `HOST_FUNCTION_TYPE_CREATE_CONTRACT`

Tenga en cuenta que las transacciones de Soroban sólo pueden contener una operación por transacción.

Más información [here](../../encyclopedia/contract-development/contract-interactions/stellar-transaction.mdx#invokehostfunctionop).

**Umbral**: Medio\
**Resultado**: `InvokeHostFunctionResult`\
**Parámetros**:

| Parámetros      | Tipo                               | Descripción                                             |
| --------------- | ---------------------------------- | ------------------------------------------------------- |
| Función de host | Función de host                    | La función de host a invocar                            |
| Aut             | Entrada de autorización de Soroban | Autorizaciones por dirección para esta función de host. |

**Posibles errores**:

| Error                                 | Código | Descripción                                                                                                                                       |
| ------------------------------------- | ------ | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| HOST_FUNCTION_MALFORMED               | \-1    | Una o más de las entradas a la operación estaban mal formadas.                                                                                    |
| HOST_FUNCTION_TRAPPED                 | \-2    | La invocación de la función atrapada en el tiempo de ejecución del Soroban.                                                                       |
| HOST_FUNCTION_RESOURCE_LIMIT_EXCEDED  | \-3    | La invocación de función no pudo completarse dentro de las restricciones de recursos actualmente configuradas de la red.                          |
| INVOKE_HOST_FUNCTION_ENTRY_ARCHIVED   | \-4    | Una entrada del libro mayor requerida para la huella de esta función está en un estado archivado, y debe ser restaurada.                          |
| HOST_FUNCTION_INSUFICIENT_REFUNDABLEE | \-5    | La comisión de Soroban reembolsable proporcionada no fue suficiente para pagar los recursos de cómputo requeridos por esta invocación de función. |
