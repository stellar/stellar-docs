La pila estelar está formada por los siguientes componentes: las redes (Mainnet, Testnet, y Futurenet), Stellar Core, Horizon API, RPC, y SDKs, cada una de las cuales desempeña un papel específico en la provisión de infraestructura financiera resistente a los fracasos, disponible para cualquier persona, y lo suficientemente rápido y barato como para servir casos de uso del mundo real.

![Stack estelar](/assets/stellar-tech-stack.png)

## Redes

Stellar tiene tres redes: la red pública (Mainnet, también llamada Pubnet o la red pública), la red de pruebas (Testnet), y una red de desarrollo (Futurenet). Mainnet es la red principal utilizada por las aplicaciones en producción. La red de pruebas es más pequeña, red libre de usar mantenida por SDF que funciona como el Mainnet pero no se conecta al dinero real y es el mejor lugar para que los desarrolladores prueben sus aplicaciones. Futurenet es una red de desarrollo que se puede utilizar para probar más características de la vanguardia sangrante.

Lea más sobre las diferentes redes en la [sección de redes](./networks.mdx).

## Núcleo estelar

Stellar Core es el programa utilizado por los nodos (o computadoras) individuales que componen la red. Stellar Core mantiene un libro de contabilidad distribuido y se compromete al consenso para validar y procesar las transacciones. Generalmente, los nodos alcanzan el consenso, aplican un conjunto de transacciones y actualizan el libro principal cada 5-7 segundos.

Los nodos alcanzan un consenso usando el Protocolo del Consenso Estelar, que puede obtener más información aquí: [Consenso Stellar Protocol](./stellar-consensus-protocol.mdx)

Cualquiera puede ejecutar un nodo de Núcleo Stellar, pero no tienes que construir sobre Stellar. Le recomendamos que lo haga si emite un activo y quiere asegurar la exactitud del contador, si desea participar en la gobernanza de la red votando sobre la versión del protocolo, tarifas mínimas, y límites de recursos y mayores, y/o si desea contribuir a la salud y descentralización general de Stellar. Mira nuestro tutorial sobre instalar, configurar y mantener tu propio nodo aquí: [Ejecutar un Tutorial de Nodo Validator](../../validators/README.mdx).

## Horizón

Horizon es el servidor de API HTTP RESTful orientado al cliente en la capa de plataforma que permite el acceso programático para enviar transacciones y consultar los datos históricos de la red. Actúa como interfaz para las aplicaciones que quieran acceder a la red Stellar. Puede comunicarse con Horizon usando un SDK, un navegador web, o con simples herramientas de comando como cURL.

No necesita ejecutar su propia instancia Horizon — cuando está comenzando, puede utilizar la instancia gratuita SDF Horizon para acceder a la red — pero se recomienda que lo haga cuando esté listo para lanzar un producto terminado. Échale un vistazo a cómo hacerlo aquí: [Ejecutar Tutorial de Servicios de Plataforma](../../data/horizon/admin-guide/README.mdx)

Aprende todo lo que hay que saber sobre el uso de Horizon en el Horizon [documentation](../../data/horizon/README.mdx).

## RPC

Soroban RPC es un servidor JSON RPC que proporciona una interfaz para que los usuarios y aplicaciones interactúen con contratos inteligentes en la cadena de bloques Stellar. Cuando una aplicación desea interactuar con contratos inteligentes, envía una solicitud al servidor RPC. El servidor interpreta estas peticiones, las traduce en un formato comprensible por los nodos blockchain, y las reenvia. Después de procesar las solicitudes, los nodos blockchain devuelven los resultados. El servidor RPC recibe estos resultados y los envía de vuelta a la aplicación solicitante.

SDF tiene puntos finales RPC disponibles para Futurenet y Testnet. Estos servicios son de uso libre y son aptos para el desarrollo y la prueba.

SDF no proporciona un punto final RPC disponible públicamente para Mainnet. Los desarrolladores deberían [seleccionar un proveedor de ecosistemas](../../data/rpc/rpc-providers.mdx) que funcione para su proyecto antes de migrar a Mainnet. En algunos casos, los proyectos pueden elegir ejecutar su propia instancia RPC.

## SDKs

Los SDKs simplifican parte del trabajo de acceso a Horizon y al Soroban RPC convirtiendo los datos en formatos más amigables y permitiéndole programar en el idioma de su elección. Los SDKs de Stellar le muestran cómo solicitar datos y crear y enviar transacciones. Los SDKs de Soroban le permiten escribir contratos inteligentes en Rust e interactuar con contratos inteligentes en una gran variedad de otros idiomas.

Ver la [biblioteca SDK](../../tools/sdks/library.mdx) de Stellar para acceder a nuestros SDKs y su documentación.
