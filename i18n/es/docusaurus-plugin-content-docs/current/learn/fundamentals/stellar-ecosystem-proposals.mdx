Cada SEP es un plano distinto destinado a ayudar a los usuarios a construir un producto o servicio que interactúa con otros productos y servicios en la red Stellar.

Cuando se construye en Stellar, generalmente se utiliza la API Horizon para interactuar con la red. Sin embargo, en cualquier momento usted desea que su producto o servicio interactúe con otros productos o servicios en el ecosistema, debe crear infraestructura adicional para manejar esos componentes de una interacción.

Los Everistas definen los estándares para construir esa infraestructura sobre la red Stellar. Están diseñados para ayudar a diferentes entidades, como emisores de activos, carteras, intercambios, y otros proveedores de servicios a interactuar utilizando una única integración común. Generalmente, definen dos lados de una interacción —a menudo un lado del servidor y un lado del cliente— y utilizarlos como un plano le permite conectarse a varias contrapartes sin comenzar desde cero cada vez.

Los documentos de código abierto son creados públicamente y de código abierto que viven en el [repositorio de GitHub](https://github.com/stellar/stellar-protocol/tree/master/ecosystem) y tienen un proceso de aprobación ligero. Nuevos Etiquetas y actualizaciones se discuten constantemente. Fomentamos la participación en estos debates para ayudar a crear nuevos estándares y hacer que los servicios Stellar sean más accesibles.

## Notables Etiquetas

Existen muchos SEP, que cubren una amplia variedad de normas para la interoperación. Independientemente de lo que esté construyendo, puede que quiera echar un vistazo a la lista completa para ver si hay un estándar para su caso de uso.

Aquí, cubriremos unos pocos aspectos notables que definen los estándares para los casos de uso estelar más comunes.

### SEP-0001 - Archivo de información estelar

Define cómo crear y alojar un archivo stellar.toml: un lugar común donde Internet puede encontrar información sobre su integración estelar. Usted puede almacenar una gran cantidad de información en su archivo stellar.toml incluyendo información de organización, información de moneda e información de contacto. TOML es un formato de archivo de configuración simple y comúnmente utilizado diseñado para ser legible tanto por humanos como por máquinas.

Usando la operación `set_options`, puedes vincular tu cuenta Stellar al dominio que aloja tu estelar. oml, creando una conexión en cadena entre esta información y esa cuenta.

**Utilizado por anclas, emisores y validadores.**

[Link to GitHub](https://github.com/stellar/stellar-protocol/blob/master/ecosystem/sep-0001.md)

### SEP-0005 - Métodos de derivación clave para cuentas estelares

Describe métodos para la derivación de claves para Stellar, mejorando el almacenamiento de claves y moviendo las claves entre carteras y aplicaciones. La guía en este SEP mejora el ecosistema estelar por:

- Hacer que la derivación de clave sea la misma entre carteras y aplicaciones
- Permitir a los usuarios mantener las claves en las carteras de hardware
- Permitir a los usuarios mantener las teclas en el almacenamiento en frío de forma más fiable (usando códigos mnemónicos)
- Permitir a los usuarios generar múltiples claves de una sola semilla (por ejemplo, primero para almacenar fondos y segundo como firmante para una cuenta compartida)

**Utilizado por carteras y otras aplicaciones.**

[Link to GitHub](https://github.com/stellar/stellar-protocol/blob/master/ecosystem/sep-0005.md)

### SEP-0006 - API de Depositos y Retiro

Define la forma estándar para que los anclajes y carteras interactúen en nombre de los usuarios. Con la guía de este SEP, las carteras y otros clientes pueden interactuar directamente con los anclajes sin que el usuario tenga que dejar la cartera para ir al sitio del ancla.

Este SEP define un protocolo estándar que permite las siguientes características dentro de una cartera u otro cliente Stellar:

- Deposita activos externos con un ancla
- Retire activos de un ancla
- Ejecutar depósito/retiro entre activos no equivalentes
- Cuota de depósito y retiro de la estructura para un anclaje al usuario
- Manejar las necesidades de KYC, incluyendo la transmisión de información de KYC sobre el usuario al ancla a través de SEP-12
- Compruebe el estado de los depósitos o retiros en curso que involucren al usuario
- Ver historial de depósitos y retiros involucrando al usuario

SEP-0024 es la alternativa a SEP-0006 que soporta depósitos alojados y retiros.

**Utilizado por anclajes, carteras y otras aplicaciones.**

[Link to GitHub](https://github.com/stellar/stellar-protocol/blob/master/ecosystem/sep-0006.md)

### SEP-0007 - Esquema URI para facilitar la firma Delegada

Define el esquema URI estándar que puede utilizarse para generar una URI que servirá como una solicitud para firmar una transacción. Con la guía de este SEP, las aplicaciones que no son de cartera pueden hacer que sus usuarios firmen una transacción sin ver la clave secreta del usuario de la cartera de ningún modo, ya que la URI (petición) típicamente será firmada por la cartera de confianza del usuario donde se almacenan las claves secretas.

Este SEP define un protocolo estándar que permite las siguientes características dentro de una cartera u otro cliente Stellar:

- Deeplinks pagos (en línea)
- Pagos con código QR (online y offline)
- Transacciones de punto de venta (sin conexión)
- Pagos entre pares (online y offline)

**Utilizado por carteras y otras aplicaciones.**

[Link to GitHub](https://github.com/stellar/stellar-protocol/blob/master/ecosystem/sep-0007.md)

### SEP-0010 - Autenticación estelar

Define una forma estándar para que los clientes (tales como carteras o intercambios) creen sesiones web autenticadas para los usuarios que posean una cuenta Stellar. Este SEP también soporta la autenticación de usuarios de cuentas compartidas o agrupadas de Stellar. Los clientes pueden usar cuentas silenciadas para distinguir usuarios o subcuentas de cuentas compartidas.

Demuestra que el usuario tiene una cuenta Stellar y que controla la cuenta con una única clave maestra o con los firmantes suficientes necesarios.

**Utilizado por carteras e intercambios.**

[Link to GitHub](https://github.com/stellar/stellar-protocol/blob/master/ecosystem/sep-0010.md)

### SEP-0012 - API de KYC

Permite compartir los datos de KYC y define una forma estándar para que los clientes Stellar suban KYC y otra información a los anclajes y otros servicios. Este SEP se hizo teniendo en cuenta estos objetivos:

- Permitir a un cliente introducir su información KYC en su cartera una vez y usarla a través de muchos servicios sin volver a introducir información manualmente
- Manejar imagen y datos binarios
- Soporta el conjunto de campos definidos en SEP-9
- Autenticación de soporte vía SEP-10
- Apoyar la provisión de datos para SEP-6, SEP-24, SEP-31, y otros
- Dar a los clientes control sobre sus datos apoyando el borrado completo de datos

**Utilizado por anclajes, carteras y otras aplicaciones.**

[Link to GitHub](https://github.com/stellar/stellar-protocol/blob/master/ecosystem/sep-0012.md)

### SEP-0020 - Autoverificación de nodos validadores

Define cómo los validadores autoverifican estableciendo el dominio principal de su cuenta Stellar en su sitio web, donde publican información en cadena sobre su nodo y organización en un estelar. archivo oml. Esto permite a otros participantes descubrir otros nodos y agregarlos a sus conjuntos de quórum sin necesidad de una base de datos centralizada.

**Utilizado por validadores.**

[Link to GitHub](https://github.com/stellar/stellar-protocol/blob/master/ecosystem/sep-0020.md)

### SEP-0024 - Depósito y retirada alojados

Define la forma estándar para que los anclajes y carteras interactúen en nombre de los usuarios de forma interactiva. Esto significa que la aplicación del usuario debe abrir una vista web alojada por un anclaje de terceros para que el usuario proporcione la información necesaria para completar la transacción.

Los usuarios utilizan aplicaciones que implementan SEP-0024 para conectarse a empresas que aceptarán valor fuera de la cadena (como USD) a cambio de valor en cadena (como USDC) y viceversa.

SEP-0006 es la alternativa a SEP-0024 que soporta una solución tipo API para el mismo caso de uso.

**Utilizado por anclajes, carteras y otras aplicaciones.**

[Link to GitHub](https://github.com/stellar/stellar-protocol/blob/master/ecosystem/sep-0024.md)

### SEP-0030 - Recuperación de Cuentas: Recuperación Multi-Demo de Cuentas Estelares

Define la API estándar que habilita a un individuo (p.ej. un usuario o billetera) para recuperar el acceso a una cuenta Stellar que posee después de que el individuo haya perdido su clave privada sin proporcionar ningún control de terceros sobre la cuenta. Utilizando este protocolo, el usuario o la cartera anunciará la cuenta y un número de teléfono, correo electrónico, u otra forma de autenticación con uno o más servidores que implementan el protocolo y añaden esos servidores como firmantes de la cuenta. Si se utilizan dos o más servidores con la configuración apropiada del firmante, ningún servidor individual tendrá el control de la cuenta, pero de forma colectiva, pueden ayudar al individuo a recuperar el acceso a la cuenta.

El protocolo también permite a los individuos pasar el control de una cuenta Stellar a otro individuo.

Este SEP permite los siguientes casos de uso para un usuario:

- Recuperar: Recuperar el acceso a las cuentas de Stellar para las que pueden haber perdido claves.
- Compartir: Obtener acceso a cuentas Stellar que otro usuario pretende compartir con ellos.

**Utilizado por carteras y otras aplicaciones.**

[Link to GitHub](https://github.com/stellar/stellar-protocol/blob/master/ecosystem/sep-0030.md)

### SEP-0031 - API de pago transfronterizo

Define el protocolo para dos cuentas financieras que existen fuera de la red Stellar (ancestros) para interactuar entre sí.

**Utilizado por anclas.**

[Link to GitHub](https://github.com/stellar/stellar-protocol/blob/master/ecosystem/sep-0031.md)
