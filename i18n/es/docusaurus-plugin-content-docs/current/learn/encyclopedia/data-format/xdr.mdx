---
title: XDR
---

Stellar almacena y comunica datos contables, transacciones, resultados, historial y mensajes en un formato binario llamado Representación de Datos Externos (XDR). XDR está optimizado para el rendimiento de la red pero no se puede leer en humanos. Horizon y los SDKs Stellar convierten XDRs en formatos más amigables.

XDR se especifica en [RFC 4506](http://tools.ietf.org/html/rfc4506.html) y es similar a herramientas como Buffers de Protocolo o Thrift. XDR proporciona algunas características importantes:

- Es muy compacto, por lo que puede ser transmitido rápidamente y almacenado con un mínimo de espacio en disco.
- Los datos codificados en XDR se almacenan de forma fiable y previsible. Los campos están siempre en el mismo orden, lo que simplifica la firma criptográfica y la verificación de mensajes XDR.
- Las definiciones XDR incluyen ricas descripciones de tipos de datos y estructuras, que no es posible en formatos más sencillos como JSON, TOML o YAML.

## Parsing XDR

Puesto que XDR es un formato binario y no tan conocido como formatos más simples como JSON, todos los SDK Stellar incluyen herramientas para analizar XDR y lo harán automáticamente al recuperar datos.

Además, el servidor Horizon API generalmente expone las partes más importantes de los datos XDR en JSON, para que sean más fáciles de analizar si no está usando un SDK. Los datos XDR todavía están incluidos (codificados como una cadena base64) dentro del JSON en caso de que necesite acceso directo a él.

## Archivos .X

Las estructuras de datos en XDR se especifican en un archivo de definición de interfaz (IDL). Los archivos IDL utilizados para la Red Stellar están disponibles en [GitHub](https://github.com/stellar/stellar-core/tree/master/src/protocol-curr/xdr).
