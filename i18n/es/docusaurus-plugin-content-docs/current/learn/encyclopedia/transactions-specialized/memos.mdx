Los memorandos son un campo de datos opcional y no estructurado que se puede utilizar para incrustar cualquier información identificativa adicional sobre la transacción relevante para el remitente o el receptor.

Anteriormente se utilizaban para diferenciar entre cuentas individuales en una cuenta agrupada, algo para lo que ahora usamos cuentas multiplexadas.

Para más información sobre cuentas multiplexadas, consulta nuestra [Entrada de enciclopedia de Cuentas Agrupadas - Cuentas Multiplexadas y Memorandos](./pooled-accounts-muxed-accounts-memos.mdx)

Los memorandos pueden ser uno de los siguientes tipos:

`MEMO_TEXT`: Una cadena codificada utilizando ASCII o UTF-8, de hasta 28 bytes de longitud.\
`MEMO_ID`: Un entero sin signo de 64 bits.\
`MEMO_HASH`: Un hash de 32 bytes.\
`MEMO_RETURN`: Un hash de 32 bytes destinado a ser interpretado como el hash de la transacción que el remitente está reembolsando.

## Ejemplos de contenido de memorandos

- Notificando que la transacción es un reembolso o reintegro
- Referencia a una factura que la transacción está pagando
- Cualquier información adicional de enrutamiento interno
- Enlaces a datos relevantes
