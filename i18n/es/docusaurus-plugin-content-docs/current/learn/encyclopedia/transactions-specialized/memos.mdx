Los Memos son un campo opcional de datos no estructurados que puede ser utilizado para incrustar cualquier información adicional de identificación sobre la transacción relevante para el remitente o el receptor.

Anteriormente estaban acostumbrados a diferenciarse entre cuentas individuales en una cuenta combinada, algo que usamos cuentas silenciadas por ahora.

Para más información sobre cuentas muxadas, consulta [Cuentas agrupadas - Cuentas retiradas y Memos Encyclopedia Entry](./pooled-accounts-muxed-accounts-memos.mdx)

Los Memos pueden ser uno de los siguientes tipos:

`MEMO_TEXT`: Una cadena codificada usando ASCII o UTF-8, hasta 28 bytes de largo.\
`MEMO_ID`: Un entero sin signo de 64 bits.\
`MEMO_HASH`: Un hash de 32 bytes.\
`MEMO_RETURN`: Un hash de 32 bytes destinado a ser interpretado como el hash de la transacción que el remitente está reembolsando.

## Ejemplos de contenido de memo

- Notificar que la transacción es un reembolso o reembolso
- Referencia a una factura que la transacción está pagando
- Cualquier otra información de ruta interna
- Enlaces a datos relevantes
