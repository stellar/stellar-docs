title: Contrato cruzado
description: Interactuar con un contrato de Soroban dentro de otro contrato de Soroban.
---

Una invocación a través del contrato es un poderoso tipo de interacción (aunque cara) de contrato. Una invocación del contrato es similar a iniciar un nuevo proceso porque el código que se ejecuta estará en un espacio de direcciones separado. lo que significa que no comparten ningún dato distinto de lo que se pasó en la invocación. Mientras que una invocación del contrato normalmente transfiere el control a un contrato _diferente_, es posible transferir el control al contrato en ejecución. Independientemente de si el contrato que recibe control es un contrato diferente o el contrato en curso, el valor devuelto por `get_invoking_contract` será el valor anterior de `get_current_contract`. Una invocación del contrato sólo puede acceder a los métodos públicos de un contrato.

Si un contrato contiene una función pública `f`, entonces invocar `f` puede hacerse haciendo una llamada a la función Rust a `f::invoke`.

Algunos contratos, como el contrato token, sólo exportan las funciones de invocación del contrato. Al hacerlo, son capaces de asignar esos nombres amigables de las funciones. Por ejemplo, [initialize](https://github.com/stellar/soroban-token-contract/blob/42380647bb817bf01c739c19286f18be881e0e41/src/contract.rs#L55-L57)

```rust
#[contractimpl(export_if = "export")]
impl TokenTrait for Token {
    fn initialize(e: Env, admin: Identifier, decimal: u32, name: Binary, symbol: Binary) {
```

is [exported](https://github.com/stellar/soroban-token-contract/blob/42380647bb817bf01c739c19286f18be881e0e41/src/lib.rs#L26) as

```rust
pub use crate::contract::initialize::invoke as initialize;
```

Esta función es entonces fácilmente [llamada por el contrato de la piscina de liquidez](https://github.com/stellar/soroban-examples/blob/4060d3bd5ee7846020b68ee583665a4d4cf4b315/liquidity_pool/src/lib.rs#L164-L171).
