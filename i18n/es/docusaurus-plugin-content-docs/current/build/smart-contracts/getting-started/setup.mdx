Los contratos inteligentes estelares son pequeños programas escritos en el lenguaje de programación [Rust].

Para construir y desarrollar contratos sólo necesita un par de requisitos:

- A [Rust] toolchain
- Un editor que soporta Rust
- [CLI Estelar]

## Instalar Rust

### Linux, macOS u otro sistema operativo similar a Unix

Si usa macOS, Linux u otro sistema operativo tipo Unix, el método más simple para instalar una toolchain de Rust es instalar `rustup`. Instala `rustup` con el siguiente comando.

```sh
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

### Ventanas

En Windows, descarga y ejecuta [rustup-init.exe](https://static.rust-lang.org/rustup/dist/i686-pc-windows-gnu/rustup-init.exe). Puede continuar con la configuración predeterminada pulsando Enter.

:::tip

El CLI estelar utiliza emojis en su salida. Para renderizarlos correctamente en Windows, se recomienda usar el [Terminal de Windows](https://learn.microsoft.com/en-us/windows/terminal/). Mira [cómo instalar Windows Terminal](https://learn.microsoft.com/en-us/windows/terminal/install) en Microsoft Learn. Si el CLI se utiliza en el Command Prompt de Windows o Windows PowerShell el CLI funcionará como se espera, pero los emojis aparecerán como signos de interrogación.

:::

Si ya estás usando [WSL](https://learn.microsoft.com/en-us/windows/wsl/install), también puedes seguir las instrucciones para Linux.

### Otro

Para otros métodos de instalación de [Rust], vea: https://www.rust-lang.org/tools/install

## Instalar el destino

Instala el objetivo `wasm32-unknown-unknown`.

```sh
objetivo de rustup añadir wasm32-unknown-unknown
```

## Instalar la CLI Estelar

El [Stellar CLI](https://github.com/stellar/stellar-cli) puede ejecutar contratos inteligentes en futurenet, testnet, mainnet, así como en un sandbox local.

### Instalar

Hay algunas maneras de instalar la [última versión lanzada](https://github.com/stellar/stellar-cli/releases) de Stellar CLI.

Instalar con carga desde el código fuente:

```sh
carga install --locked stellar-cli --features opt
```

Instala con `cargo-binstall`:

```sh
cargo install --locked cargo-binstall
cargo binstall -y stellar-cli
```

Instalar con Homebrew:

```sh
brew install stellar/tap/stellar-cli
```

:::info

Informar problemas y compartir comentarios sobre el CLI Stellar [here](https://github.com/stellar/stellar-cli/issues/new/choose).

:::

### Documentación

La documentación de referencia completa generada automáticamente está disponible [here](../../../tools/stellar-cli.mdx).

### Autocompletado

Puedes usar `stellar completation` para generar terminación de conchas para `bash`, `elvish`, `fish`, `powershell`, y `zsh`. Usted debería probarlo de forma absoluta. ¡Se sentirá como un súper poder!

Para habilitar el autocompletado en el shell actual de bash, ejecute:

```sh
origen <(finalización estelar --shell bash)
```

Para activar el autocompletado permanentemente en futuras shells de bash, ejecute:

```sh
echo "source <(stellar completion --shell bash)" >> ~/.bashrc
```

Los usuarios de shells que no sean de bash pueden necesitar adaptar los comandos anteriores a sus necesidades.

### Configurar el CLI para Testnet

Stellar tiene una red de pruebas llamada Testnet que puede utilizar para desplegar y probar sus contratos inteligentes. Es una red en vivo, pero no es lo mismo que la red pública Stellar. Es una red separada que se usa para el desarrollo y la prueba, por lo que no se puede usar para aplicaciones en producción. Pero es un buen lugar para probar sus contratos antes de desplegarlos en la red pública.

Para configurar su CLI para interactuar con Testnet, ejecute el siguiente comando:

<Tabs groupId="platform" defaultValue={getPlatform()}>

<TabItem value="unix" label="macOS/Linux">

```sh
red estelar añadir \
  --global testnet \
  --rpc-url https://soroban-testnet.stellar.org:443 \
  --network-passphrase "Test SDF Network ; septiembre 2015"
```

</TabItem>

<TabItem value="windows" label="Windows (PowerShell)">

```powershell
red estelar añadir `
  --global testnet `
  --rpc-url https://soroban-testnet.stellar.org:443 `
  --network-passphrase "Test SDF Network ; septiembre 2015"
```

</TabItem>

</Tabs>

Ten en cuenta la bandera `--global`. Esto crea un archivo en el archivo `~/.config/soroban/network/testnet.toml` de tu carpeta principal con la configuración especificada. Esto significa que puede usar la bandera `--network testnet` en cualquier comando Stellar CLI para usar esta red desde cualquier directorio o ruta de archivo en su sistema.

Si quieres configuraciones de red específicas del proyecto, puedes omitir la bandera `--global`, y las redes se añadirán a la carpeta `.soroban/network` de tu directorio de trabajo.

### Configurar una identidad

Cuando despliega un contrato inteligente en una red, necesita especificar una identidad que se utilizará para firmar las transacciones.

Vamos a configurar una identidad llamada `alice`. Puedes usar cualquier nombre que quieras, pero puede ser bueno tener algunas identidades nombradas que puedes usar para pruebas, tales como [`alice`, `bob`, y `carol`](https://en. ikipedia.org/wiki/Alice_y_Bob).

```sh
las claves estelares generan --global alice --network testnet
```

Puedes ver la clave pública de `alice` con:

```sh
dirección de llaves estelares
```

Como la red configura, el `--global` significa que la identidad se almacena en `~/.config/soroban/identity/alice.toml`. Puedes omitir la bandera `--global` para almacenar la identidad en la carpeta `.soroban/identity` de tu proyecto en su lugar.

Por defecto, `stellar keys generate` financiará la cuenta usando [Friendbot](../../learn/fundamentals/networks.mdx#friendbot). Para desactivar este comportamiento, agregue `--no-fund` al comando cuando lo ejecute.
