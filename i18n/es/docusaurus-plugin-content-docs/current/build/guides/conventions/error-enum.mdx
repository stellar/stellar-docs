---
title: Organizar errores del contrato con un tipo de error de número
hide_table_of_contents: true
description: Gestionar y comunicar errores del contrato usando una estructura enum almacenada como valores de Estado.
---

Una forma conveniente de gestionar y comunicar de forma significativa los errores del contrato es recopilarlos en una estructura 'enum'. Estos errores son un tipo especial de entero de enum que se almacenan en el libro como valores de estado que contienen un código `u32`. Primero, crea la estructura `Error` en tu contrato inteligente.

```rust
#[contracterror]
#[derive(Copiar, Clonar, Depurar, Eq, ParcialEq, ParcialOrd, Ord)]
#[repr(u32)]
pub esum Error {
    FirstError = 1,
    FootherError = 2,
    Aaí aún.Error = 3,
    Error genérico = 4
}
```

Luego, con un error cuando se cumplen las condiciones. Este ejemplo se mostrará con el error especificado.

```rust
#[contractimpl]
impl Contract {
    pub fn causeerror(env: Env, error_code: u32) -> Result<(), Error> {
        match error_code {
            1 => Err(Error::FirstError),
            2 => Err(Error::AnotherError),
            3 => Err(Error::YetAnotherError),
            _ => Err(Error::GenericError),
        }
    }
}
```

Cuando se convierte a XDR, el valor se convierte en un `ScVal`, que contiene un `ScStatus`, que contiene el valor entero del error como error de contrato.

```json
{ "status": { "contractError": 1 }
```
