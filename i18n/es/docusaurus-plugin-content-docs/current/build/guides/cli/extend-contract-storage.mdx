---
title: Extender la entrada de almacenamiento TTL de un contrato desplegado
hide_table_of_contents: true
description: Utilice el CLI para extender el tiempo de vida (TTL) de la entrada de almacenamiento persistente de un contrato.
---

Puede utilizar el Stellar CLI para extender la TTL de la entrada de almacenamiento persistente de un contrato. Para una entrada de almacenamiento que utiliza un simple `Symbol` como clave de almacenamiento, puedes ejecutar un comando así:

```bash
contrato estelar extender \
    --source S... \
    --network testnet \
    --id C... \
    --key COUNTER \
    --ledgers-to extend 535679 \
    --durabilidad persistente
```

Este ejemplo utiliza 535,679 ledgers como el nuevo archivo TTL. Este es el valor máximo permitido para este argumento en la CLI. Esto corresponde a aproximadamente 30 días (promedio de 5 segundos de tiempo de cierre del contable).

Si su entrada de almacenamiento usa una clave de almacenamiento más avanzada, como `Balance(Address)` en un contrato de tokens, necesitarás proporcionar la clave en un formulario XDR codificado en base64:

```bash
contrato estelar extender \
    --source S... \
    --network testnet \
    --id C... \
    --key-xdr AAABgAAAAHXkotywnA8z+r365/0701QSlWouXn8m0UOoshCtNHOYQAAAA4AAAAAAAHQmFsYW5jZQAAAAAB \
    --ledgers-para extender 535679 \
    --durabilidad persistente
```

:::info

Asegúrate de revisar nuestra [guía sobre la creación de claves de ledger XDR](../rpc/generate-ledger-keys-python.mdx) para ayudarte a generarlas.

:::
