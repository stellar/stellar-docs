Puedes usar el Stellar CLI para extender el TTL de una instancia de contrato así:

```bash
stellar contract extend \
    --source S... \
    --network testnet \
    --id C... \
    --ledgers-to-extend 535679 \
    --durability persistent
```

Este ejemplo utiliza 535.679 ledgers como el nuevo TTL de archivo. Este es el valor máximo permitido para este argumento en el CLI. Esto corresponde a aproximadamente 30 días (promediando tiempos de cierre de ledger de 5 segundos).

Cuando extiendes una instancia de contrato, esto incluye:

- la instancia de contrato en sí
- cualquier entrada `env.storage().instance()` en el contrato
- el código Wasm del contrato
