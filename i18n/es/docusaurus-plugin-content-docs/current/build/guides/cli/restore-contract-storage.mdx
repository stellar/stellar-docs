Si la entrada de almacenamiento persistente de un contrato ha sido archivada, puedes restaurarla utilizando el CLI de Stellar. Para una entrada de almacenamiento que utiliza un `Symbol` simple como su clave de almacenamiento, puedes ejecutar un comando así:

```bash
stellar contract restore \
    --source S... \
    --network testnet \
    --id C... \
    --key COUNTER \
    --durability persistent
```

Si tu entrada de almacenamiento utiliza una clave de almacenamiento más avanzada, como `Balance(Address)` en un contrato de token, necesitarás proporcionar la clave en un formato XDR codificado en base64:

```bash
stellar contract restore \
    --source S... \
    --network testnet \
    --id C... \
    --key-xdr AAAABgAAAAHXkotywnA8z+r365/0701QSlWouXn8m0UOoshCtNHOYQAAAA4AAAAHQmFsYW5jZQAAAAAB \
    --durability persistent
```

:::info

Asegúrate de consultar nuestra [guía sobre cómo crear claves de ledger en XDR](../rpc/generate-ledger-keys-python.mdx) para obtener ayuda para generarlas.

:::
