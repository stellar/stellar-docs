:::caution

Las carteras inteligentes en Stellar están actualmente en desarrollo activo. Esta página es donde se captura esa obra. Hasta que no haya una interfaz auditada se sugiere no utilizar carteras inteligentes en producción.

:::

El lanzamiento del Protocolo 21 permite el soporte **nativo** de verificación secp256r1 en contratos inteligentes estelares en Mainnet como se describe en [CAP-0051](https://github.com/stellar/stellar-protocol/blob/master/core/cap-0051.md). Esta integración significa que secp256r1 se construye directamente en el protocolo y las aplicaciones pueden firmar transacciones y enviarlas a la red usando secp256r1 sin depender de un tercero. como tienes que hacer en otras redes.

El soporte Secp256r1, a su vez, permite algo extremadamente emocionante: ¡monederos inteligentes alimentados con contraseña! La capacidad de firmar transacciones y acceder a cuentas con una clave de acceso en lugar de usar claves secretas o frases de semilla es un **abrazo** paso adelante para facilitar la usabilidad en blockchain sin sacrificar la seguridad.

Esta página está pensada para: 1. exhibir todo el trabajo que se está realizando en Stellar con contraseñas, y 2. le dan una comprensión básica de monederos inteligentes, la curva criptográfica secp256r1, el estándar WebAuthn, y contraseñas.

:::note

Esta página pretende mantener a los usuarios actualizados sobre el trabajo realizado con las claves de acceso por la Fundación para el Desarrollo de Stellar (SDF) y el ecosistema Stellar. La documentación oficial de las claves de acceso es un trabajo en curso mientras colaboramos para crear una interfaz auditada del contrato de cartera inteligente.

:::

## ¿Qué significa todo esto?

### Seca256r1

El [esquema de firma secp256r1](https://wiki.hyperledger. rg/display/BESU/SECP256R1+Support) es una curva elíptica que a menudo se utiliza con el algoritmo de firma digital de la curva elíptica (ECDSA), que es un esquema de firma de clave pública ampliamente utilizado.

Secp256r1 es un algoritmo de firmas común utilizado en WebAuthn, que es el estándar detrás de las contraseñas disponibles en navegadores, computadoras y teléfonos. Habilitar la verificación secp256r1 permite a los desarrolladores diseñar contratos que incorporen contraseñas para firmar transacciones de contratos inteligentes y cuentas de acceso en lugar de usar frases de semilla o claves de firma.

La blockchain estelar utiliza nativamente la curva elíptica Ed25519, que es extremadamente rápido y seguro, pero no es tan ampliamente soportado en implementaciones de WebAuthn como secp256r1.

### WebAuthn

[WebAuthn (autenticación web)](https://www.okta.com/blog/2019/03/what-is-webauthn/) es un estándar web para autenticación segura sin contraseña. Utiliza criptografía de clave pública para eliminar la dependencia de claves secretas y mejora la seguridad y usabilidad de dapps.

WebAuthn proporciona autenticación descentralizada (sin contraseñas de autoridad central), seguridad mejorada (las claves secretas permanecen en el dispositivo del usuario), mejor experiencia de usuario (los usuarios no tienen que preocuparse por recordar sus claves secretas), y una interoperabilidad más amplia (WebAuthn ya está soportada por los principales navegadores y plataformas).

### Claves de acceso

Passkeys son una implementación del estándar WebAuthn. La capacidad de usar contraseñas para firmar transacciones y acceder a cuentas elimina la necesidad de que los usuarios recuerden sus claves secretas o frases de semilla de 12 a 24 palabras, que ha sido una barrera masiva para la entrada de usuarios finales que entran en el espacio de la cadena de bloques. Las llaves secretas y las frases de semilla pueden ser reparadoras, difíciles de recordar, introducidas incorrectamente y propensas a violaciones de la seguridad.

Las claves de acceso ofrecen un más rápido, método más seguro de autenticación de identidad mediante el uso de datos cifrados almacenados en un dispositivo y la verificación del usuario con tokens de hardware (como YubiKeys), datos biométricos (como huellas dactilares o reconocimiento facial), u otros métodos criptográficos.

### Carteras inteligentes

Las carteras inteligentes son carteras digitales que aprovechan la composición inteligente del contrato para ofrecer una funcionalidad y seguridad mejoradas para la gestión de activos digitales. A diferencia de las carteras de criptomonedas tradicionales, las carteras inteligentes integran características como soporte multifirma, transacciones programables y compatibilidad mejorada con la experiencia del usuario con aplicaciones descentralizadas (dApps).

Las billeteras inteligentes permiten a los usuarios interactuar sin problemas con los ecosistemas de blockchain, proporcionando una interfaz fácil de usar para enviar, recibir y almacenar activos digitales. A menudo incorporan contraseñas y estándares WebAuthn para asegurar la autentificación sin contraseña, reduciendo los riesgos asociados con las frases tradicionales de semillas y las claves privadas.

Mediante la utilización de contratos inteligentes, las billeteras inteligentes pueden automatizar transacciones e implementar protocolos de seguridad avanzados, como transferencias bloqueadas por tiempo, límites de gasto y detección de fraude. Estas características hacen de las carteras inteligentes una elección ideal para los usuarios que buscan un seguro, conveniente, y solución versátil para la gestión de sus activos digitales en el paisaje evolucionante de la tecnología blockchain.

### Cómo trabajan juntos

Secp256r1 proporciona la base criptográfica para la generación de claves y firmas digitales, WebAuthn ofrece un marco estandarizado para la autenticación sin contraseña utilizando criptografía de clave pública, y las contraseñas implementan estas tecnologías para proporcionar una experiencia de usuario segura y sin fisuras. ¡Agrupa todo esto juntos en un contrato inteligente estelar y tienes los cimientos para el mejor desenlace del monedero inteligente del planeta!

**Generación y almacenamiento de llaves:**

- Cuando un usuario configura una clave de acceso o se registra con un servicio WebAuthn, se genera un par de claves usando una curva elíptica soportada como secp256r1.
- La clave secreta se almacena de forma segura en el dispositivo del usuario, mientras que la clave pública se envía al proveedor de servicios.

**Autenticación:**

- Para la autenticación, el proveedor de servicios envía un desafío al dispositivo del usuario.
- El dispositivo firma el reto con la clave secreta.
- El proveedor de servicios verifica la firma usando la clave pública.

**Experiencia de usuario:**

- Las claves de acceso simplifican este proceso para los usuarios, permitiéndoles autenticarse usando datos biométricos en lugar de recordar su clave secreta.
- WebAuthn proporciona el marco estándar para estas interacciones, garantizando seguridad e interoperabilidad entre plataformas y dispositivos.
