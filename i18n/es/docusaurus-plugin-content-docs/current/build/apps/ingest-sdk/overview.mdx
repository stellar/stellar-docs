Este tutorial recorre cómo una aplicación puede aprovechar los patrones comunes de datos de streaming para ingerir datos de transacciones de red estelares usando algunos paquetes seleccionados del Stellar Go Repo [github. om/stellar/go](https://github.com/stellar/go/blob/master/) colectivamente conocido como el SDK 'Ingestion':

## Los paquetes SDK de Ingestion

- `github.com/stellar/go/amount` paquete de utilidad para convertir precios de operaciones de transacción de red a cadena
- `github.com/stellar/go/historyarchive` `github.com/stellar/go/support/datastore` `github.com/stellar/go/support/storage` paquete de utilidad con cómodas envolturas para acceder a archivos de historia, y evitar aspectos http de bajo nivel
- `github.com/stellar/go/ingest` proporciona funcionalidad de análisis sobre los datos del editor de red, convierte en modelo `LedgerTransaction` más centrado en el desarrollador
- `github.com/stellar/go/ingest/ledgerbackend` proporciona la implementación del núcleo cautivo backend
- `github.com/stellar/go/network` proporciona convenientes configuraciones preconfiguradas para redes Testnet y Mainnet
- `github.com/stellar/go/xdr` un completo enlace de Golang al modelo de datos de red estelar

## Configuración del proyecto Ingestion

### Requisitos del proyecto

Para construir un ejemplo de canalización de la ingestión de red a partir de datos de transacción de red Stellar, necesitarás:

- Una estación de trabajo de desarrollador con [Go](https://go.dev/learn/) instalado el tiempo de ejecución del lenguaje de programación
- Un IDE para editar el código Go [VSCode](https://code.visualstudio.com/download) es bueno si se necesita uno
- Una carpeta de proyecto Go recién inicializada y vacía. `mkdir pipeline; cd pipeline; go mod init example/pipeline`
- `stellar-core` debe ser [installed](../../../validators/admin-guide/installation.mdx) en tu estación de trabajo y disponible en tu PATH o/s

El [modelo de datos de red Stellar](../../learn/fundamentals/stellar-data-structures/README.mdx) está definido en un formato IDL expresado en [XDR encoding](https://github.com/stellar/stellar-xdr). Nuestra aplicación de ejemplo sólo está interesada en un pequeño subconjunto del modelo global de datos de transacción relacionado con la compra y venta de activos, i. . un pago, y define su propio modelo de datos internamente:

<CodeExample>

```
::AppPayment
   Timestamp: uint
   BuyerAccountId: string
   SellerAccountId: string
   AssetCode: string
   Amount: string
}
```

</CodeExample>

La aplicación de ejemplo ejecutará un [pipeline de ingestión de red](https://github.com/stellar/go/blob/master/ingest/doc.go) para derivar un modelo de `ApplicationPayment` más pequeño del [modelo de datos de transacción de red Stellar](../../.. aprender/fundamentals/stellar-data-structures/README. dx) como 'fuente de origen' y así habilitar la aplicación para evitar grandes recursos de cómputo que habrían sido requeridos para mantener el almacenamiento completo del modelo de datos de red Stellar.

El pipeline de ingestión realizará tres roles distintos del procesador de secuencias:

### Capítulo Entrante

Actúa como la "fuente de origen" del tubo. Recupera [LedgerCloseMeta](https://github.com/stellar/go/blob/f30d11432e81c7a7cbb739a694520f729bbb31dd/xdr/xdr_generated.go#L18358) generados desde una red estelar usando núcleo cautivo. `LedgerCloseMeta` es la agregación de nivel superior en el modelo de datos estelares de los cuales todos los [datos de transacción de red estelar](../../../learn/fundamentals/stellar-data-structures/README.mdx) están anidados. Publica el `LedgerCloseMeta` en el pipeline.

### Transformador

Suscríbete para recibir `LedgerCloseMeta` desde el pipeline. Utiliza el paquete Go SDK [github.com/stellar/go/xdr](https://github. om/stellar/go/tree/master/xdr) para analizar el modelo de datos de red anidado para operaciones de pago y convertirlos en una nueva instancia de instancias del modelo de datos de aplicación 'ApplicationPayment'. Publica `ApplicationPayment` al pipeline.

### Capítulo Saliente

Actúa como la terminación del pipeline, se suscribe para recibir `ApplicationPayment` y publica los datos fuera del pipeline y a un almacén de datos externo, un ZeroMQ Publisher Socket, que es esencialmente un broker de mensajes.

### Summary

Consulte [Ingestion Pipeline Sample Application](./ingestion-pipeline-code. dx) para un código completo que demuestre el uso de los paquetes SDK 'ingestion' para crear estos adaptadores y transformers y ejecutar un pipeline en vivo contra la red Stellar.
