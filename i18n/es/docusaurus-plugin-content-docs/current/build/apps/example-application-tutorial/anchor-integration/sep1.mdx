El [archivo `stellar.toml`](../../../../tokens/publishing-asset-info.mdx#completing-your-stellartoml) es un lugar común donde Internet puede encontrar información sobre la integración estelar de una organización. Independientemente del tipo de transferencia que queramos utilizar (SEP-6 o SEP-24), necesitaremos comenzar con SEP-1.

Para los anclajes, estamos interesados en las palabras clave `CURRENCIES` que emiten, las `TRANSFER_SERVER` y/o `TRANSFER_SERVER_SEP0024` que indican si el anclaje soporta SEP-6, SEP-24, o ambos, y el `WEB_AUTH_ENDPOINT` que permite a una cartera configurar una sesión de usuario autenticada.

BasicPay está interoperando con el anclaje de pruebas ubicado en `testanchor.stellar.org` y puedes ver su archivo toml [here](https://testanchor.stellar.org/.well-known/stellar.toml).

```js title=/src/lib/stellar/sep1.js
import { StellarTomlResolver } from "stellar-sdk";

// Obtiene y devuelve el archivo stellar.toml alojado por un dominio proporcionado.
export async function fetchStellarToml(domain) {
  let stellarToml = await StellarTomlResolver.resolve(domain);
  return stellarToml;
}
```

**Fuente:** https://github.com/stellar/basic-payment-app/blob/main/src/lib/stellar/sep1.js

Estrictamente hablando la función `StellarTomlResolver` del SDK de JavaScript es la única función que _necesitamos_ para recuperar y usar la información proporcionada por el ancla (eck, podríamos incluso escribir nuestra propia función basada en «fetch», y eludir el SDK por completo). Sin embargo, hemos creado unas cuantas funciones "ayudantes" para hacer el resto de nuestras consultas un poco más detalladas y claras sobre lo que buscamos desde el servidor anchor. ¡Asegúrate de revisar el archivo fuente `sep1.js` enlazado arriba!

Usando la información `stellar.toml` para un activo con un `home_domain`, podemos mostrar al usuario algunas opciones (dependiendo de la infraestructura disponible). Empezaremos con la autenticación SEP-10.
