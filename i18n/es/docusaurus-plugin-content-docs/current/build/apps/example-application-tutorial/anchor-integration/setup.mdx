Un ancla es un término Stellar-específico para los ramps que conectan la red Stellar con los rieles financieros tradicionales, tales como entidades financieras o empresas de tecnología avanzada. Cuando un usuario deposita con un ancla, ese anclaje acreditará su cuenta Stellar con la cantidad equivalente de fichas digitales. El usuario puede entonces mantener, transferir o operar con esos tokens como cualquier otro activo Stellar. Cuando un usuario retira esas fichas, el ancla las canjea por dinero en efectivo en mano o dinero en el banco. Lee más sobre los anclajes en esta [sección de anclas](../../../../learn/fundamentals/anchors.mdx).

Cuando un cliente descarga una aplicación de cartera que está conectada a un servicio de fondeo, su cuenta Stellar puede ser creada por la aplicación de cartera o el servicio de fondo. En este ejemplo, la cuenta ha sido creada por la aplicación de la billetera, BasicPay. Las estrategias de creación de cuentas se describen más en profundidad [here](../../application-design-considerations.mdx#account-creation-strategies).

En este ejemplo, usaremos un ancla en la red Testnet de Stellar para simular una transferencia bancaria dentro y fuera de la billetera del usuario usando [SEP-6: Deposit and Withdrawal API](https://github. om/stellar/stellar-protocol/blob/master/ecosystem/sep-0006.md) y/o [SEP-24: Alojamiento de Depósito y Retirado](https://github.com/stellar/stellar-protocol/blob/master/ecosystem/sep-0024.md).

:::info

En Stellar se definen los estándares de interoperabilidad decididos por la comunidad. Lea más en nuestra [sección de Etiquetas](../../../learn/fundamentals/stellar-ecosystem-proposals.mdx).

:::

Nuestras integraciones también utilizarán los siguientes SEP:

- [SEP-1: Stellar TOML](https://github.com/stellar/stellar-protocol/blob/master/ecosystem/sep-0001.md) - un archivo que proporciona un lugar para que Internet encuentre información sobre la integración estelar de una organización
- [SEP-9: KYC Fields](https://github.com/stellar/stellar-protocol/blob/master/ecosystem/sep-0009.md) - define una lista de campos KYC estándar para usarse en protocolos del ecosistema Stellar
- [SEP-10: Stellar Web Authentication](https://github.com/stellar/stellar-protocol/blob/master/ecosystem/sep-0010. d) - define la forma estándar para que los clientes creen sesiones web autenticadas en nombre de un usuario que tiene una cuenta Stellar
- [SEP-12: API de KYC](https://github.com/stellar/stellar-protocol/blob/master/ecosystem/sep-0012.md) - define una forma estándar para que los clientes Stellar suban información de KYC a los anclos

## Buscando activos anclados

BasicPay se encarga de todos los detalles de transferencia de fondos en la página `/dashboard/transfers`. Véalo en acción aquí: https://basicpay.pages.dev/dashboard/transfers

Necesitamos nuestra aplicación para saber cómo puede comunicarse con los ancestros para obtener información sobre activos e infraestructura. Lo primero que haremos es determinar si el usuario tiene líneas de confianza para cualquier activo que tenga un campo `home_domain` establecido en la red Stellar. La presencia de ese campo en la cuenta de un emisor nos dice que el activo _puede_ ser conectado a los raíles Stellar existentes que permiten transferencias del activo. Si está presente, mostraremos algunos elementos interactivos al usuario para ese activo/dominio.

```js title=/src/routes/dashboard/transfers/+page.js
import { fetchAssetsWithHomeDomains } from "$lib/stellar/horizonQueries";

/** @type {import('./$types'). ageLoad} */
export async function load({ parent }) {
  const { balances } = await parent();
  return {
    /** @type {import('$lib/stellar/horizonQueries'). omeDomainBalance[]} */
    homeDomainBalances: await fetchAssetsWithHomeDomains(balances),
  };
}
```

**Fuente:** https://github.com/stellar/basic-payment-app/blob/main/src/routes/dashboard/transfers/+page.js

Ahora sabemos si hay algún activo que el usuario tenga que tener la infraestructura necesaria para las transferencias autenticadas y podemos consultar el dominio en cuestión para los detalles relevantes.
