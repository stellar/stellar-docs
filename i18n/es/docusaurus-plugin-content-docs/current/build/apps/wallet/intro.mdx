## Trabajando con el SDK

Empecemos con la clase principal que proporciona toda la funcionalidad de SDK. Se aconseja que se comparta un objeto monetario a través de la aplicación. Crear una cartera con una configuración predeterminada conectada a la red Testnet de Stellar es simple:

<CodeExample>

```kotlin
val wallet = Wallet(StellarConfiguration.Testnet)
```

```ts
let cartera = walletSdk.Wallet.TestNet();
```

```dart
var wallet = Wallet.testNet;
```

</CodeExample>

La instancia del monedero puede configurarse aún más. Por ejemplo, para conectarse a la red pública:

<CodeExample>

```kotlin
val walletTestnet = Wallet(StellarConfiguration(Network.TESTNET, "https://Budgeton-testnet.stellar.org"))
```

```typescript
let wallet = new Wallet({
  stellarConfiguration: StellarConfiguration.MainNet(),
});
```

```dart
var wallet = Wallet(StellarConfiguration.publicNet);
```

</CodeExample>

## Básicos estelar

El SDK de la cartera proporciona alguna funcionalidad extra encima del [SDK Horizon]. Para la interacción con la red Stellar, la cartera SDK cubre sólo los fundamentos utilizados en un flujo típico de cartera. Para casos de uso más avanzado, el [Horizon SDK] subyacente debería ser usado en su lugar.

Para interactuar con la instancia Horizon configurada en los pasos anteriores, simplemente hace:

<CodeExample>

```kotlin
val stellar = wallet.stellar()
```

```typescript
const stellar = wallet.stellar();
```

```dart
var stellar = wallet.stellar();
```

</CodeExample>

Este ejemplo creará una clase Stellar que administra la conexión al servicio Horizon.

:::note

La configuración predeterminada se conecta a la instancia pública Stellar Horizon. Puede cambiar este comportamiento como se describe [above](#working-with-the-sdk).

:::

Puedes leer más sobre cómo trabajar con la red Stellar en la [sección correspondiente](./stellar.mdx).

## Ancla Básicos

El uso primario del SDK es proporcionar una forma fácil de conectarse a los anclos a través de conjuntos de protocolos conocidos como SEP. Veamos cómo conectarnos con el ancla de prueba Stellar:

<CodeExample>

```kotlin
val anchor = wallet.anchor("https://testanchor.stellar.org")
```

```typescript
let anchor = wallet.anchor({ homeDomain: "testanchor.stellar.org" });
```

```dart
var anchor = wallet.anchor("testanchor.stellar.org")
```

</CodeExample>

<LanguageSpecific ts={<AllowHttpInfo />} />

Y la interacción más básica de obtener un [SEP-1]\: Archivo de información estelar:

<CodeExample>

```kotlin
suspender anchorToml(): TomlInfo {
  return anchor.getInfo()
}
```

```typescript
let resp = await anchor.sep1();
```

```dart
var resp = await anchor.sep1();
```

</CodeExample>

La clase de anclaje también soporta [SEP-10]\: autenticación estelar y [SEP-24]\: funciones de depósito alojado y retiro.
