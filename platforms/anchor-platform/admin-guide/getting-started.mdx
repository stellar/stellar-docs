# Getting Started

This guide will help you quickly get the Anchor Platform running locally using Docker Compose.

## Prerequisites

- [Docker](https://www.docker.com/get-started) and Docker Compose installed
- At least 4GB of available RAM
- Ports 3000, 5432, 5433, 8080, 8085, 8091, 29092, and 29096 available

## Quick Start

The fastest way to get started is using the `quick-run` docker-compose setup included in the Anchor Platform repository.

### 1. Navigate to the quick-run directory

```bash
cd anchor-platform/quick-run
```

### 2. Start all services

```bash
docker-compose up -d
```

This command will start all required services:

- **Platform** (ports 8080, 8085) - SEP server, platform API, event processor, and stellar observer
- **Reference Server** (port 8091) - Reference anchor backend implementation
- **SEP-24 UI** (port 3000) - Interactive flow reference UI for testing
- **Kafka** (port 29092) - Event processing message broker
- **PostgreSQL** (ports 5432, 5433) - Databases for platform and reference server

### 3. Verify the platform is running

Wait a few moments for services to initialize, then verify the platform is responding:

```bash
curl http://localhost:8080/.well-known/stellar.toml
```

You should see the Stellar TOML configuration file returned.

### 4. Check service status

```bash
docker-compose ps
```

All services should show as "Up" in the status column.

## What's Included

The `quick-run` setup includes:

- **docker-compose.yaml** - Complete service definitions with all dependencies
- **dev.env** - Pre-configured environment variables
- **config/** - Required configuration files:
  - `assets.yaml` - Asset definitions
  - `clients.yaml` - Client configurations
  - `reference-config.yaml` - Reference server settings
  - `stellar.localhost.toml` - SEP-1 TOML file

## Configuration

### Environment Variables

The `dev.env` file contains all configuration. Key settings include:

- **Stellar Network**: Configured to use Soroban testnet
- **Database**: PostgreSQL with default credentials
- **Kafka**: Plaintext configuration (no SASL/SSL)
- **SEP Protocols**: All major SEPs enabled (SEP-1, SEP-6, SEP-10, SEP-12, SEP-24, SEP-31, SEP-38, SEP-45)

### Customizing Configuration

To modify settings:

1. Edit `dev.env` for environment variables
2. Edit files in `config/` for service-specific configurations
3. Restart services: `docker-compose restart`

## Accessing Services

Once running, you can access:

- **SEP Server**: http://localhost:8080
- **Platform API**: http://localhost:8085
- **Reference Server**: http://localhost:8091
- **SEP-24 UI**: http://localhost:3000

## Stopping Services

To stop all services:

```bash
docker-compose down
```

To stop and remove all data (including databases):

```bash
docker-compose down -v
```

## Viewing Logs

View logs for all services:

```bash
docker-compose logs -f
```

View logs for a specific service:

```bash
docker-compose logs -f platform
docker-compose logs -f reference-server
docker-compose logs -f kafka
```

## Troubleshooting

### Port conflicts

If you see port binding errors, ensure the required ports are not in use:

```bash
# Check if ports are in use
lsof -i :8080
lsof -i :8085
lsof -i :8091
```

### Services not starting

Check service logs for errors:

```bash
docker-compose logs platform
docker-compose logs reference-server
```

### Database connection issues

Ensure databases are running:

```bash
docker-compose ps db reference-db
```

If databases are not running, restart them:

```bash
docker-compose restart db reference-db
```
