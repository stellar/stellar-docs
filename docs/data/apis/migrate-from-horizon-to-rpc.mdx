Applications using [Horizon's REST-like API](./horizon/api-reference/README.mdx) will need to be updated to use the [RPC JSON-RPC API](./rpc/api-reference/README.mdx) when migrating from [Horizon](./horizon/README.mdx) to [RPC](./rpc/README.mdx). This guide provides an overview of the key differences between the two APIs and how to migrate your application.

## Request / Response Format

Horizon's REST-like API uses HTTP methods and status codes to communicate with clients. Responses are JSON in the HAL format. See [Horizon's Response Format](./horizon/api-reference/structure/response-format.mdx).

RPC's JSON-RPC API uses JSON-RPC 2.0 to communicate with clients. Requests to the API are JSON objects that contain one or more method invocations. Responses are also JSON objects that contain a result for each invocation in the request. See [JSON-RPC](./rpc/api-reference/structure/json-rpc.mdx).

Both formats utilise JSON for the overall structure which are relatively simple and do not require any special client code, although there are client [SDKs](../../tools/sdks/README.mdx) available. Some values contained within are XDR encoded and can be decoded using Stellar [SDKs](../../tools/sdks/README.mdx).

## Endpoint Mapping

Applications that use the following Horizon endpoints can typically migrate directly to the RPC using the referenced methods.

Endpoints without mappings do not have a direct replacement in the RPC API. To build similar functionality in an application, please consider partnering with an indexer or using the information listed below to build your own indexed representation of horizon endpoints. Consider using other [Data](../analytics/README.mdx) products for analytics use cases.

:::tip

The [`getTransactions`](./rpc/api-reference/methods/getTransactions.mdx) method can be used to retrieve events batched by transaction. The events are contained in the meta XDR of the transaction (field `resultMetaXdr`).

:::

:::warning

The [`getEvents`](./rpc/api-reference/methods/getEvents.mdx) method is not a direct replacement for Horizon's endpoints.

The method returns a stream of events that in the current protocol only include events from contracts. In the near future as a result of [CAP-67](https://github.com/stellar/stellar-protocol/blob/master/core/cap-0067.md) this method will be expanded to include events from non-contract operations.

In the interim the [`getTransactions`](./rpc/api-reference/methods/getTransactions.mdx) method can be used to retrieve the meta XDR of transactions containing non-contract operations to determine what movements of value have occurred. The meta XDR also contains events from contracts.

:::
