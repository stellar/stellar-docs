---
title: Migrating from Horizon to RPC
sidebar_position: 0
---

When migrating from Horizon to RPC, you will need to update your API calls to use the new RPC functions available on the RPC endpoint. Below is a table of the most common API calls and their equivalent RPC function calls.

| Horizon Endpoint | RPC Functions |
| --- | --- |
| [POST /transactions] | [`sendTransaction`] |
| [GET /ledgers] | [`getLedgers`] |
| [GET /operations] | [`getTransactions`] |
| [GET /transactions] | [`getTransactions`] |
| [GET /transactions/id] | [`getTransaction`] |
| [GET /accounts], [GET /offers], [GET /liquidity_pools], [GET /claimable_balances], [GET /trustlines], etc | [`getLedgerEntries`] |
| [GET /payments] | [`getEvents`]\* |
| [GET /trades] | [`getEvents`]\* |
| [GET /effects] | [`getEvents`]\* |

\* The [`getEvents`] RPC function is not a direct replacement today for the [`GET /payments`], [`GET /trades`], [`GET /effects`]. The [`getEvents`] function returns a list of events that occurred in a ledger that currently only come from contract invocations and events relating to non-contract operations have not yet been implemented. They will be implemented in [CAP-67].
